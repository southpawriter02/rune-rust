v0.3.2a: The Campfire (Core Logic)
Goal: Implement the RestService to handle resource deduction (Rations/Water), apply recovery formulas, and manage the "Exhausted" state when supplies are missing.

1. Architecture & Data Flow
The RestService acts as a transaction coordinator between the InventoryService (consumables), StatCalculationService (max values), and TraumaService (stress).
Workflow: The Rest Transaction
	1	Input: System calls RestService.PerformRest(Character, RestType).
	2	Resource Validation: Service checks Inventory for item_ration (1) and item_water (1).
	◦	If Wilderness: Deduct items. If missing, flag IsExhausted.
	◦	If Sanctuary: No supplies required (or auto-supplied by settlement).
	3	Calculation:
	◦	Calculate Max HP/Stamina (ensure baseline is current).
	◦	Calculate Recovery Amount based on RestType, Attribute.Sturdiness, and Attribute.Will.
	4	Application:
	◦	Update CurrentHP, CurrentStamina, PsychicStress.
	◦	If IsExhausted, apply [Exhausted] Status Effect (halves regen, prevents running).
	5	Time: Advance World Time by 8 hours (480 minutes).
	6	Output: Return RestResult (deltas for UI display).

2. Logic Decision Trees
A. Resource Logic
Input: Character, RestType
	1	Is RestType == Sanctuary?
	◦	Yes: Supplies Not Needed. Proceed to Recovery (Full).
	2	Is RestType == Wilderness?
	◦	Check: Does Inventory contain 1 Ration AND 1 Water?
	▪	Yes: Remove 1 Ration, 1 Water. Supplies Used. Proceed to Recovery (Standard).
	▪	No: Supplies Missing. Proceed to Recovery (Partial) AND Apply [Exhausted].
B. Recovery Logic (Standard)
Input: Character, Attributes
	1	HP Recovery:
	◦	Base = 10 + (Sturdiness * 2)
	◦	NewHP = Math.Min(MaxHP, CurrentHP + Base)
	◦	Note: Wilderness rest does not fully heal severe wounds.
	2	Stamina Recovery:
	◦	NewStamina = MaxStamina (Rest always restores full fatigue).
	3	Stress Recovery:
	◦	Reduction = Will * 5
	◦	NewStress = Math.Max(0, CurrentStress - Reduction)
	4	Corruption:
	◦	No Change. (Corruption requires specific rituals/items).
C. Recovery Logic (Sanctuary)
Input: Character
	1	HP: CurrentHP = MaxHP (Full Heal).
	2	Stamina: CurrentStamina = MaxStamina (Full Refresh).
	3	Stress: CurrentStress = 0 (Safe Haven).
	4	Corruption: No Change.

public class RestService : IRestService
{
    private readonly IInventoryService _inventory;
    private readonly IStatusEffectService _statusEffects;
    private readonly ITraumaService _trauma;

    public async Task<RestResult> PerformRestAsync(Character character, RestType type)
    {
        bool hasSupplies = true;
        bool applyExhaustion = false;

        // 1. Resource Consumption
        if (type == RestType.Wilderness)
        {
            bool hasRation = await _inventory.HasItemAsync(character, "Ration");
            bool hasWater = await _inventory.HasItemAsync(character, "Water");

            if (hasRation && hasWater)
            {
                await _inventory.RemoveItemAsync(character, "Ration", 1);
                await _inventory.RemoveItemAsync(character, "Water", 1);
            }
            else
            {
                hasSupplies = false;
                applyExhaustion = true;
            }
        }

        // 2. Calculate Recovery
        int hpDelta = 0;
        int stressDelta = 0;
        int oldHp = character.CurrentHp;
        int oldStress = character.PsychicStress;

        if (type == RestType.Sanctuary)
        {
            // Full Reset
            character.CurrentHp = character.MaxHp;
            character.CurrentStamina = character.MaxStamina;
            character.PsychicStress = 0;
            _statusEffects.RemoveEffect(character, StatusEffectType.Exhausted); // Sanctuary cures exhaustion
        }
        else // Wilderness
        {
            // HP: Base + Sturdiness scaling
            int healAmount = 10 + (character.GetAttribute(Attribute.Sturdiness) * 2);
            if (applyExhaustion) healAmount /= 2; // Penalty

            character.CurrentHp = Math.Min(character.MaxHp, character.CurrentHp + healAmount);

            // Stamina: Always full unless exhausted
            character.CurrentStamina = applyExhaustion ? (character.MaxStamina / 2) : character.MaxStamina;

            // Stress: Will scaling
            int stressRelief = character.GetAttribute(Attribute.Will) * 5;
            if (applyExhaustion) stressRelief = 0; // No mental relief if starving

            character.PsychicStress = Math.Max(0, character.PsychicStress - stressRelief);

            if (applyExhaustion)
            {
                _statusEffects.ApplyEffect(character, StatusEffectType.Exhausted, duration: -1); // Permanent until cured
            }
        }

        // 3. Result
        hpDelta = character.CurrentHp - oldHp;
        stressDelta = oldStress - character.PsychicStress;

        return new RestResult(hpDelta, character.MaxStamina - character.CurrentStamina, stressDelta, hasSupplies, applyExhaustion);
    }
}

# Changelog: v0.3.2a - The Campfire
**Release Date:** 2025-XX-XX

## Summary
The foundation of the Resting System is now live. Characters can now recover HP, Stamina, and Stress by resting, but wilderness survival demands resources. This update introduces the mechanic of consuming Rations and Water to rest effectively, and the consequences of failing to do so.

## Features
- **Resting Types:**
    - **Sanctuary Rest:** Requires a safe haven (Runic Anchor). Fully restores HP, Stamina, and clears all Stress. Cures Exhaustion. No supplies needed.
    - **Wilderness Rest:** Available anywhere (subject to Ambush in future updates). Restores partial HP based on Sturdiness and reduces Stress based on Willpower. Requires supplies.
- **Resource Management:**
    - Resting in the wild consumes **1 Ration** and **1 Water**.
    - **Exhaustion:** Resting without supplies applies the **[Exhausted]** status (halves regen, penalties to stats) and significantly reduces the benefits of the rest.
- **Recovery Formulas:**
    - Wilderness Heal: `10 + (Sturdiness * 2)`
    - Wilderness Stress Relief: `Will * 5`

## Technical
- Implemented `RestService` to handle state transitions and inventory transactions.
- Added `[Exhausted]` to the Status Effect registry.
