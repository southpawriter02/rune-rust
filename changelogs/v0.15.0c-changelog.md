# v0.15.0c Changelog

## Summary
Transition skill checks from sum-based to success-counting mechanics with 6-tier outcome classification.

## Added

### Domain Layer
- **`SkillOutcome`** enum in `Domain/Enums/` - 6-tier outcome classification:
  - `CriticalFailure` (fumble or margin ≤ -3)
  - `Failure` (margin -2 to -1)
  - `MarginalSuccess` (margin 0)
  - `FullSuccess` (margin 1-2)
  - `ExceptionalSuccess` (margin 3-4)
  - `CriticalSuccess` (margin ≥ 5)

- **`SkillOutcomeExtensions`** in `Domain/Extensions/`:
  - `IsSuccess()` - True for MarginalSuccess+
  - `IsFailure()` - True for Failure and CriticalFailure
  - `IsCritical()` - True for CriticalSuccess/CriticalFailure
  - `GetMarginRange()` - Returns margin bounds for tier
  - `ToSuccessLevel()` - Maps to legacy 4-tier enum
  - `GetDisplayName()` / `GetAbbreviation()` - Formatted strings

### SkillCheckResult Properties (v0.15.0c)
- `NetSuccesses` - Primary value for success-counting
- `Margin` - NetSuccesses - DifficultyClass
- `Outcome` - 6-tier SkillOutcome classification
- `IsFumble` - True when DiceResult.IsFumble

### SkillCheckResultDto Properties
- `NetSuccesses`, `Outcome`, `IsFumble` added to DTO

## Changed

### SkillCheckResult
- Constructor now calculates success-counting properties
- `ClassifyOutcome()` static method determines 6-tier outcome
- `ToString()` updated for success-counting format: `"Skill: X net (YS - ZB) vs DC N → Outcome [margin: ±M]"`
- `ToLegacyString()` added for backward compatibility

### SkillCheckService
- **Contested checks** now compare `NetSuccesses` instead of `TotalResult`
- Logging uses `Outcome` and `NetSuccesses`

### SearchService
- Uses `TotalResult` for DC comparison (intentional - roll total vs discovery DC)
- Added pragma suppressions for obsolete warnings

### FleeAttemptResult
- `ToString()` now uses `NetSuccesses`

## Deprecated
- `SkillCheckResult.TotalResult` - Use `NetSuccesses` for success-counting
- `SkillCheckResult.SuccessLevel` - Use `Outcome` for 6-tier classification

## Tests Updated
- `SkillCheckResultTests.cs` - 16 tests for success-counting mechanics
- `DiceDescriptorServiceTests.cs` - Fixed DC values for new mechanics
