Here is the comprehensive implementation plan for v0.2.4c: The Omen (Telegraphs & Chants).
This version completes the Adversary milestone by implementing multi-turn attacks for enemies. It distinguishes between standard attacks and "Ultimate" moves that require setup turns, introducing the Chant/Charge mechanic to the AI. This creates tactical windows where players must decide whether to defend, flee, or interrupt the enemy before the blow lands.

v0.2.4c: The Omen (Telegraphs & Chants)
Goal: Implement the TelegraphedAbilityService and integrate Chant logic into the Enemy AI. Enemies must clearly broadcast powerful attacks one turn in advance via narrative logs ("The Draugr raises its axe high..."), giving players a window to react.

1. Architecture & Data Flow
The Charge-Release Cycle
	1	AI Decision: The EnemyAIService selects a high-impact ability tagged with ChargeTurns > 0.
	2	Initiation (Turn 1):
	◦	The CombatService detects the charge requirement.
	◦	Instead of dealing damage immediately, it applies the [Chanting] or [Charging] status to the enemy.
	◦	The Omen: A specific TelegraphMessage is logged (e.g., "The Construct's core begins to glow violent red.").
	3	Interruption Window (Player Turn):
	◦	The enemy is effectively "Stunned" (cannot dodge/parry) but often gains [Fortified].
	◦	If the enemy takes damage > InterruptThreshold, the Charge is broken.
	4	Execution (Turn 2):
	◦	The AI detects it is [Chanting].
	◦	It forces the execution of the stored payload (The "Release").
	◦	The Ability resolves (Damage/Status).

2. Logic Decision Trees
A. AI Planning Logic (Chant Awareness)
Input: Enemy Combatant
	1	Check Status: Is Enemy [Chanting]?
	◦	Yes: Force Action -> CompleteChant. (Do not evaluate other abilities).
	◦	No: Proceed to standard Utility Scoring (v0.2.4b).
	2	Scoring Modification for Charge Abilities:
	◦	Base Score: High (70+).
	◦	Risk Penalty: If Enemy HP < 30% -> Score -50 (Too risky to stand still).
	◦	Opportunity Bonus: If Player is [Stunned] -> Score +30 (Free hit).
B. Interruption Logic
Input: DamageTaken Event
	1	Check Status: Is Victim [Chanting]?
	◦	No: Proceed standard damage.
	◦	Yes:
	1	Calculate Threshold: MaxHP * 0.10 (10% HP damage to break).
	2	Compare: Is Damage > Threshold?
	▪	Yes: Remove [Chanting]. Log "Interrupted!". Apply [Stunned] (1 turn).
	▪	No: Log "The enemy maintains focus through the pain."

7. Draft Changelog (v0.2.4c)
# v0.2.4c Changelog: The Omen (Telegraphs & Chants)
**Release Date:** 2025-12-22

## Summary
This release implements the "Telegraph" system for enemy AI. Powerful enemy abilities now require a setup turn ("Charging"), displaying a foreboding message to the player. Players can use this window to defend or attempt to interrupt the attack by dealing sufficient damage.

## New Features
*   **Telegraphed Attacks:** Enemies can now use multi-turn abilities.
*   **Chanting Status:** Visual indicator for enemies preparing an attack.
*   **Interruption Mechanic:** Dealing damage > Threshold breaks enemy concentration.

## Technical Changes
*   Extended `ActiveAbility` with `ChargeTurns` and `TelegraphMessage`.
*   Updated `EnemyAIService` to respect `Chanting` state locks.
*   Updated `CombatService` to process interruptions on damage events.
