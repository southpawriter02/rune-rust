	•	Crafting Flow:
	1	Input: Player types craft [ItemName].
	2	Validation: CraftingService checks Inventory for required materials and Character for required Specialization/Attribute.
	3	Resolution: DiceService rolls WITS vs Recipe DC.
	4	Transaction: On success, InventoryService removes materials and adds the result. On fail, materials are consumed (partial or full).
2. Logic Decision Trees
	•	Validation Logic:
	1	Does Character have the Recipe? (Default: Yes for Basic, check list for Advanced).
	2	Are ingredients present? (Check Inventory).
	3	Are tools/station present? (Check Room.Features or Inventory).
	4	Result: Proceed or Return Error.
	•	Resolution Logic:
	1	Roll WITS + Trade Bonus (from Specialization).
	2	Compare to Recipe.BaseDC.
	3	Result:
	▪	Net Successes < 0: Failure (Lose materials).
	▪	Net Successes 0-4: Success (Standard Quality).
	▪	Net Successes 5+: Masterwork (Enhanced Stats).

public async Task<CraftingResult> CraftItemAsync(Character character, string recipeName)
{
    var recipe = _recipeRegistry.GetByName(recipeName);
    if (recipe == null) return CraftingResult.Failure("Unknown recipe.");

    // 1. Validate Ingredients
    if (!await _inventory.HasIngredientsAsync(character, recipe.Ingredients))
    {
        return CraftingResult.Failure("Missing ingredients.");
    }

    // 2. Roll Check
    var wits = character.GetAttribute(Attribute.Wits);
    var roll = _dice.Roll(wits, $"Crafting {recipe.Name}");
    var netSuccesses = roll.Successes - recipe.BaseDc; // Simplified for v0.3.1a

    // 3. Process Result
    if (netSuccesses >= 0)
    {
        await _inventory.RemoveIngredientsAsync(character, recipe.Ingredients);
        await _inventory.AddItemAsync(character, recipe.OutputItemId);
        return CraftingResult.Success($"Created {recipe.Name}!");
    }
    else
    {
        await _inventory.RemoveIngredientsAsync(character, recipe.Ingredients); // Loss on fail
        return CraftingResult.Failure("Crafting failed. Materials ruined.");
    }
}

# Changelog: v0.3.1 - The Crafting Bench
**Release Date:** 2025-XX-XX

## Summary
The Crafting System is now live, allowing players to create items, repair gear, and brew potions. This update transforms "Junk" items into valuable "Scrap" and introduces the risk/reward mechanics of Runeforging.

## Features
- **Four Crafting Trades:**
    - **Bodging:** Repair gear and create improvised tools from Scrap.
    - **Field Medicine:** Craft bandages and poultices (requires campfire/safe zone).
    - **Alchemy:** Brew potions. Careful: fumbles cause explosions.
    - **Runeforging:** Inscribe gear with buffs. Failure causes **Corruption**.
- **Quality System:** High WITS rolls can produce **Masterwork** (Optimized) items with better stats.
- **Salvage:** Use `salvage <item>` to break down useless gear into Scrap.
- **Repair:** Use `repair <item>` to restore durability using Scrap.

## Technical
- Implemented `CraftingService` and `RecipeRegistry`.
- Integrated `TraumaService` for crafting penalties.
- Added `durability` tracking to Equipment.
