## Overview

This sub-version transforms the Bestiary (Scavenger's Journal) from a passive lore repository into an active tactical tool. The **AnalysisService** bridges the Data Capture System (v0.1.3) with the Combat Engine (v0.2.0+), allowing players to query enemy statistics mid-combat based on their collected knowledge.

**Design Philosophy: "Knowledge as Power"**
- Players who explore thoroughly and collect lore fragments are rewarded with tactical foresight
- The Jötun-Reader archetype (Adepts) embodies this pillar, receiving a +1 tier bonus when inspecting enemies
- Information is progressively revealed across 4 knowledge tiers, encouraging completionism

**Current State Analysis:**
- `IDataCaptureService.GetCompletionPercentageAsync()` exists but is only used for journal display
- `Combatant.EnemySource.TemplateId` links enemies to `CodexEntry` via keyword matching
- `ICodexEntryRepository` lacks a `GetByTemplateIdAsync()` method for direct lookup
- No `inspect` command exists in `CommandParser`
- No enemy intel popup exists in `CombatScreenRenderer`

**Target State:**
- New `AnalysisService` queries DataCaptureService for completion percentage
- New `KnowledgeTier` enum and `EnemyIntel` DTO for tiered data presentation
- New `ICodexEntryRepository.GetByTemplateIdAsync()` for template → entry lookup
- `CommandParser` supports `inspect <target>` during combat
- `InspectWindowRenderer` displays a modal popup overlay with masked enemy data

## Architecture & Data Flow

### Components

| Component | Layer | Responsibility |
|-----------|-------|----------------|
| `KnowledgeTier` | Core/Enums | Enum defining intel levels: Unknown, Obscured, Known, Mastered |
| `EnemyIntel` | Core/Models | Immutable DTO containing masked enemy data for UI display |
| `IAnalysisService` | Core/Interfaces | Service contract for enemy inspection |
| `AnalysisService` | Engine/Services | Calculates knowledge tier and masks enemy data accordingly |
| `ICodexEntryRepository` (extended) | Core/Interfaces | New `GetByTemplateIdAsync()` method |
| `CodexEntryRepository` (updated) | Persistence/Repositories | Implements template ID lookup |
| `ParseResult` (extended) | Engine/Services | New `RequiresInspect` and `InspectTarget` properties |
| `CommandParser` (updated) | Engine/Services | Parses `inspect <target>` command |
| `CombatService` (updated) | Engine/Services | Handles inspect action, calls AnalysisService |
| `InspectWindowRenderer` | Terminal/Rendering | Spectre.Console modal popup for intel display |
| `CombatScreenRenderer` (updated) | Terminal/Services | Integrates inspect window overlay |

## Logic Decision Trees

### A. Knowledge Tier Calculation

**Input:** `completionPercent` (0-100), `playerArchetype` (enum)

```
START: Receive completion percentage
  │
  ├─► Map to Base Tier:
  │     │
  │     ├─► 0% - 24%   → Unknown   (Tier 0)
  │     ├─► 25% - 74%  → Obscured  (Tier 1)
  │     ├─► 75% - 99%  → Known     (Tier 2)
  │     └─► 100%       → Mastered  (Tier 3)
  │
  └─► Apply Archetype Modifier:
        │
        ├─► Is archetype Adept or Mystic (Jötun-Reader)?
        │     │
        │     └─► YES: Effective Tier = Min(BaseTier + 1, Mastered)
        │           - Log: "Scholar's Insight applied"
        │
        └─► NO: Effective Tier = Base Tier

OUTPUT: Effective KnowledgeTier
```

### B. Data Masking Logic

**Input:** `KnowledgeTier tier`, `Combatant enemy`

```
Tier 0 (Unknown):
  ├─► Show: Name, FlavorDescription
  ├─► HP Display: "???"
  ├─► Armor Display: "???"
  ├─► Type Tags: Hidden (empty list)
  ├─► Weaknesses: Hidden
  └─► Abilities: Hidden

Tier 1 (Obscured):
  ├─► Show: Name, FlavorDescription
  ├─► HP Display: Narrative ("Healthy"/"Wounded"/"Critical")
  ├─► Armor Display: Narrative ("Light"/"Medium"/"Heavy")
  ├─► Type Tags: Visible (e.g., "Mechanical", "Undead")
  ├─► Weaknesses: Hidden
  └─► Abilities: Hidden

Tier 2 (Known):
  ├─► Show: Name, FlavorDescription
  ├─► HP Display: Exact values ("45/60")
  ├─► Armor Display: Exact soak value ("4")
  ├─► Type Tags: Visible
  ├─► Weaknesses: Visible (e.g., "Fire", "Lightning")
  └─► Abilities: Hidden

Tier 3 (Mastered):
  ├─► Show: All above
  ├─► Abilities: Full list with damage and cooldowns
  └─► AI Hints: Behavior patterns (e.g., "Prioritizes healers")
```

### C. Target Resolution Logic

**Input:** `targetName` (string), `combatState.TurnOrder`

```
START: Receive target name
  │
  ├─► Normalize: ToLowerInvariant()
  │
  ├─► Search TurnOrder for matching Combatant:
  │     │
  │     ├─► Exact match on Name (case-insensitive)?
  │     │     └─► YES: Return combatant
  │     │
  │     ├─► Partial match (Name.Contains)?
  │     │     ├─► Single match? Return combatant
  │     │     └─► Multiple matches? Return first + log warning
  │     │
  │     └─► No match? Return null
  │
  └─► Validate Result:
        │
        ├─► Is result null?
        │     └─► Return error: "No enemy matching '{target}' found."
        │
        ├─► Is result a player combatant?
        │     └─► Return error: "You cannot inspect allies."
        │
        └─► Valid enemy? Proceed to analysis
```

### D. HP Narrative Mapping (Obscured Tier)

**Input:** `currentHp`, `maxHp`

```
HealthPercent = (CurrentHp / MaxHp) × 100

HealthPercent >= 75%  → "Healthy"
HealthPercent >= 50%  → "Bloodied"
HealthPercent >= 25%  → "Wounded"
HealthPercent < 25%   → "Critical"
```

### E. Armor Narrative Mapping (Obscured Tier)

**Input:** `armorSoak`

```
ArmorSoak == 0     → "Unarmored"
ArmorSoak <= 2     → "Light Armor"
ArmorSoak <= 4     → "Medium Armor"
ArmorSoak <= 6     → "Heavy Armor"
ArmorSoak > 6      → "Fortified"
```

### E. Engine Layer: Analysis Service Implementation

**File:** `RuneAndRust.Engine/Services/AnalysisService.cs`

```csharp
using Microsoft.Extensions.Logging;
using RuneAndRust.Core.Entities;
using RuneAndRust.Core.Enums;
using RuneAndRust.Core.Interfaces;
using RuneAndRust.Core.Models.Combat;

namespace RuneAndRust.Engine.Services;

/// <summary>
/// Analyzes enemy combatants to provide knowledge-tiered intelligence (v0.3.22b).
/// Integrates Data Capture completion with combat display.
/// </summary>
/// <remarks>See: SPEC-CAPTURE-001 for Data Capture System integration.</remarks>
public class AnalysisService : IAnalysisService
{
    private readonly IDataCaptureService _captureService;
    private readonly ICodexEntryRepository _codexRepo;
    private readonly ILogger<AnalysisService> _logger;

    // Archetypes that receive Scholar's Insight bonus
    private static readonly HashSet<ArchetypeType> ScholarArchetypes = new()
    {
        ArchetypeType.Adept,   // Jötun-Reader
        ArchetypeType.Mystic   // Also knowledge-focused
    };

    public AnalysisService(
        IDataCaptureService captureService,
        ICodexEntryRepository codexRepo,
        ILogger<AnalysisService> logger)
    {
        _captureService = captureService;
        _codexRepo = codexRepo;
        _logger = logger;
    }

    /// <inheritdoc/>
    public async Task<EnemyIntel> InspectAsync(Combatant target, Character player)
    {
        _logger.LogInformation(
            "[Combat] Inspecting {TargetName} for Character {CharacterId}",
            target.Name, player.Id);

        // 1. Get template ID from enemy source
        var templateId = target.EnemySource?.TemplateId;

        // 2. Calculate knowledge tier
        var tier = await GetKnowledgeTierAsync(templateId, player.Id, player.Archetype);
        var bonusApplied = ScholarArchetypes.Contains(player.Archetype);

        _logger.LogDebug(
            "[Combat] Knowledge tier calculated: {Tier} (Completion: ?, Bonus: {Bonus})",
            tier, bonusApplied);

        // 3. Build masked intel DTO
        return BuildEnemyIntel(target, tier, bonusApplied);
    }

    /// <inheritdoc/>
    public async Task<KnowledgeTier> GetKnowledgeTierAsync(
        string? templateId,
        Guid characterId,
        ArchetypeType archetype)
    {
        // Handle missing template (dynamic spawn)
        if (string.IsNullOrEmpty(templateId))
        {
            _logger.LogDebug("[Combat] No template ID, defaulting to Unknown tier");
            return ApplyArchetypeBonus(KnowledgeTier.Unknown, archetype);
        }

        // Lookup CodexEntry by template ID
        var entry = await _codexRepo.GetByTemplateIdAsync(templateId);
        if (entry == null)
        {
            _logger.LogDebug(
                "[Combat] No Codex entry found for template '{TemplateId}'",
                templateId);
            return ApplyArchetypeBonus(KnowledgeTier.Unknown, archetype);
        }

        // Get completion percentage
        var completion = await _captureService.GetCompletionPercentageAsync(entry.Id, characterId);

        _logger.LogDebug(
            "[Combat] Codex entry '{Title}' completion: {Completion}%",
            entry.Title, completion);

        // Calculate base tier from completion
        var baseTier = completion switch
        {
            >= 100 => KnowledgeTier.Mastered,
            >= 75 => KnowledgeTier.Known,
            >= 25 => KnowledgeTier.Obscured,
            _ => KnowledgeTier.Unknown
        };

        // Apply archetype bonus
        return ApplyArchetypeBonus(baseTier, archetype);
    }

    /// <summary>
    /// Applies Scholar's Insight bonus for knowledge-focused archetypes.
    /// </summary>
    private KnowledgeTier ApplyArchetypeBonus(KnowledgeTier baseTier, ArchetypeType archetype)
    {
        if (!ScholarArchetypes.Contains(archetype))
            return baseTier;

        var boostedTier = (KnowledgeTier)Math.Min((int)baseTier + 1, (int)KnowledgeTier.Mastered);

        if (boostedTier != baseTier)
        {
            _logger.LogDebug(
                "[Combat] Scholar's Insight applied: {BaseTier} → {BoostedTier}",
                baseTier, boostedTier);
        }

        return boostedTier;
    }

    /// <summary>
    /// Builds the EnemyIntel DTO with fields masked according to tier.
    /// </summary>
    private EnemyIntel BuildEnemyIntel(Combatant target, KnowledgeTier tier, bool bonusApplied)
    {
        return new EnemyIntel(
            Name: target.Name,
            Description: GetDescription(target),
            Tier: tier,
            HpDisplay: GetHpDisplay(target, tier),
            ArmorDisplay: GetArmorDisplay(target, tier),
            TypeTags: tier >= KnowledgeTier.Obscured ? target.Tags : new List<string>(),
            Weaknesses: tier >= KnowledgeTier.Known ? GetWeaknesses(target) : new List<string>(),
            Abilities: tier >= KnowledgeTier.Mastered ? GetAbilities(target) : new List<AbilityDisplayInfo>(),
            AiHint: tier >= KnowledgeTier.Mastered ? GetAiHint(target) : null,
            CompletionPercent: 0, // Will be set by caller if needed
            WasArchetypeBonusApplied: bonusApplied
        );
    }

    #region Display Helpers

    private static string GetDescription(Combatant target)
    {
        // Use enemy source description or generate default
        return target.EnemySource?.Name switch
        {
            not null => $"A {target.EnemySource.Name.ToLower()} stands before you.",
            _ => "An unknown threat looms in the darkness."
        };
    }

    private static string GetHpDisplay(Combatant target, KnowledgeTier tier)
    {
        return tier switch
        {
            KnowledgeTier.Known or KnowledgeTier.Mastered
                => $"{target.CurrentHp}/{target.MaxHp}",

            KnowledgeTier.Obscured
                => GetHealthNarrative(target.CurrentHp, target.MaxHp),

            _ => "???"
        };
    }

    private static string GetHealthNarrative(int current, int max)
    {
        var percent = max > 0 ? (current * 100) / max : 0;
        return percent switch
        {
            >= 75 => "Healthy",
            >= 50 => "Bloodied",
            >= 25 => "Wounded",
            _ => "Critical"
        };
    }

    private static string GetArmorDisplay(Combatant target, KnowledgeTier tier)
    {
        return tier switch
        {
            KnowledgeTier.Known or KnowledgeTier.Mastered
                => target.ArmorSoak.ToString(),

            KnowledgeTier.Obscured
                => GetArmorNarrative(target.ArmorSoak),

            _ => "???"
        };
    }

    private static string GetArmorNarrative(int soak)
    {
        return soak switch
        {
            0 => "Unarmored",
            <= 2 => "Light Armor",
            <= 4 => "Medium Armor",
            <= 6 => "Heavy Armor",
            _ => "Fortified"
        };
    }

    private static List<string> GetWeaknesses(Combatant target)
    {
        var weaknesses = new List<string>();

        // Check for common weakness indicators in tags
        if (target.Tags.Contains("Mechanical"))
            weaknesses.Add("Lightning");
        if (target.Tags.Contains("Undead"))
            weaknesses.Add("Fire");
        if (target.Tags.Contains("Beast"))
            weaknesses.Add("Poison");
        if (target.Tags.Contains("Corrupted"))
            weaknesses.Add("Psychic");

        return weaknesses;
    }

    private static List<AbilityDisplayInfo> GetAbilities(Combatant target)
    {
        return target.Abilities.Select(a => new AbilityDisplayInfo(
            Name: a.Name,
            Description: a.Description ?? "No description available.",
            CooldownDisplay: a.Cooldown > 0 ? $"{a.Cooldown} turn CD" : "No cooldown"
        )).ToList();
    }

    private static string? GetAiHint(Combatant target)
    {
        // Generate behavioral hint based on archetype
        return target.Archetype switch
        {
            EnemyArchetype.Tank => "Prioritizes protecting allies. Will Defend when HP is low.",
            EnemyArchetype.DPS => "Prioritizes lowest-HP targets. Uses abilities aggressively.",
            EnemyArchetype.Support => "Stays in back row. Prioritizes buffing or healing allies.",
            EnemyArchetype.Disabler => "Targets high-threat enemies with control abilities.",
            _ => null
        };
```

```csharp
/// <summary>
/// Handles the inspect command during combat (v0.3.22b).
/// </summary>
/// <param name="targetName">The name of the enemy to inspect.</param>
/// <returns>The enemy intel if target found, null if not found or invalid.</returns>
public async Task<EnemyIntel?> HandleInspectAsync(string targetName)
{
    _logger.LogDebug("[Combat] Handling inspect command for target: {Target}", targetName);

    // 1. Resolve target from turn order
    var target = ResolveCombatTarget(targetName);
    if (target == null)
    {
        _logger.LogWarning("[Combat] Inspect target '{Target}' not found", targetName);
        return null;
    }

    // 2. Validate target is enemy
    if (target.IsPlayer)
    {
        _logger.LogWarning("[Combat] Cannot inspect player combatant");
        return null;
    }

    // 3. Get player character for archetype bonus
    var playerCombatant = _combatState.TurnOrder.FirstOrDefault(c => c.IsPlayer);
    if (playerCombatant?.CharacterSource == null)
    {
        _logger.LogError("[Combat] Player character not found in combat state");
        return null;
    }

    // 4. Query analysis service
    var intel = await _analysisService.InspectAsync(target, playerCombatant.CharacterSource);

    _logger.LogInformation(
        "[Combat] Inspection complete. Target: {Target}, Tier: {Tier}",
        target.Name, intel.Tier);

    return intel;
}

/// <summary>
/// Resolves a combat target by name from the current turn order.
/// </summary>
private Combatant? ResolveCombatTarget(string targetName)
{
    var normalized = targetName.ToLowerInvariant().Trim();

    // Exact match first
    var exactMatch = _combatState.TurnOrder
        .FirstOrDefault(c => c.Name.ToLowerInvariant() == normalized);

    if (exactMatch != null)
        return exactMatch;

    // Partial match fallback
    var partialMatches = _combatState.TurnOrder
        .Where(c => c.Name.ToLowerInvariant().Contains(normalized))
        .ToList();

    if (partialMatches.Count == 1)
        return partialMatches[0];

    if (partialMatches.Count > 1)
    {
        _logger.LogWarning(
            "[Combat] Multiple matches for '{Target}', using first: {Match}",
            targetName, partialMatches[0].Name);
        return partialMatches[0];
    }

    return null;
}
```

## Changelog: v0.3.22b - The Inspector

**Release Date:** 2026-XX-XX

### Summary
"The Inspector" transforms the Bestiary into an active tactical tool. Players can now analyze enemies mid-combat to reveal statistics, weaknesses, and abilities based on their collected lore fragments. Knowledge-focused archetypes like Adepts (Jötun-Readers) receive a "Scholar's Insight" bonus, perceiving one tier higher than their actual collection progress.

### Features
- **Inspect Command:** Use `inspect <target>` (or `analyze`/`scan`) during combat to query enemy intel.
- **Knowledge Tiers:** Enemy information is revealed across 4 tiers:
  - **Unknown (0-24%):** Name and flavor only. Stats hidden.
  - **Obscured (25-74%):** Narrative HP/Armor, type tags visible.
  - **Known (75-99%):** Exact stats and weaknesses visible.
  - **Mastered (100%):** Full abilities and AI behavior hints.
- **Scholar's Insight:** Adept and Mystic archetypes automatically perceive one tier higher.
- **Inspect Window:** New modal popup displays tiered intel with color-coded borders.
- **Dynamic Spawn Handling:** Enemies without Codex entries default to Unknown tier gracefully.

### Technical
- Added `KnowledgeTier` enum with Unknown, Obscured, Known, Mastered values
- Added `EnemyIntel` record with masked display fields
- Implemented `AnalysisService` bridging DataCaptureService and CombatService
- Added `GetByTemplateIdAsync()` to `ICodexEntryRepository` for template lookups
- Extended `ParseResult` with `RequiresInspect` and `InspectTarget` properties
- Created `InspectWindowRenderer` using Spectre.Console panel overlays
- Extended `CombatService` with `HandleInspectAsync()` and target resolution
