1. Architecture & Data Flow
We need to bridge the gap between the CombatService logic and the user's screen.
Workflow: Status Effect Rendering
	1	State Change: CombatService updates CombatState (e.g., applies [Bleeding]).
	2	Mapping: CombatService.GetViewModel() maps ActiveStatusEffect domain objects to StatusEffectView records.
	3	Formatting: CombatScreenRenderer iterates through StatusEffectView items, converting types to Spectre.Console markup (e.g., [red]🩸 2[/]).
	4	Display: The Turn Order table renders the formatted string in the "Condition" column.
Workflow: Victory Sequence
	1	Victory Trigger: CombatService.EndCombat() returns a CombatResult.
	2	State Check: GameService detects CombatResult.Victory == true.
	3	Transition: GameService pauses the loop and calls VictoryScreenRenderer.Render(result).
	4	Input: Player presses key to claim loot.
	5	Resume: GameService transitions state to GamePhase.Exploration.

2. Logic Decision Trees
A. Status Effect Formatting Logic
Input: List of ActiveStatusEffects on a Combatant
	1	Is List Empty?
	◦	Yes: Return string "[grey]Normal[/]".
	2	Iterate List: For each effect:
	◦	Lookup Icon: Get symbol based on StatusEffectType (e.g., Bleed = 🩸).
	◦	Lookup Color: Get color (Bleed = Red, Fortified = Blue).
	◦	Format: Create string [{Color}]{Icon} {StackCount}[/].
	3	Concatenate: Join all strings with a space separator.
	4	Output: "[red]🩸 3[/] [blue]🛡️ 1[/]"
B. Victory Screen Logic
Input: CombatResult
	1	Render Header: "VICTORY" in Green/Gold.
	2	Render XP: Show XpEarned value.
	3	Render Loot:
	◦	If LootFound is empty -> Show "No items found."
	◦	Else -> Create Table.
	▪	Row color based on Item.Quality (Jury-Rigged = Grey, Myth-Forged = Gold).
	▪	Columns: Name, Type, Value.
	4	Prompt: "Press any key to continue..."

using Spectre.Console;
using RuneAndRust.Core.Models.Combat;

public class VictoryScreenRenderer
{
    public void Render(CombatResult result)
    {
        AnsiConsole.Clear();

        var panel = new Panel(
            new Rows(
                new FigletText("VICTORY").Color(Color.Gold1).Centered(),
                new Text($"XP Earned: {result.XpEarned}", new Style(Color.Cyan1)).Centered(),
                new Rule("Loot Recovered"),
                BuildLootTable(result.LootFound)
            )
        )
        .Border(BoxBorder.Double)
        .BorderColor(Color.Green);

        AnsiConsole.Write(panel);
        AnsiConsole.WriteLine();
        AnsiConsole.MarkupLine("[grey]Press any key to continue...[/]");
        Console.ReadKey(true);
    }

    private Table BuildLootTable(List<Item> loot)
    {
        var table = new Table().Border(TableBorder.None);
        table.AddColumn("Item");
        table.AddColumn("Type");

        if (!loot.Any())
        {
            table.AddRow("[grey]No items found.[/]", "");
            return table;
        }

        foreach (var item in loot)
        {
            // Use color map from SPEC-ECONOMY-LOOT
            string color = item.Quality switch
            {
                QualityTier.MythForged => "gold1",
                QualityTier.Optimized => "purple",
                QualityTier.ClanForged => "green",
                _ => "white"
            };

            table.AddRow($"[{color}]{item.Name}[/]", item.ItemType.ToString());
        }
        return table;
    }
}
