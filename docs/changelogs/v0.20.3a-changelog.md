# v0.20.3a Changelog — Jötun-Reader Tier 1 Foundation

## Summary

Implements the Jötun-Reader specialization foundation: domain-layer enums and value objects,
Tier 1 abilities (Deep Scan, Pattern Recognition, Ancient Tongues), configuration, and tests.

---

## Added

### Domain Layer — Enums

- **`JotunReaderAbilityId`** — 9 strongly-typed ability identifiers across 4 tiers
- **`LoreDiscoveryType`** — 6 discovery categories with base insight (named to avoid collision with existing `DiscoveryType`)
- **`ExaminationSuccessLevel`** — 5-tier success levels (Failure → Master) mapping to information layers
- **`ScriptType`** — 4 ancient script types unlocked by Ancient Tongues

### Domain Layer — Value Objects

- **`LoreInsightResource`** — Immutable resource (10/10 default) with discovery generation, spending, and rest restoration
- **`DiscoveryRecord`** — Immutable discovery event record with factory method and lore tracking
- **`DeepScanResult`** — Immutable Deep Scan result with success/failure factories (named to avoid collision with existing `ExaminationResult`)

### Application Layer

- **`IJotunReaderAbilityService`** — Interface for all Tier 1 operations and PP prerequisite helpers
- **`JotunReaderTier1AbilityService`** — Full implementation with structured logging, seeded Random for testability

### Configuration

- Updated `specializations.json`: Added `lore-insight` special resource and Tier 1 ability definitions (Deep Scan, Pattern Recognition, Ancient Tongues)

### Unit Tests

- **`LoreInsightResourceTests`** — 13 tests: creation, spending, discovery generation, restoration, display
- **`JotunReaderTier1AbilityServiceTests`** — 18 tests: Deep Scan, Pattern Recognition, Ancient Tongues, PP prerequisites

## Naming Decisions

Two naming collisions with existing types were resolved:

- `DiscoveryType` → **`LoreDiscoveryType`** (avoids perception-layer `DiscoveryType` in `ExaminationEnums.cs`)
- `ExaminationResult` → **`DeepScanResult`** (avoids perception-layer `ExaminationResult` value object)
