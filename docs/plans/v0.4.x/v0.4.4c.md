## Overview

**v0.4.4c** introduces **School Mastery**, the primary long-term progression vector for the Mystic archetype. While character levels increase raw stats, School Mastery represents the refinement of magical technique. It allows Mystics to specialize in specific schools (Destruction, Restoration, Alteration, Divination), unlocking efficiency bonuses, access to higher-tier spells, and unique passive abilities.

### Relationship to Ecosystem

| Component | Interaction |
|-----------|-------------|
| **v0.4.3c (MagicService)** | Queries Mastery level to calculate final AP/Flux costs. |
| **v0.4.4a (Resonance)** | Mastery reduces the severity of Paradox events (v0.4.4d). |
| **v0.4.4b (Galdr)** | Completing Galdr chants awards significant Mastery XP. |
| **v0.4.4e (Grimoire)** | Mastery Tier dictates which spells can be learned/inscribed. |

## Scope & Objectives

The primary goal is to implement a persistent progression system that tracks experience per magic school and awards gameplay benefits.

### In-Scope
- **Per-School XP Tracking**: Independent progression for Destruction, Restoration, Alteration, Divination.
- **Tier System**: 5 defined tiers (Novice to Archon) with specific thresholds.
- **Bonus Calculation**: Logic for cost reduction (AP/Flux) based on tier.
- **Passive System**: Implementation of "Adept" tier passive bonuses.
- **Event Integration**: Hooking into spell casts and kills to award XP.
- **UI/TUI**: A "Mastery" view to inspect progress and bonuses.

### Out-of-Scope
- **Spell Content**: The actual Tier 2/3 spells are defined in v0.4.4e (The Grimoire).
- **Ultimate Abilities**: "Archon" tier ultimates are placeholders for v0.4.6.

## Architecture & Design

### Dependency Graph

```
┌─────────────────────┐       ┌──────────────────────┐
│    MagicService     │──────►│    MasteryService    │
│ (Executes Spells)   │       │   (Tracks Progress)  │
└──────────┬──────────┘       └──────────┬───────────┘
           │                             │
           ▼                             ▼
┌─────────────────────┐       ┌──────────────────────┐
│   SpellCastEvent    │──────►│  MasteryEventHandler │
└─────────────────────┘       └──────────────────────┘
```

### Data Flow: XP Acquisition

1. **Trigger**: Player casts "Firebolt" (Destruction).
2. **Event**: `MagicService` publishes `SpellCastEvent(SpellId, CasterId, TargetId)`.
3. **Process**: `MasteryService` (via handler) receives event.
4. **Validation**: Verify caster is Mystic and spell has a School.
5. **Calculation**: Base XP (10) + Bonuses (Talents/Resonance).
6. **Update**: `SchoolMastery` model updated.
7. **Check**: If `CurrentXP >= NextTierXP` -> `LevelUp()`.
8. **Feedback**: `MasteryTierReachedEvent` published -> UI Notification.

## Detailed Logic & Mechanics

### 1. Mastery Tiers & Thresholds

| Tier | Code | XP Required | AP Cost | Flux Cost | Unlocks |
|------|------|-------------|---------|-----------|---------|
| **Novice** | `Tier.Novice` | 0 | 100% | 100% | Tier 1 Spells |
| **Apprentice** | `Tier.Apprentice` | 100 | 95% | 95% | Tier 2 Spells |
| **Adept** | `Tier.Adept` | 300 | 90% | 90% | **School Passive** |
| **Master** | `Tier.Master` | 600 | 85% | 85% | Tier 3 Spells |
| **Archon** | `Tier.Archon` | 1000 | 80% | 80% | Ultimate Ability |

### 2. XP Sources

| Source | XP Value | Notes |
|--------|----------|-------|
| **Standard Cast** | +10 | Successful cast of any instant spell. |
| **Galdr Complete** | +15 | Successful completion of a Chanted spell. |
| **Killing Blow** | +25 | Target dies from spell damage (Bonus). |
| **Learn Spell** | +50 | Inscribing a new spell into Grimoire. |

**Logic Example (XP Calculation):**
```csharp
int xp = 10;
if (isGaldr) xp += 5; // Total 15
if (targetDied) xp += 25;
// Apply global multipliers (e.g. Artifacts)
return xp;
```

### 3. School Passives (Adept Tier)

These are hard-coded effects active when the Mystic reaches "Adept" in the respective school.

- **Destruction: "Entropic Resonance"**
    - *Effect:* Critical hits with Destruction spells roll an additional damage die.
    - *Implementation:* `MagicService` checks `MasteryService.HasPassive(caster, School.Destruction)` during damage calc.

- **Restoration: "Life Attunement"**
    - *Effect:* Healing spells restore +10% more HP.
    - *Implementation:* Modifier applied in `HealEffect` calculation.

- **Alteration: "Reality Anchor"**
    - *Effect:* Buffs/Debuffs last +1 turn.
    - *Implementation:* `StatusEffectService` checks caster mastery when applying duration.

- **Divination: "Third Eye"**
    - *Effect:* `Analyze` reveals exact HP and Intent immediately.
    - *Implementation:* `CombatInspector` checks mastery level.

### Decision Trees

#### A. AddMasteryXP
1. **Input**: Character, School, XP Amount.
2. **Get Mastery**: Retrieve `SchoolMastery` object for that school.
3. **Add XP**: `CurrentXP += Amount`.
4. **Check Tier**: Iterate `MasteryThresholds`.
    - If `CurrentXP >= Threshold` AND `CurrentTier < ThresholdTier`:
        - Set `CurrentTier = ThresholdTier`.
        - Trigger `MasteryTierReachedEvent`.
        - Return `LevelUpResult`.
5. **Save**: Persist changes.

#### B. GetCostModifiers
1. **Input**: Character, Spell.
2. **Get Tier**: Retrieve `SchoolMastery.Tier` for `Spell.School`.
3. **Lookup**: Map Tier to Multiplier (e.g., Master -> 0.85).
4. **Return**: `(ApModifier, FluxModifier)`.
