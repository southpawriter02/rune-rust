### Features
- [x] **Monster Types** - Monsters loaded from definitions (default: Goblins, Skeletons, Orcs, etc.)
- [x] **Monster Tiers** - Configurable tier system (default: Common, Named, Elite, Boss)
- [x] **Loot Tables** - Data-driven loot with weighted drops
- [x] **Gold/Currency** - Configurable currency system (name, icon, drop rates)
- [x] **Loot Command** - Pick up all loot after combat
- [x] **Monster Descriptions** - Examine monsters for info, uncover unique details from examination checks
- [x] **Monster Special Attacks** - Monsters can have traits from a predefined/shared pool of behaviors (run away, flying, swimming, heal self), damage resistances/vulnerabilities,abilities, etc.
- [x] **Damage Types** - Configurable damage types with resistances/vulnerabilities

### Customization Features
- [x] **Monster Definitions** - JSON files defining monster stats, abilities, loot, behavior
- [x] **Damage Type Definitions** - Configurable damage types (Fire, Ice, Poison, Holy, etc.)
- [x] **Resistance System** - Percentage-based resistances/vulnerabilities per damage type
- [x] **Tier Definitions** - Configurable tier names, stat multipliers, loot bonuses
- [x] **Currency Configuration** - Custom currency names ("Gold", "Coins", "Runes", "Souls")

### Data Model Updates
- [x] Create `MonsterDefinition` loaded from configuration (not enum)
- [x] Create `DamageTypeDefinition` entity
- [x] Create `TierDefinition` for monster difficulty scaling
- [x] Create `LootTable` with weighted item/currency drops
- [x] Create `CurrencyDefinition` for customizable money systems
- [x] Add `Resistances` dictionary to creatures
- [x] Add `Currency` dictionary to Player (supports multiple currency types)
