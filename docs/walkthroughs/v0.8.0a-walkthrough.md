# v0.8.0a Walkthrough - Character Creation Wizard

## Overview

This version implements the Character Creation Wizard, a 6-step workflow for creating new characters in the GUI application.

## Architecture

```
┌─────────────────────────────────────────────────┐
│           CharacterCreationWizard               │
│  ┌───────────────────────────────────────────┐  │
│  │              WizardViewModel               │  │
│  │  ┌─────────────────────────────────────┐  │  │
│  │  │    CharacterCreationData            │  │  │
│  │  │  - Name, RaceId, ClassId            │  │  │
│  │  │  - Stats Dictionary                 │  │  │
│  │  │  - PortraitId                       │  │  │
│  │  └─────────────────────────────────────┘  │  │
│  │                                           │  │
│  │  Steps: [1]→[2]→[3]→[4]→[5]→[6]          │  │
│  │   Name  Race Class Stats App  Review     │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

## Wizard Steps

| Step | ViewModel | Purpose |
|------|-----------|---------|
| 1 | `NameEntryStepViewModel` | Enter character name (2-20 chars) |
| 2 | `RaceSelectionStepViewModel` | Select race (Human, Elf, Dwarf) |
| 3 | `ClassSelectionStepViewModel` | Select class (Warrior, Mage, Rogue) |
| 4 | `StatAllocationStepViewModel` | Allocate 27 points (8-15 per stat) |
| 5 | `AppearanceStepViewModel` | Select portrait |
| 6 | `ReviewStepViewModel` | Review and confirm |

## Validation

Each step validates before navigation:
- **Name**: 2-20 characters required
- **Race**: Selection required
- **Class**: Selection required
- **Stats**: Cannot overspend points
- **Appearance**: Optional (defaults to first portrait)
- **Review**: Always valid
