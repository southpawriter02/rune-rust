Here is the comprehensive implementation plan for v0.1.3: Data Captures (The Codex).
Consistent with our "Walking Skeleton" approach and the complexity of the Data Capture system (fragment assembly, redaction logic, and distinct UI), this version is split into three sub-milestones: Foundation (a), Logic (b), and Interface (c).

v0.1.3: The Codex (Data Capture System)
Goal: Implement the "Scavenger's Journal"—a non-inventory progression system where players collect fragmented lore, assemble bestiary entries, and unlock mechanics explanations.

A. Terminal Layer (UI Renderer) File: RuneAndRust.Terminal/Services/JournalRenderer.cs
public void RenderEntry(CodexEntry entry, int completionPct)
{
    var panel = new Panel(
        Align.Center(
            new Markup($"[bold gold1]{entry.Title}[/]\n") +
            new Markup($"[italic grey]Category: {entry.Category}[/]\n\n") +
            RenderRedactedText(entry.FullText, completionPct)
        )
    );
    panel.Header = new PanelHeader($"[Completion: {completionPct}%]");
    AnsiConsole.Write(panel);
}

private Markup RenderRedactedText(string fullText, int pct)
{
    if (pct >= 100) return new Markup(fullText);

    // Simple redaction logic: if pct is 50%, obscure every other word
    // Real logic should use the 'Unlocks' dictionary from v0.1.3a
    return new Markup("[grey]The [red]██████[/] caused the collapse...[/]");
}
B. Terminal Layer (Command) File: RuneAndRust.Engine/Commands/JournalCommand.cs
// Invoked via 'journal', 'codex', 'j'
public async Task ExecuteAsync(string[] args)
{
    var entries = await _captureService.GetPlayerCodex(_gameState.CurrentCharacter.Id);

    if (args.Length > 0)
    {
        // Show specific entry details
        var target = entries.FirstOrDefault(e => e.Title.Contains(args));
        _renderer.RenderEntry(target, ...);
    }
    else
    {
        // Show Index
        _renderer.RenderIndex(entries);
    }
}

3. Architecture Decision Tree (v0.1.3)
	•	Q: Why store Captures separate from Inventory?
	◦	A: To follow the "No Inventory Burden" design pillar [SPEC-SYSTEM-DATA-CAPTURES]. Lore should never compete with survival gear for slot space.
	•	Q: Should we implement the "glitch" visual effects for text now?
	◦	A: Yes, basic versions. Use Spectre.Console markup to create static/obfuscation effects ([blink], [dim]) for incomplete entries. Full dynamic shaders come in the GUI update.
	•	Q: How do we seed the initial lore?
	◦	A: Use DataSeeder.cs (Persistence). We need at least 3 Codex Entries (e.g., "The Ginnungagap Glitch") and 3 Bestiary Entries (e.g., "Rusted Servitor") to test the assembly logic.
