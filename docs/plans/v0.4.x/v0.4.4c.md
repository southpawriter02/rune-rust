## Overview

**v0.4.4c** introduces **School Mastery**, the primary long-term progression vector for the Mystic archetype. While character levels increase raw stats (Might, Wits, Presence), School Mastery represents the refinement of magical technique—the "muscle memory" of the soul. It allows Mystics to specialize in specific schools (Destruction, Restoration, Alteration, Divination), unlocking efficiency bonuses, access to higher-tier spells, and unique passive abilities that define their playstyle.

### Relationship to Ecosystem

This module sits at the intersection of "The Saga" (Progression) and "The Weaver" (Magic), acting as the bridge between casting spells and getting better at them.

| Component | Interaction | Direction |
|-----------|-------------|-----------|
| **v0.4.3c (MagicService)** | Queries Mastery level to calculate final AP/Flux costs. | `MagicService` -> `MasteryService` |
| **v0.4.4a (Resonance)** | Mastery reduces the severity of Paradox events (v0.4.4d). | `ParadoxService` -> `MasteryService` |
| **v0.4.4b (Galdr)** | Completing Galdr chants awards significant Mastery XP. | `GaldrService` -> `Event` -> `MasteryService` |
| **v0.4.4e (Grimoire)** | Mastery Tier dictates which spells can be learned/inscribed. | `GrimoireService` -> `MasteryService` |
| **v0.4.1 (Specialization)** | Mastery is distinct from Specialization Points (SP) but may synergize. | Parallel Systems |

## Scope & Objectives

The primary goal is to implement a persistent progression system that tracks experience per magic school and awards gameplay benefits, ensuring players feel rewarded for sticking to a specific style of magic.

### In-Scope
- **Per-School XP Tracking**: Independent progression for Destruction, Restoration, Alteration, Divination.
- **Tier System**: 5 defined tiers (Novice to Archon) with specific thresholds.
- **Bonus Calculation**: Logic for cost reduction (AP/Flux) based on tier.
- **Passive System**: Implementation of "Adept" tier passive bonuses with complex triggers.
- **Event Integration**: Hooking into spell casts and kills via `MediatR` or internal event bus.
- **UI/TUI**: A dedicated "Mastery" view (`mastery` command) to inspect progress and bonuses.
- **Persistence**: Saving/Loading mastery state via `RuneAndRust.Persistence`.

### Out-of-Scope
- **Spell Content**: The actual Tier 2/3 spells are defined in v0.4.4e (The Grimoire).
- **Ultimate Abilities**: "Archon" tier ultimates are placeholders for v0.4.6; we only implement the "unlock flag" here.
- **Graphical FX**: No particle effects for leveling up (TUI only).

### Data Flow: XP Acquisition

1. **Trigger**: Player casts "Firebolt" (Destruction).
2. **Event**: `MagicService` publishes `SpellCastEvent(SpellId, CasterId, TargetId, School, IsGaldr)`.
3. **Process**: `MasteryEventHandler` catches the event.
4. **Validation**:
   - Is `CasterId` the player? (Enemy mastery is out of scope for now).
   - Does the spell have a valid `MagicSchool`?
5. **Calculation**:
   - Base XP (10).
   - Add Galdr bonus (+5) if `IsGaldr` is true.
   - Apply any global XP multipliers (e.g., "Scholar's Ring").
6. **Update**: `MasteryService.AddXp(CasterId, School, Amount)` is called.
7. **Check**: If `CurrentXP >= NextTierXP`:
   - Increment Tier.
   - Fire `MasteryTierReachedEvent`.
8. **Feedback**: UI Listeners display a notification: "Destruction Mastery Increased to Apprentice!"

## Detailed Logic & Mechanics

### 1. Mastery Tiers & Thresholds

The progression curve is exponential to prevent early-game grinding from unlocking endgame power too quickly.

| Tier | Enum Code | XP Required (Cumulative) | AP Cost Mod | Flux Cost Mod | Unlocks |
|------|-----------|--------------------------|-------------|---------------|---------|
| **Novice** | `Tier.Novice` | 0 | 1.00 (100%) | 1.00 (100%) | Tier 1 Spells |
| **Apprentice** | `Tier.Apprentice` | 100 | 0.95 (95%) | 0.95 (95%) | Tier 2 Spells |
| **Adept** | `Tier.Adept` | 300 | 0.90 (90%) | 0.90 (90%) | **School Passive** |
| **Master** | `Tier.Master` | 600 | 0.85 (85%) | 0.85 (85%) | Tier 3 Spells |
| **Archon** | `Tier.Archon` | 1000 | 0.80 (80%) | 0.80 (80%) | Ultimate Ability |

**Note**: Modifiers are multiplicative with other bonuses, not additive, to prevent 0-cost spells.

### 2. XP Acquisition Logic

The XP calculation is centralized in `MasteryService.CalculateXpGain()`.

**Algorithm:**
```csharp
public int CalculateXpGain(SpellCastEvent evt, Character caster)
{
    int baseXp = 10;

    // Galdr Bonus: Chanted spells are harder, thus more rewarding
    if (evt.IsGaldr)
    {
        baseXp += 15; // Total 25
    }

    // Kill Bonus: Handled separately via EnemyKilledEvent,
    // but effectively +25 XP

    // First-Time Cast Bonus: Not implemented yet (needs tracking history)

    // Intelligence Scaling (Wits)
    // Smart characters learn faster. +5% XP per point of WITS above 5.
    double witsMod = 1.0 + (Math.Max(0, caster.Attributes.Wits - 5) * 0.05);

    return (int)(baseXp * witsMod);
}
```

### 3. Cost Modification Algorithms

When `MagicService` asks "How much AP does this spell cost?", it delegates the reduction logic here.

**Algorithm:**
```csharp
public (int Ap, int Flux) CalculateCosts(Character caster, Spell spell)
{
    var mastery = GetMastery(caster, spell.School);
    var tier = mastery?.Tier ?? MasteryTier.Novice;

    double modifier = tier switch
    {
        MasteryTier.Novice => 1.0,
        MasteryTier.Apprentice => 0.95,
        MasteryTier.Adept => 0.90,
        MasteryTier.Master => 0.85,
        MasteryTier.Archon => 0.80,
        _ => 1.0
    };

    // Rounding Strategy: Standard Rounding, Minimum 1
    int ap = Math.Max(1, (int)Math.Round(spell.BaseApCost * modifier));
    int flux = Math.Max(1, (int)Math.Round(spell.BaseFluxCost * modifier));

    return (ap, flux);
}
```

### 4. School Passives Deep Dive

These are the "Adept" tier rewards. They are not active abilities but conditional modifiers.

#### A. Destruction: "Entropic Resonance"
* **Concept**: Fire breeds fire. Destruction spells become more volatile.
* **Mechanic**: When rolling for damage, if the result is the maximum value on the die (e.g., rolling a 6 on a d6), roll an additional die of the same type and add it. (Exploding Dice).
* **Implementation**: `DamageCalculator` service checks for this flag.

#### B. Restoration: "Life Attunement"
* **Concept**: The healer anticipates the wound.
* **Mechanic**: Over-healing (healing target above MaxHP) grants temporary HP equal to 50% of the excess amount, lasting 3 turns.
* **Implementation**: `HealingService` logic.

#### C. Alteration: "Reality Anchor"
* **Concept**: Changes to reality resist reverting.
* **Mechanic**: Buffs applied by the caster have their duration extended by 1 turn. Debuffs applied by the caster have their duration extended by 1 turn.
* **Implementation**: `StatusEffectService.ApplyEffect()` checks mastery.

#### D. Divination: "Third Eye"
* **Concept**: Seeing the pattern before it manifests.
* **Mechanic**: Passive Initiative bonus equal to Divination Tier (+0 to +4). Additionally, inspecting an enemy (`look enemy`) requires no Action Points.
* **Implementation**: `TurnOrderService` for initiative, `CommandParser` for AP cost waiver.

## Integration & Workflows

### Scenario 1: The Level Up
1. Player casts `Ignite` (Destruction) on a `Rust-Husk`.
2. `MagicService` fires `SpellCastEvent`.
3. `MasteryService` calculates XP: 10 (Base). Current XP: 95 -> 105.
4. Threshold crossed (100). Tier increases Novice -> Apprentice.
5. `MasteryTierReachedEvent` fired.
6. `NotificationService` pushes toast: "Destruction skill improved: Apprentice".
7. `MessageLog` adds: "The fire obeys you more willingly now. (Destruction is now Apprentice)."

### Scenario 2: The Passive Trigger
1. Player (Restoration Adept) casts `Heal` (Restore 10 HP) on Self (Current: 18/20 HP).
2. Healing amount: 10.
3. `HealingService` checks `MasteryService.HasPassive(Restoration)`. Result: True.
4. Logic detects Overheal: (18+10) - 20 = 8 Excess.
5. Passive Effect: 50% of 8 = 4 TempHP.
6. Player ends with 20/20 HP and 4 TempHP.

## Migration Strategy

### Handling Existing Saves
Since `v0.4.4c` introduces a new field `List<SchoolMastery> Masteries` to the `Character` entity, existing save files (JSON/SQLite) from `v0.4.3` will lack this data.

**Migration Logic (`DataSeeder` or `SaveManager`):**
1. On load, check if `character.Masteries` is null.
2. If null, initialize an empty list.
3. **Retroactive Credit**:
   - There is no history of past spells cast, so we cannot retroactively award XP accurately.
   - *Decision*: Start all existing characters at **Novice (0 XP)**.
   - *Alternative*: If the character has "high level" spells known (Tier 2/3) from previous debugging/testing, automatically bump their mastery to the minimum required tier for those spells.

**Code Snippet (Migration):**
```csharp
public void EnsureMasteryIntegrity(Character character)
{
    if (character.Masteries == null)
    {
        character.Masteries = new List<SchoolMastery>();
    }

    foreach (var school in Enum.GetValues<MagicSchool>())
    {
        if (!character.Masteries.Any(m => m.School == school))
        {
            character.Masteries.Add(new SchoolMastery
            {
                School = school,
                CurrentXP = 0,
                CurrentTier = MasteryTier.Novice
            });
        }
    }

    // Auto-fix for known spells
    foreach (var spellId in character.KnownSpells)
    {
        var spell = _spellRegistry.GetSpell(spellId);
        var requiredTier = spell.Tier; // e.g., 2 (Apprentice)
        var mastery = character.Masteries.First(m => m.School == spell.School);

        if ((int)mastery.CurrentTier < requiredTier)
        {
            mastery.CurrentTier = (MasteryTier)requiredTier;
            mastery.CurrentXP = MasteryConstants.GetXpForTier(mastery.CurrentTier);
        }
    }
}
```
