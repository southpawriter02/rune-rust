## 1. Overview

### 1.1 Purpose

This document provides a comprehensive design specification for v0.15.3d, the Intimidation System phase. This subsystem implements the fear-based coercion mechanics of the Rhetoric skill, allowing players to threaten NPCs into compliance through either physical presence (MIGHT) or sheer force of will (WILL). Unlike other social interactions, intimidation **always costs faction reputation** regardless of outcome. The key components are:

1. **IntimidationTarget Enum**: Categorizes NPC resistance from Coward (DC 8) to FactionLeader (DC 24), determining the base difficulty of the intimidation attempt based on the target's resolve and position.

2. **IntimidationContext Value Object**: Extends `SocialContext` with intimidation-specific factors including attribute choice (MIGHT or WILL), target type, relative strength comparison, reputation effects, artifact display, ally presence, and target backup.

3. **IntimidationResult Value Object**: Captures intimidation-specific outcomes including compliance level, reputation cost, combat initiation on fumble, and stress costs.

4. **RelativeStrength Enum**: Evaluates the relative power dynamic between player and NPC (Weaker, Equal, Stronger), applying dice modifiers to either side.

5. **IntimidationApproach Enum**: Distinguishes between Physical (MIGHT-based, looming, weapons) and Mental (WILL-based, cold stare, implied doom) approaches.

6. **CostOfFear Value Object**: Tracks the mandatory reputation cost of intimidation (Success: -5, Critical: -3, Failure: -10), reflecting that using fear as a tool always damages relationships.

7. **IntimidationService**: Orchestrates intimidation attempts using the chosen attribute (MIGHT or WILL) + Rhetoric, applying all context modifiers, and managing the [Challenge Accepted] fumble.

8. **[Challenge Accepted] Fumble Consequence**: A catastrophic fumble where the NPC refuses to be cowed, immediately initiating combat with the [Furious] buff (+2d10 damage), refusing surrender, and inflicting +5 Psychic Stress.

This subsystem creates high-risk social interactions where players must weigh immediate gains against long-term reputation damage and the possibility of violent escalation.

### 1.2 Current vs. Target Implementation

| Aspect | Current Implementation | Target Implementation |
|--------|------------------------|----------------------|
| **Target Classification** | Generic DC | `IntimidationTarget` enum with 5 tiers (DC 8-24) |
| **Attribute Choice** | Fixed attribute | Player chooses MIGHT or WILL |
| **Physical Approach** | Not implemented | MIGHT-based: looming, weapon display, flexing |
| **Mental Approach** | Not implemented | WILL-based: cold stare, implied doom |
| **Relative Strength** | Not tracked | Weaker/Equal/Stronger with dice modifiers |
| **Reputation Effects** | Not integrated | [Honored]/[Feared] provides +1d10 |
| **Artifact Display** | Not tracked | Wielding visible [Artifact] +1d10 |
| **Ally Presence** | Not tracked | Intimidating ally present +1d10 |
| **Target Backup** | Not tracked | NPC has backup nearby, NPC +1d10 |
| **Reputation Cost** | Not implemented | Always costs faction reputation |
| **Fumble Consequence** | Generic failure | [Challenge Accepted] - immediate combat |
| **[Furious] Buff** | Not implemented | NPC gains +2d10 damage on fumble |

### 1.3 Scope

**In Scope:**
- Define `IntimidationTarget` enum with 5 resistance tiers
- Define `IntimidationContext` value object with intimidation-specific factors
- Define `IntimidationResult` value object for intimidation outcomes
- Define `RelativeStrength` enum for power dynamic evaluation
- Define `IntimidationApproach` enum for MIGHT vs WILL choice
- Define `CostOfFear` value object for mandatory reputation cost
- Create `IIntimidationService` interface for intimidation operations
- Implement `IntimidationService` with attribute choice mechanics
- Implement [Challenge Accepted] fumble consequence with all sub-effects
- Implement [Furious] status effect with +2d10 damage
- Configuration file `intimidation-targets.json`
- Unit tests for intimidation mechanics (~3 tests)

**Out of Scope:**
- Negotiation system - v0.15.3e
- Interrogation system - v0.15.3f
- Cultural protocols - v0.15.3g
- Extended influence - v0.15.3h
- Specialization integration - v0.15.3i

namespace RuneRust.Domain.Enums;

/// <summary>
/// Categorizes NPC resistance to intimidation based on their resolve,
/// experience, and position. Higher tiers represent individuals who

/// or positions that demand fearlessness.
/// </summary>
public enum IntimidationTarget
{
    /// <summary>
    /// Timid individuals who frighten easily. Nervous merchants,
    /// sheltered nobles, inexperienced youth. DC 8.
    /// </summary>
    Coward = 0,

    /// <summary>
    /// Average individuals with normal resolve. Standard guards,
    /// craftsmen, innkeepers, commoners. DC 12.
    /// </summary>
    Common = 1,

    /// <summary>
    /// Battle-hardened individuals who have faced death and survived.
    /// Experienced soldiers, mercenaries, veteran guards. DC 16.
    /// </summary>
    Veteran = 2,

    /// <summary>
    /// Exceptional individuals with ironclad willpower. Champions,
    /// elite commanders, crime lords, fanatical zealots. DC 20.
    /// </summary>
    Elite = 3,

    /// <summary>
    /// Leaders whose position demands fearlessness. Jarls, guild masters,
    /// clan chieftains, high priests. Showing fear would undermine their
    /// authority. DC 24.
    /// </summary>
    FactionLeader = 4
}

// Location: src/Domain/ValueObjects/IntimidationContext.cs

namespace RuneRust.Domain.ValueObjects;

using RuneRust.Domain.Enums;

/// <summary>
/// Encapsulates all context factors relevant to an intimidation attempt,
/// extending the base SocialContext with intimidation-specific modifiers
/// including attribute choice, relative strength, reputation effects,
/// equipment display, and ally/backup presence.
/// </summary>
public sealed record IntimidationContext
{
    /// <summary>
    /// The base social context containing common interaction data.
    /// </summary>
    public required SocialContext BaseContext { get; init; }

    /// <summary>
    /// The resistance tier of the target NPC, determining base DC.
    /// </summary>
    public required IntimidationTarget TargetType { get; init; }

    /// <summary>
    /// The approach chosen by the player (Physical or Mental),
    /// determining which attribute (MIGHT or WILL) is used.
    /// </summary>
    public required IntimidationApproach Approach { get; init; }

    /// <summary>
    /// The power dynamic between player and NPC based on level comparison.
    /// Affects dice modifiers for either side.
    /// </summary>
    public required RelativeStrength RelativeStrength { get; init; }

    /// <summary>
    /// Whether the player has [Honored] or [Feared] status with the
    /// target's faction. Provides +1d10 if true.
    /// </summary>
    public required bool HasReputation { get; init; }

    /// <summary>
    /// Whether the player is visibly wielding an [Artifact] item.
    /// Provides +1d10 if true.
    /// </summary>
    public required bool WieldingArtifact { get; init; }

    /// <summary>
    /// Whether the player has an intimidating ally present
    /// (Kriger companion, summoned creature, etc.). Provides +1d10 if true.
    /// </summary>
    public required bool IntimidatingAlly { get; init; }

    /// <summary>
    /// Whether the NPC has backup (visible allies nearby).
    /// Provides NPC +1d10 if true.
    /// </summary>
    public required bool TargetHasBackup { get; init; }

    /// <summary>
    /// The faction ID affected by this intimidation attempt.
    /// Used for reputation cost calculation.
    /// </summary>
    public required string TargetFactionId { get; init; }

    /// <summary>
    /// Calculates the effective DC after applying all modifiers.
    /// Note: Intimidation has no DC modifiers, only dice modifiers.
    /// </summary>
    /// <returns>The effective difficulty class.</returns>
    public int CalculateEffectiveDc()
    {
        // Intimidation uses base DC from target type only
        // All other factors affect dice pools, not DC
        return TargetType.GetBaseDc();
    }

    /// <summary>
    /// Calculates the bonus dice added to the player's pool.
    /// </summary>
    /// <returns>Number of bonus dice for player.</returns>
    public int CalculatePlayerBonusDice()
    {
        int bonus = 0;

        // Weaker target: +1d10
        if (RelativeStrength == RelativeStrength.PlayerStronger)
        {
            bonus += 1;
        }

        // [Honored] or [Feared] reputation: +1d10
        if (HasReputation)
        {
            bonus += 1;
        }

        // Wielding visible [Artifact]: +1d10
        if (WieldingArtifact)
        {
            bonus += 1;
        }

        // Intimidating ally present: +1d10
        if (IntimidatingAlly)
        {
            bonus += 1;
        }

        return bonus;
    }

    /// <summary>
    /// Calculates the bonus dice added to the NPC's resistance.
    /// </summary>
    /// <returns>Number of bonus dice for NPC.</returns>
    public int CalculateNpcBonusDice()
    {
        int bonus = 0;

        // Stronger target: NPC +1d10
        if (RelativeStrength == RelativeStrength.PlayerWeaker)
        {
            bonus += 1;
        }

        // NPC has backup nearby: NPC +1d10
        if (TargetHasBackup)
        {
            bonus += 1;
        }

        return bonus;
    }

    /// <summary>
    /// Gets the attribute type used for this intimidation attempt.
    /// </summary>
    /// <returns>MIGHT for Physical approach, WILL for Mental approach.</returns>
    public AttributeType GetAttributeType()
    {
        return Approach switch
        {
            IntimidationApproach.Physical => AttributeType.Might,
            IntimidationApproach.Mental => AttributeType.Will,
            _ => throw new ArgumentOutOfRangeException(nameof(Approach), Approach, "Unknown intimidation approach")
        };
    }
}

namespace RuneRust.Application.Services;

using Microsoft.Extensions.Logging;
using RuneRust.Application.Interfaces;
using RuneRust.Application.Builders;
using RuneRust.Domain.Enums;
using RuneRust.Domain.ValueObjects;
using RuneRust.Domain.Entities;
using RuneRust.Domain.Extensions;

/// <summary>
/// Service for performing intimidation checks against NPCs.
/// Orchestrates the complete intimidation flow including target classification,
/// context building, dice rolling, reputation cost application, and fumble handling.
/// </summary>
public sealed class IntimidationService : IIntimidationService
{
    private readonly IDiceRollerService _diceRollerService;
    private readonly IFumbleConsequenceService _fumbleConsequenceService;
    private readonly IStatusEffectService _statusEffectService;
    private readonly ISkillStressService _skillStressService;
    private readonly IFactionService _factionService;
    private readonly ICombatService _combatService;
    private readonly INpcService _npcService;
    private readonly IPlayerService _playerService;
    private readonly IPartyService _partyService;
    private readonly IEquipmentService _equipmentService;
    private readonly ISocialContextBuilder _socialContextBuilder;
    private readonly ILogger<IntimidationService> _logger;

    public IntimidationService(
        IDiceRollerService diceRollerService,
        IFumbleConsequenceService fumbleConsequenceService,
        IStatusEffectService statusEffectService,
        ISkillStressService skillStressService,
        IFactionService factionService,
        ICombatService combatService,
        INpcService npcService,
        IPlayerService playerService,
        IPartyService partyService,
        IEquipmentService equipmentService,
        ISocialContextBuilder socialContextBuilder,
        ILogger<IntimidationService> logger)
    {
        _diceRollerService = diceRollerService;
        _fumbleConsequenceService = fumbleConsequenceService;
        _statusEffectService = statusEffectService;
        _skillStressService = skillStressService;
        _factionService = factionService;
        _combatService = combatService;
        _npcService = npcService;
        _playerService = playerService;
        _partyService = partyService;
        _equipmentService = equipmentService;
        _socialContextBuilder = socialContextBuilder;
        _logger = logger;
    }

    /// <inheritdoc />
    public async Task<IntimidationResult> IntimidateAsync(IntimidationContext context)
    {
        _logger.LogInformation(
            "Attempting intimidation against {TargetId} using {Approach} approach. Target type: {TargetType}, DC: {DC}",
            context.BaseContext.TargetId,
            context.Approach.GetDisplayName(),
            context.TargetType.GetDisplayName(),
            context.CalculateEffectiveDc());

        // Get player's attribute value based on chosen approach
        var attributeType = context.GetAttributeType();
        int attributeValue = await _playerService.GetAttributeValueAsync(attributeType);
        int rhetoricSkill = await _playerService.GetSkillValueAsync(SkillType.Rhetoric);

        // Calculate dice pools
        int playerBonusDice = context.CalculatePlayerBonusDice();
        int npcBonusDice = context.CalculateNpcBonusDice();

        int totalPlayerDice = attributeValue + rhetoricSkill + playerBonusDice;
        int effectiveDc = context.CalculateEffectiveDc();

        _logger.LogDebug(
            "Dice pool: {Attribute} ({AttrValue}) + Rhetoric ({RhetValue}) + Bonus ({Bonus}) = {Total}d10 vs DC {DC}",
            attributeType, attributeValue, rhetoricSkill, playerBonusDice, totalPlayerDice, effectiveDc);

        // Perform the roll
        var rollResult = await _diceRollerService.RollDicePoolAsync(totalPlayerDice);

        // Determine outcome
        SkillOutcome outcome = DetermineOutcome(rollResult, effectiveDc, npcBonusDice);

        _logger.LogInformation(
            "Intimidation result: {Successes} successes, {Botches} botches. Outcome: {Outcome}",
            rollResult.Successes, rollResult.Botches, outcome);

        // Calculate reputation cost (ALWAYS applies)
        string factionName = await _factionService.GetFactionNameAsync(context.TargetFactionId);
        var reputationCost = CostOfFear.Calculate(outcome, context.TargetFactionId, factionName);

        // Apply reputation cost
        await _factionService.ModifyReputationAsync(context.TargetFactionId, reputationCost.ReputationCost);

        _logger.LogInformation(
            "Applied Cost of Fear: {Cost} to faction {Faction}",
            reputationCost.ReputationCost, context.TargetFactionId);

        // Handle fumble if applicable
        FumbleConsequence? fumbleConsequence = null;
        bool combatInitiated = false;
        int stressCost = 0;

        if (outcome == SkillOutcome.Fumble)
        {
            fumbleConsequence = await HandleChallengeAcceptedAsync(context);
            combatInitiated = true;
            stressCost = 5; // Fumble stress cost
        }

        // Calculate compliance level
        var compliance = DetermineCompliance(outcome);

        // Calculate disposition change
        int dispositionChange = CalculateDispositionChange(outcome);

        // Determine unlocked options
        var unlockedOptions = await DetermineUnlockedOptionsAsync(context, outcome);

        // Build base result
        var baseResult = new SocialResult
        {
            InteractionType = SocialInteractionType.Intimidation,
            TargetId = context.BaseContext.TargetId,
            Outcome = outcome,
            DispositionChange = dispositionChange,
            SuccessMargin = rollResult.Successes - (effectiveDc / 4) // Approximation
        };

        return new IntimidationResult
        {
            BaseResult = baseResult,
            Outcome = outcome,
            TargetCompliance = compliance,
            ReputationCost = reputationCost,
            DispositionChange = dispositionChange,
            CombatInitiated = combatInitiated,
            StressCost = stressCost,
            UnlockedOptions = unlockedOptions,
            FumbleConsequence = fumbleConsequence
        };
    }

    /// <inheritdoc />
    public async Task<IntimidationTarget> ClassifyTargetAsync(string npcId)
    {
        var npc = await _npcService.GetNpcAsync(npcId);

        // Classification based on NPC properties
        if (npc.IsFactionLeader)
        {
            return IntimidationTarget.FactionLeader;
        }

        if (npc.IsElite || npc.WillPower >= 5)
        {
            return IntimidationTarget.Elite;
        }

        if (npc.IsVeteran || npc.CombatExperience >= 3)
        {
            return IntimidationTarget.Veteran;
        }

        if (npc.IsTimid || npc.WillPower <= 1)
        {
            return IntimidationTarget.Coward;
        }

        return IntimidationTarget.Common;
    }

    /// <inheritdoc />
    public async Task<RelativeStrength> CalculateRelativeStrengthAsync(string npcId)
    {
        int playerLevel = await _playerService.GetLevelAsync();
        int npcLevel = await _npcService.GetNpcLevelAsync(npcId);

        return RelativeStrengthExtensions.FromLevels(playerLevel, npcLevel);
    }

    /// <inheritdoc />
    public async Task<bool> HasReputationBonusAsync(string factionId)
    {
        var reputationTier = await _factionService.GetReputationTierAsync(factionId);

        // [Honored] or [Feared] provides bonus
        return reputationTier == ReputationTier.Honored ||
               reputationTier == ReputationTier.Feared;
    }

    /// <inheritdoc />
    public async Task<bool> IsWieldingArtifactAsync()
    {
        var mainHand = await _equipmentService.GetEquippedWeaponAsync();
        return mainHand?.IsArtifact ?? false;
    }

    /// <inheritdoc />
    public async Task<bool> HasIntimidatingAllyAsync()
    {
        var partyMembers = await _partyService.GetPartyMembersAsync();

        foreach (var member in partyMembers)
        {
            // Check for intimidating companion types
            if (member.IsKriger || member.IsMonster || member.HasIntimidatingPresence)
            {
                return true;
            }
        }

        return false;
    }

    /// <inheritdoc />
    public async Task<bool> TargetHasBackupAsync(string npcId)
    {
        return await _npcService.HasNearbyAlliesAsync(npcId);
    }

    /// <inheritdoc />
    public async Task<string> GetTargetFactionAsync(string npcId)
    {
        var npc = await _npcService.GetNpcAsync(npcId);
        return npc.FactionId;
    }

    /// <inheritdoc />
    public async Task<IntimidationContext> BuildContextAsync(string npcId, IntimidationApproach approach)
    {
        var baseContext = await _socialContextBuilder.BuildAsync(npcId, SocialInteractionType.Intimidation);
        var targetType = await ClassifyTargetAsync(npcId);
        var relativeStrength = await CalculateRelativeStrengthAsync(npcId);
        var factionId = await GetTargetFactionAsync(npcId);
        var hasReputation = await HasReputationBonusAsync(factionId);
        var wieldingArtifact = await IsWieldingArtifactAsync();
        var intimidatingAlly = await HasIntimidatingAllyAsync();
        var targetHasBackup = await TargetHasBackupAsync(npcId);

        return new IntimidationContextBuilder()
            .WithBaseContext(baseContext)
            .WithTargetType(targetType)
            .WithApproach(approach)
            .WithRelativeStrength(relativeStrength)
            .WithReputation(hasReputation)
            .WithArtifact(wieldingArtifact)
            .WithIntimidatingAlly(intimidatingAlly)
            .WithTargetBackup(targetHasBackup)
            .WithFaction(factionId)
            .Build();
    }

    /// <summary>
    /// Determines the skill outcome based on roll result and DC.
    /// </summary>
    private SkillOutcome DetermineOutcome(DiceRollResult roll, int dc, int npcBonusDice)
    {
        // Check for fumble first
        if (roll.IsFumble)
        {
            return SkillOutcome.Fumble;
        }

        // Calculate required successes (DC / 4, rounded up)
        int requiredSuccesses = (dc + 3) / 4;

        // Subtract NPC bonus dice worth of successes if applicable
        int effectiveSuccesses = roll.Successes - npcBonusDice;

        if (effectiveSuccesses < requiredSuccesses)
        {
            return SkillOutcome.Failure;
        }

        // Check for critical success (3+ excess successes)
        int excessSuccesses = effectiveSuccesses - requiredSuccesses;
        if (excessSuccesses >= 3)
        {
            return SkillOutcome.CriticalSuccess;
        }

        return SkillOutcome.Success;
    }

    /// <summary>
    /// Determines the compliance level based on outcome.
    /// </summary>
    private static TargetCompliance DetermineCompliance(SkillOutcome outcome)
    {
        return outcome switch
        {
            SkillOutcome.CriticalSuccess => TargetCompliance.Complete,
            SkillOutcome.Success => TargetCompliance.Full,
            SkillOutcome.Failure => TargetCompliance.None,
            SkillOutcome.Fumble => TargetCompliance.None,
            _ => TargetCompliance.None
        };
    }

    /// <summary>
    /// Calculates disposition change based on outcome.
    /// </summary>
    private static int CalculateDispositionChange(SkillOutcome outcome)
    {
        return outcome switch
        {
            SkillOutcome.CriticalSuccess => -2, // Completely cowed, fears player
            SkillOutcome.Success => -1,          // Reluctantly complies, resents player
            SkillOutcome.Failure => -1,          // Offended by the attempt
            SkillOutcome.Fumble => -3,           // Enraged
            _ => 0
        };
    }

    /// <summary>
    /// Determines unlocked dialogue options based on outcome.
    /// </summary>
    private async Task<IReadOnlyList<string>> DetermineUnlockedOptionsAsync(
        IntimidationContext context,
        SkillOutcome outcome)
    {
        if (outcome == SkillOutcome.Failure || outcome == SkillOutcome.Fumble)
        {
            return Array.Empty<string>();
        }

        var options = new List<string>();
        var npc = await _npcService.GetNpcAsync(context.BaseContext.TargetId);

        if (outcome == SkillOutcome.CriticalSuccess)
        {
            // Critical success unlocks more options
            options.Add("Demand information");
            options.Add("Force cooperation");
            options.Add("Extract a favor");

            if (npc.HasSecrets)
            {
                options.Add("Learn a secret");
            }
        }
        else if (outcome == SkillOutcome.Success)
        {
            // Standard success unlocks basic compliance
            options.Add("Demand passage");
            options.Add("Force compliance");
        }

        return options;
    }

    /// <summary>
    /// Handles the [Challenge Accepted] fumble consequence.
    /// </summary>
    private async Task<FumbleConsequence> HandleChallengeAcceptedAsync(IntimidationContext context)
    {
        _logger.LogWarning(
            "[Challenge Accepted] fumble triggered against {TargetId}. Initiating combat.",
            context.BaseContext.TargetId);

        // Apply [Furious] status to NPC
        await _statusEffectService.ApplyStatusAsync(
            context.BaseContext.TargetId,
            StatusEffectId.Furious,
            duration: -1, // Entire combat
            source: "ChallengeAccepted");

        // Apply psychic stress to player
        await _skillStressService.ApplyStressAsync(StressType.Psychic, 5);

        // Initiate combat with special flags
        await _combatService.InitiateCombatAsync(
            initiatorId: context.BaseContext.TargetId,
            targetId: "player",
            npcInitiativeBonus: 2,
            npcRefusesSurrender: true);

        // Create fumble consequence record
        return await _fumbleConsequenceService.CreateConsequenceAsync(
            FumbleType.ChallengeAccepted,
            new Dictionary<string, object>
            {
                ["TargetId"] = context.BaseContext.TargetId,
                ["FuriousBuff"] = true,
                ["RefusesSurrender"] = true,
                ["StressCost"] = 5,
                ["CombatInitiated"] = true
            });
    }
}

    {
        _logger.LogInformation(
            "Processing intimidation command against {TargetId} using {Approach}",
            request.TargetNpcId,
            request.Approach);

        // Build context
        var context = await _intimidationService.BuildContextAsync(
            request.TargetNpcId,
            request.Approach);

        // Perform intimidation
        var result = await _intimidationService.IntimidateAsync(context);

        _logger.LogInformation(
            "Intimidation complete. Outcome: {Outcome}, Reputation Cost: {Cost}",
            result.Outcome,
            result.ReputationCost.ReputationCost);

        return result;
    }
}

## 19. Use Cases

### 19.1 Basic Intimidation (Physical Approach)

**Scenario**: Player wants to physically intimidate a shopkeeper into lowering prices.

**Actors**: Player (Level 4, MIGHT 4, Rhetoric 2), Shopkeeper (Common tier, Level 2)

**Steps**:
1. Player initiates intimidation against shopkeeper
2. Player selects Physical approach (MIGHT)
3. System classifies target as Common (DC 12)
4. System calculates relative strength: PlayerStronger (+1d10)
5. System builds context with no additional modifiers
6. Player rolls 4 (MIGHT) + 2 (Rhetoric) + 1 (stronger) = 7d10
7. Result: 4 successes, 0 botches → Success
8. Shopkeeper complies reluctantly
9. Reputation cost: -5 to Merchant Guild

**Outcome**: Shopkeeper lowers prices, player loses 5 reputation with Merchant Guild.

### 19.2 High-Stakes Intimidation with Modifiers

**Scenario**: Player wants to intimidate a veteran guard captain to release a prisoner.

**Actors**: Player (Level 6, WILL 5, Rhetoric 3), Guard Captain (Veteran tier, Level 5)

**Context**:
- Player is [Feared] by the City Guard
- Player wields Fjall-bane Axe (Artifact)
- Kriger companion is present
- Guard captain has two guards nearby (backup)

**Steps**:
1. Player initiates intimidation
2. Player selects Mental approach (WILL) - cold, quiet threat
3. System classifies target as Veteran (DC 16)
4. System calculates relative strength: Equal (no modifier)
5. System identifies modifiers:
   - [Feared] reputation: +1d10
   - [Artifact] visible: +1d10
   - Intimidating ally: +1d10
   - NPC has backup: NPC +1d10
6. Player rolls 5 (WILL) + 3 (Rhetoric) + 3 (bonuses) = 11d10
7. NPC resistance: 1d10 (backup bonus)
8. Result: 5 successes - 1 (NPC bonus) = 4 net successes
9. Required: DC 16 / 4 = 4 successes → Success
10. Guard captain reluctantly orders prisoner release
11. Reputation cost: -5 to City Guard

**Outcome**: Prisoner released, player loses 5 reputation, guard captain remembers the insult.

### 19.3 Fumble: Challenge Accepted

**Scenario**: Player attempts to intimidate an elite mercenary and fumbles.

**Actors**: Player (Level 3, MIGHT 3, Rhetoric 1), Elite Mercenary (Elite tier, Level 6)

**Steps**:
1. Player initiates intimidation (Physical approach)
2. System classifies target as Elite (DC 20)
3. System calculates relative strength: PlayerWeaker (NPC +1d10)
4. Player has no additional bonuses
5. Player rolls 3 (MIGHT) + 1 (Rhetoric) = 4d10
6. Result: 0 successes, 2 botches → FUMBLE
7. [Challenge Accepted] triggers:
   - Combat initiated immediately
   - Mercenary gains [Furious] (+2d10 damage)
   - Mercenary will not accept surrender
   - +5 Psychic Stress to player
   - -10 reputation to Mercenary Guild
8. Mercenary attacks with initiative bonus

**Outcome**: Player must fight or flee against an enraged, empowered opponent.

## 23. Implementation Notes

### 23.1 Design Decisions

1. **Attribute Choice**: Allowing MIGHT or WILL provides flexibility for different character builds while keeping intimidation as a Rhetoric skill check.

2. **Mandatory Reputation Cost**: Unlike persuasion or deception, intimidation always damages relationships. This reflects the game's thematic position that fear-based coercion has social consequences.

3. **Challenge Accepted Severity**: The fumble consequence is severe (combat with buffed enemy) to discourage casual intimidation attempts and create meaningful risk.

4. **No Stress Cost (Normally)**: Unlike deception's Liar's Burden, intimidation doesn't inherently cause stress. The stress only comes from the violence of a fumble.

### 23.2 Performance Considerations

1. **Context Building**: The `BuildContextAsync` method makes multiple service calls; consider caching NPC data when building multiple contexts.

2. **Faction Lookups**: Reputation checks and modifications are frequent; ensure FactionService has appropriate caching.

### 23.3 Error Handling

1. **Invalid Target**: If target NPC cannot be intimidated (e.g., non-sapient creature), return early with appropriate error.

2. **Missing Faction**: If NPC has no faction, use a default "Neutral" faction for reputation tracking.

3. **Combat Already Active**: If combat is already active, intimidation attempts should be blocked or handled specially.
