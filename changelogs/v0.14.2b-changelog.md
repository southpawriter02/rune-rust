# v0.14.2b Changelog - Armor Schema

## Summary

Implemented `armor.schema.json` with comprehensive validation for armor and accessory definitions, including defense bonuses, penalty constraints, equipment slots, and material/quality configurations.

## Added

### Schema Files
- **[config/schemas/armor.schema.json](file:///Volumes/GitHub/github/southpawriter02/rune-rust/config/schemas/armor.schema.json)** - JSON Schema Draft-07 for armor configuration
  - `ArmorDefinition` - Full armor validation with slot, type, defense, penalties
  - `AccessoryDefinition` - Ring, amulet, belt, cloak validation with stat bonuses
  - `StatRequirements` - Might, agility, will prerequisite validation
  - `StatBonuses` - Health bonus and custom stat modifiers

### Validation Rules
| Property | Constraint | Description |
|----------|------------|-------------|
| `id` | `^[a-z][a-z0-9_]*$` | Lowercase with underscores |
| `defenseBonus` | `>= 0` | Non-negative defense values |
| `damageReduction` | `>= 0` | Non-negative damage reduction |
| `initiativePenalty` | `<= 0` | Non-positive penalty values |
| `movementPenalty` | `<= 0` | Non-positive penalty values |
| `slot` | Enum | Armor, Shield, Helmet, Boots, Gloves, Ring, Amulet, Belt, Cloak |
| `armorType` | Enum | Light, Medium, Heavy |
| `quality` | Enum | Common, Uncommon, Rare, Epic, Legendary |
| `material` | Enum | Leather, Chain, Plate, Cloth, Wood, Iron, Steel, Mithril |
