# v0.0.2 Changelog - Core Game Loop Polish

**Release Date:** 2026-01-06

## Overview

This release stabilizes the foundation and adds essential missing commands for the core game loop. Players can now drop items, use consumables, examine objects, and view their status.

## New Features

### Commands
- **Drop Command** (`drop <item>`) - Drop items from inventory into current room
- **Use Command** (`use <item>`, `consume`, `drink`, `eat`) - Use/consume items for effects
- **Examine Command** (`examine <target>`, `inspect`, `x`) - Examine items, monsters, or rooms
- **Status Command** (`status`, `stats`) - View comprehensive character statistics
- **Load Command** - Enhanced load game flow with saved game selection

### Item Effects System
- New `ItemEffect` enum: `None`, `Heal`, `Damage`, `BuffAttack`, `BuffDefense`
- `ItemEffectService` for applying effects to players
- Health Potions now properly heal players when consumed

### Room Visit Tracking
- First-visit vs. return-visit message differentiation (prepared for future re-entry messages)
- `GameSession.VisitedRooms` tracking with `MarkRoomVisited()` and `HasVisitedRoom()` methods

## Technical Changes

### Domain Layer
- Added `Effect`, `EffectValue`, `EffectDuration` properties to `Item` entity
- Added `TryRemove()` method to `Inventory` class
- Added visited room tracking to `GameSession`

### Application Layer
- New DTOs: `ExamineResultDto`, `PlayerStatsDto`
- Added `IsFirstVisit` to `RoomDto`
- New services: `ItemEffectService`, `InputValidationService`
- New `GameSessionService` methods: `TryDropItem()`, `TryUseItem()`, `GetExamineInfo()`, `GetPlayerStats()`

### Presentation Layer
- Updated `ConsoleInputHandler` with new command parsing
- New render methods: `RenderExamineResultAsync()`, `RenderPlayerStatsAsync()`
- Updated help table with all new commands

### Infrastructure
- Updated `DependencyInjection` to register new services
