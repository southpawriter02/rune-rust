# v0.15.3 - Rhetoric Skill Expansion

**Focus:** Persuasion, deception, intimidation, negotiation, interrogation, and cultural protocols

**Source Specification:** [rhetoric.md](../../aethelgard/design/01-core/skills/rhetoric.md)

### Overview

This version transforms Rhetoric from simple social checks into a comprehensive social combat system. Characters can engage in nuanced persuasion, craft elaborate deceptions, intimidate through presence or threat, conduct multi-round negotiations, and interrogate subjects with varying methods.

### v0.15.3b - Persuasion System

#### PersuasionRequest Enum
```
PersuasionRequest
├── Trivial (DC 8) - "Tell me the time"
├── Simple (DC 12) - "Share your rations"
├── Moderate (DC 16) - "Help me fight"
├── Major (DC 20) - "Attack your faction's enemies"
├── Extreme (DC 24) - "Betray your clan leader"
```

#### PersuasionContext Value Object
- [ ] `RequestType` - Complexity of request
- [ ] `ArgumentAligned` - Does argument match NPC values?
- [ ] `EvidenceProvided` - Supporting proof
- [ ] `NpcStressed` - Under duress
- [ ] `PreviousAttempts` - Failed attempts count

**Persuasion Modifiers:**
| Factor | Modifier |
|--------|----------|
| NPC from [Friendly] faction | Player +1d10 |
| NPC from [Unfriendly] faction | NPC +1d10 |
| Argument aligns with values | Player +1d10 |
| Argument contradicts values | NPC +1d10 |
| NPC is [Stressed/Feared] | NPC +1d10 |
| Evidence supports argument | Player +2d10 |

**Outcomes:**
- [ ] Success: Request granted, often +reputation
- [ ] Critical Success: Deep conviction, additional options
- [ ] Failure: Unswayed, option removed, minor rep loss
- [ ] Fumble: [Trust Shattered] - All persuasion locked

**Unit Tests (~3):**

### v0.15.3e - Negotiation System

#### NegotiationPositionTrack Entity
**Purpose:** Track positions during multi-round negotiation

```
NegotiationPositionTrack
├── NegotiationId: string
├── PcPosition: int (0-8 scale)
├── NpcPosition: int (0-8 scale)
├── Gap: int (calculated distance)
├── RoundNumber: int
├── RoundsRemaining: int
├── Status: NegotiationStatus enum
└── History: List<NegotiationRound>
```

**Position Track:**
```
[Maximum Demand] ← [Strong] ← [Favorable] ← [Compromise] → [Unfavorable] → [Weak] → [Walk Away]
        0              2           4              5              6           7          8
```

#### NegotiationStatus Enum
```
NegotiationStatus
├── Opening
├── Bargaining
├── CrisisManagement
├── Finalization
├── DealReached
├── Collapsed
```

#### NegotiationRound Value Object
- [ ] `RoundNumber` - Which round
- [ ] `TacticUsed` - Persuade, Deceive, Pressure, Concede
- [ ] `CheckResult` - Outcome of check
- [ ] `PositionMovement` - How positions changed

**Negotiation Logic:**
- [ ] Start with gap assessment (close, moderate, far)
- [ ] Each round: choose tactic and roll
- [ ] Critical Success: Move opponent 2 steps
- [ ] Success: Move opponent 1 step
- [ ] Failure: Move self 1 step toward opponent
- [ ] Critical Failure: Move self 2 steps
- [ ] Concession: Give ground voluntarily, +2d10 next check
- [ ] Deal when positions overlap at Compromise

**Request Complexity:**
| What You're Asking | DC |
|-------------------|-----|
| Fair trade | 10 |
| Slight advantage | 14 |
| Noticeable advantage | 18 |
| Major advantage | 22 |
| One-sided deal | 26 |

**Concession Effects:**
| Concession | DC Reduction |
|------------|--------------|
| Offer item of value | -2 |
| Promise future favor | -2 |
| Information trade | -4 |
| Take personal risk | -4 |
| Faction reputation on line | -6 |

**Unit Tests (~4):**
- [ ] Position track calculates gap correctly
- [ ] Success moves opponent position
- [ ] Failure moves own position
- [ ] Concession grants bonus on next check

### v0.15.3f - Interrogation System

#### InterrogationMethod Enum
```
InterrogationMethod
├── GoodCop (DC 14, 30 min, no side effects)
├── BadCop (DC 12, 15 min, -5 disposition)
├── Deception (DC 16, 20 min, unreliable info)
├── Bribery (DC 10, 10 min, costs resources)
├── Torture (varies, trauma, unreliable, -30 rep)
```

#### InterrogationState Entity
```
InterrogationState
├── InterrogationId: string
├── InterrogatorId: string
├── SubjectId: string
├── SubjectWill: int
├── ResistanceRemaining: int
├── MethodsUsed: List<InterrogationMethod>
├── InformationGained: List<string>
├── DispositionChanges: int
└── Status: InterrogationStatus enum
```

#### SubjectResistance Value Object
| Resistance Level | Checks to Break |
|------------------|-----------------|
| Minimal (frightened civilian) | 1 |
| Low (hired muscle) | 2-3 |
| Moderate (loyal soldier) | 4-5 |
| High (trained operative) | 6-8 |
| Extreme (fanatical believer) | 10+ |

#### InformationReliability Value Object
| Method | Reliability |
|--------|-------------|
| Good Cop | 95% accurate |
| Bad Cop | 80% accurate |
| Deception | 70% (subject may lie) |
| Bribery | 90% accurate |
| Torture | 50% (will say anything) |

**Interrogation Logic:**
- [ ] Assess subject WILL to determine base resistance
- [ ] Each round: choose method and roll
- [ ] Success: Resistance -1
- [ ] Failure: No progress
- [ ] When resistance = 0, subject breaks
- [ ] Can switch methods between rounds
- [ ] Method determines reliability of information

**Unit Tests (~3):**
- [ ] Subject WILL sets resistance correctly
- [ ] Method success reduces resistance
- [ ] Reliability affects information quality

ExtendedInfluence

ConvictionLevel
