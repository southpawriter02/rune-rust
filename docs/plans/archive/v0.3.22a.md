## Overview

This sub-version addresses the "wall of text" problem in combat logs by introducing a structured `CombatLogEntry` model that replaces the current `List<string>` approach. The new system categorizes every combat message by type (Mechanics, Flavor, System, Status) and allows players to filter which categories appear in the log based on their preferences.

**Current State Analysis:**
- `CombatViewModel.CombatLog` is `List<string>` (raw markup strings)
- `CombatLogFormatter` is a stateless static class returning formatted strings
- `CombatScreenRenderer.RenderCombatLog()` displays all entries unconditionally
- `GameSettings` lacks any combat verbosity controls
- No structured metadata exists for filtering decisions

**Target State:**
- `CombatViewModel.CombatLog` becomes `List<CombatLogEntry>` (structured records)
- `CombatLogFormatter` returns `CombatLogEntry` records with category metadata
- `CombatScreenRenderer` filters entries based on `GameSettings` flags before rendering
- New `GameSettings` properties control which categories to display
- Important events (critical hits, deaths) bypass filters via `IsImportant` flag

## Use Cases

### 1. Default Experience (All Filters On)
**Scenario:** New player enters combat for the first time.
**System Behavior:**
- All `GameSettings` filter flags default to `true`
- Combat log displays all categories: damage numbers, dice rolls, flavor text, status changes
- Player sees full verbose output as intended by designers

### 2. Minimal Verbosity (Stealth/Speedrun Mode)
**Scenario:** Experienced player wants only critical information.
**Actions:**
1. Player opens Options → Combat Log submenu
2. Disables "Show Dice Rolls" and "Show Flavor Text"
3. Returns to combat

**Result:**
- Combat log only shows: damage dealt/received, deaths, status applications
- Reduces log clutter by ~40-60%
- Critical events (crits, deaths) still appear regardless of filters

### 3. Flavor-Only Mode (Roleplayer)
**Scenario:** Player wants immersive narration without mechanics.
**Actions:**
1. Disable "Show Dice Rolls" and "Show Damage Numbers"
2. Keep "Show Flavor Text" enabled

**Result:**
- Log shows: "The Draugr's blade bites deep!" instead of "Draugr hits you for 12 damage"
- Status changes show narrative descriptions
- Combat feels more like prose than spreadsheet

### 4. Critical Event Override
**Scenario:** Player has disabled all filters but lands a critical hit.
**System Behavior:**
- `CombatLogEntry.IsImportant = true` for critical hits, deaths, and boss phase changes
- Even with all filters disabled, the critical hit message appears
- Ensures players never miss pivotal combat moments

### 5. Session Persistence
**Scenario:** Player adjusts filters and restarts the game.
**System Behavior:**
1. On exit, `SettingsService.SaveAsync()` writes filter states to `options.json`
2. On next launch, `SettingsService.LoadAsync()` restores filter preferences
3. Combat log respects restored settings immediately

## Logic Decision Trees

### A. Filter Check Logic

**Input:** `CombatLogEntry entry`, `GameSettings settings`

```
START: Receive entry for render
  │
  ├─► Is entry.IsImportant == true?
  │     └─► YES: RENDER (bypass all filters)
  │
  └─► NO: Check entry.Category
        │
        ├─► Category == LogCategory.System
        │     └─► RENDER (always shown: round markers, turn announcements)
        │
        ├─► Category == LogCategory.Mechanics
        │     └─► Is settings.ShowCombatRolls == true?
        │           ├─► YES: RENDER
        │           └─► NO: SKIP
        │
        ├─► Category == LogCategory.Flavor
        │     └─► Is settings.ShowEnemyBarks == true?
        │           ├─► YES: RENDER
        │           └─► NO: SKIP
        │
        └─► Category == LogCategory.Status
              └─► Is settings.ShowStatusChanges == true?
                    ├─► YES: RENDER
                    └─► NO: SKIP
```

### B. Category Assignment Logic

**Input:** Combat event type

```
Event Type               → Category         → IsImportant
─────────────────────────────────────────────────────────
RoundStart               → System           → false
TurnStart                → System           → false
AttackHit (normal)       → Mechanics        → false
AttackHit (critical)     → Mechanics        → TRUE
AttackMiss               → Mechanics        → false
AttackGlancing           → Mechanics        → false
StatusApplied            → Status           → false
StatusExpired            → Status           → false
EnemyDeath               → System           → TRUE
PlayerDeath              → System           → TRUE
Victory                  → System           → TRUE
Defeat                   → System           → TRUE
AbilityUse               → Mechanics        → false
HealReceived             → Mechanics        → false
EnemyBark (dialogue)     → Flavor           → false
EnvironmentDescription   → Flavor           → false
BossPhaseChange          → System           → TRUE
```

### C. Settings Validation Logic

**Input:** Raw DTO values from JSON

```
For each boolean filter setting:
  └─► Is value a valid boolean?
        ├─► YES: Apply to GameSettings
        └─► NO: Default to true (show all)
```

```csharp
using RuneAndRust.Core.Enums;
using RuneAndRust.Core.Models.Combat;

namespace RuneAndRust.Engine.Services;

/// <summary>
/// Formats combat events into structured CombatLogEntry records (v0.3.22a).
/// Centralizes all combat log formatting with category assignment and importance flagging.
/// Stateless static class - no DI registration required.
/// </summary>
public static class CombatLogFormatter
{
    #region Damage Color Mapping

    /// <summary>
    /// Gets the Spectre.Console color name for a damage type.
    /// </summary>
    public static string GetDamageColor(DamageType type) => type switch
    {
        DamageType.Physical => "white",
        DamageType.Fire => "orange1",
        DamageType.Ice => "cyan1",
        DamageType.Lightning => "yellow",
        DamageType.Poison => "green",
        DamageType.Acid => "lime",
        DamageType.Psychic => "magenta",
        DamageType.Blight => "maroon",
        _ => "white"
    };

    #endregion

    #region Attack Messages

    /// <summary>
    /// Formats a successful attack message with damage type coloring.
    /// </summary>
    /// <param name="attackerName">Name of the attacker.</param>
    /// <param name="targetName">Name of the target.</param>
    /// <param name="damage">Final damage dealt.</param>
    /// <param name="damageType">Type of damage dealt.</param>
    /// <param name="turnNumber">Current turn number for entry metadata.</param>
    /// <param name="isCritical">Whether this was a critical hit.</param>
    /// <returns>A structured CombatLogEntry for the combat log.</returns>
    public static CombatLogEntry FormatHitMessage(
        string attackerName,
        string targetName,
        int damage,
        DamageType damageType,
        int turnNumber,
        bool isCritical = false)
    {
        var color = GetDamageColor(damageType);
        var critPrefix = isCritical ? "[bold yellow]CRITICAL![/] " : "";
        var damageLabel = damageType == DamageType.Physical ? "" : $" {damageType}";

        var markup = $"{critPrefix}[cyan]{EscapeMarkup(attackerName)}[/] hits [red]{EscapeMarkup(targetName)}[/] for [{color}]{damage}{damageLabel}[/] damage!";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Mechanics,
            TurnNumber: turnNumber,
            IsImportant: isCritical  // Crits always bypass filters
        );
    }

    /// <summary>
    /// Formats a miss message.
    /// </summary>
    public static CombatLogEntry FormatMissMessage(
        string attackerName,
        string targetName,
        int turnNumber)
    {
        var markup = $"[cyan]{EscapeMarkup(attackerName)}[/] attacks [red]{EscapeMarkup(targetName)}[/] but [grey]misses![/]";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Mechanics,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats a glancing blow message (reduced damage).
    /// </summary>
    public static CombatLogEntry FormatGlancingMessage(
        string attackerName,
        string targetName,
        int damage,
        DamageType damageType,
        int turnNumber)
    {
        var color = GetDamageColor(damageType);
        var damageLabel = damageType == DamageType.Physical ? "" : $" {damageType}";

        var markup = $"[cyan]{EscapeMarkup(attackerName)}[/] [grey]glances[/] [red]{EscapeMarkup(targetName)}[/] for [{color}]{damage}{damageLabel}[/] damage.";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Mechanics,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    #endregion

    #region Status Effect Messages

    /// <summary>
    /// Formats a status effect application message.
    /// </summary>
    public static CombatLogEntry FormatStatusApplied(
        string targetName,
        string effectName,
        int turnNumber)
    {
        var markup = $"[red]{EscapeMarkup(targetName)}[/] is now [yellow]{effectName}[/]!";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Status,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats a status effect expiration message.
    /// </summary>
    public static CombatLogEntry FormatStatusExpired(
        string targetName,
        string effectName,
        int turnNumber)
    {
        var markup = $"[grey]{effectName}[/] wears off from [cyan]{EscapeMarkup(targetName)}[/].";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Status,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    #endregion

    #region Death Messages

    /// <summary>
    /// Formats a death message.
    /// </summary>
    public static CombatLogEntry FormatDeathMessage(
        string targetName,
        bool isPlayer,
        int turnNumber)
    {
        var markup = isPlayer
            ? $"[bold red]{EscapeMarkup(targetName)} has fallen![/]"
            : $"[red]{EscapeMarkup(targetName)}[/] is [grey]defeated![/]";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.System,
            TurnNumber: turnNumber,
            IsImportant: true  // Deaths always bypass filters
        );
    }

    #endregion

    #region Combat Flow Messages

    /// <summary>
    /// Formats a round start message.
    /// </summary>
    public static CombatLogEntry FormatRoundStart(int roundNumber)
    {
        var markup = $"[bold white]─── Round {roundNumber} ───[/]";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.System,
            TurnNumber: 0,  // Round markers are turn-agnostic
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats a turn start message.
    /// </summary>
    public static CombatLogEntry FormatTurnStart(
        string combatantName,
        bool isPlayer,
        int turnNumber)
    {
        var color = isPlayer ? "cyan" : "red";
        var markup = $"[{color}]{EscapeMarkup(combatantName)}[/]'s turn.";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.System,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats a combat victory message.
    /// </summary>
    public static CombatLogEntry FormatVictory(int turnNumber)
    {
        return new CombatLogEntry(
            Markup: "[bold green]Victory! All enemies defeated.[/]",
            Category: LogCategory.System,
            TurnNumber: turnNumber,
            IsImportant: true  // Victory always bypasses filters
        );
    }

    /// <summary>
    /// Formats a combat defeat message.
    /// </summary>
    public static CombatLogEntry FormatDefeat(int turnNumber)
    {
        return new CombatLogEntry(
            Markup: "[bold red]Defeat! You have fallen in battle.[/]",
            Category: LogCategory.System,
            TurnNumber: turnNumber,
            IsImportant: true  // Defeat always bypasses filters
        );
    }

    #endregion

    #region Ability Messages

    /// <summary>
    /// Formats an ability use message.
    /// </summary>
    public static CombatLogEntry FormatAbilityUse(
        string casterName,
        string abilityName,
        int turnNumber)
    {
        var markup = $"[cyan]{EscapeMarkup(casterName)}[/] uses [yellow]{abilityName}[/]!";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Mechanics,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats a healing message.
    /// </summary>
    public static CombatLogEntry FormatHealMessage(
        string targetName,
        int amount,
        int turnNumber)
    {
        var markup = $"[cyan]{EscapeMarkup(targetName)}[/] recovers [green]{amount}[/] HP!";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Mechanics,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    #endregion

    #region Flavor Messages (v0.3.22a)

    /// <summary>
    /// Formats an enemy bark (dialogue/taunt) message.
    /// </summary>
    public static CombatLogEntry FormatEnemyBark(
        string enemyName,
        string barkText,
        int turnNumber)
    {
        var markup = $"[italic grey]\"{EscapeMarkup(barkText)}\"[/] [dim]— {EscapeMarkup(enemyName)}[/]";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Flavor,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    /// <summary>
    /// Formats an environmental flavor description.
    /// </summary>
    public static CombatLogEntry FormatEnvironmentFlavor(
        string description,
        int turnNumber)
    {
        var markup = $"[italic dim]{EscapeMarkup(description)}[/]";

        return new CombatLogEntry(
            Markup: markup,
            Category: LogCategory.Flavor,
            TurnNumber: turnNumber,
            IsImportant: false
        );
    }

    #endregion

    #region Legacy Compatibility (v0.3.22a)

    /// <summary>
    /// Creates a CombatLogEntry from a raw markup string for backward compatibility.
    /// Used during migration from List&lt;string&gt; to List&lt;CombatLogEntry&gt;.
    /// </summary>
    /// <param name="markup">Raw Spectre.Console markup string.</param>
    /// <param name="category">Category to assign (defaults to System).</param>
    /// <param name="turnNumber">Turn number (defaults to 0).</param>
    /// <param name="isImportant">Importance flag (defaults to false).</param>
    /// <returns>A CombatLogEntry wrapping the raw string.</returns>
    public static CombatLogEntry FromLegacyString(
        string markup,
        LogCategory category = LogCategory.System,
        int turnNumber = 0,
        bool isImportant = false)
    {
        return new CombatLogEntry(markup, category, turnNumber, isImportant);
    }

    #endregion

    #region Helper Methods

    /// <summary>
    /// Escapes special Spectre.Console markup characters in text.
    /// </summary>
    private static string EscapeMarkup(string text)
    {
        return text.Replace("[", "[[").Replace("]", "]]");
    }

    #endregion
}
```

### H. Terminal Layer: Updated CombatScreenRenderer

**File:** `RuneAndRust.Terminal/Services/CombatScreenRenderer.cs` (updated RenderCombatLog)

```csharp
using RuneAndRust.Core.Models.Combat;
using RuneAndRust.Core.Settings;

/// <summary>
/// Renders the combat log panel with filter support (v0.3.22a).
/// Filters entries based on GameSettings before display.
/// </summary>
private static void RenderCombatLog(int roundNumber, List<CombatLogEntry> combatLog)
{
    // Filter entries based on GameSettings
    var filteredEntries = FilterLogEntries(combatLog);

    var logContent = filteredEntries.Count > 0
        ? string.Join("\n", filteredEntries.Select(e => e.Markup))
        : "[grey]No combat events yet.[/]";

    var panel = new Panel(new Markup(logContent))
    {
        Header = new PanelHeader($"[bold]Round {roundNumber}[/]"),
        Border = BoxBorder.Heavy,
        BorderStyle = Style.Parse("grey"),
        Padding = new Padding(1, 0, 1, 0)
    };

    AnsiConsole.Write(panel);
}

/// <summary>
/// Filters combat log entries based on GameSettings (v0.3.22a).
/// Important entries always pass through regardless of filter settings.
/// </summary>
/// <param name="entries">The full list of combat log entries.</param>
/// <returns>Filtered list of entries to display.</returns>
private static List<CombatLogEntry> FilterLogEntries(List<CombatLogEntry> entries)
{
    var result = new List<CombatLogEntry>();

    foreach (var entry in entries)
    {
        if (ShouldShowEntry(entry))
        {
            result.Add(entry);
        }
    }

    return result;
}

/// <summary>
/// Determines if a single log entry should be displayed based on its category
/// and the current GameSettings filter state.
/// </summary>
/// <param name="entry">The entry to evaluate.</param>
/// <returns>True if the entry should be displayed, false if filtered out.</returns>
private static bool ShouldShowEntry(CombatLogEntry entry)
{
    // Important entries (crits, deaths, victory/defeat) always show
    if (entry.IsImportant)
        return true;

    // System category always shows (round markers, turn announcements)
    if (entry.Category == LogCategory.System)
        return true;

    // Check category-specific filters
    return entry.Category switch
    {
        LogCategory.Mechanics => GameSettings.ShowCombatRolls,
        LogCategory.Flavor => GameSettings.ShowEnemyBarks,
        LogCategory.Status => GameSettings.ShowStatusChanges,
        _ => true  // Unknown categories default to visible
    };
}
```

## Changelog: v0.3.22a - The Filter

**Release Date:** 2026-XX-XX

### Summary
"The Filter" introduces structured combat logging with user-controlled verbosity. Players can now customize which types of combat messages appear in the log, reducing information overload while ensuring critical events like deaths and critical hits are never missed.

### Features
- **Structured Combat Log:** Combat messages are now categorized (Mechanics, Flavor, System, Status) with metadata for intelligent filtering.
- **Log Filter Options:** New "Combat Log" submenu in Options allows toggling:
  - **Show Dice Rolls:** Toggle damage numbers and attack roll details
  - **Show Enemy Barks:** Toggle enemy dialogue and flavor text
  - **Show Status Changes:** Toggle status effect application/expiration messages
  - **Verbose Mode:** Toggle detailed mechanical breakdowns
- **Important Event Protection:** Critical hits, deaths, and victory/defeat messages bypass all filters and always appear.
- **Persistent Preferences:** Filter settings save to `options.json` and persist across sessions.

### Technical
- Added `LogCategory` enum with System, Mechanics, Flavor, Status values
- Added `CombatLogEntry` record with Markup, Category, TurnNumber, and IsImportant fields
- Refactored `CombatLogFormatter` to return structured entries instead of raw strings
- Updated `CombatViewModel.CombatLog` from `List<string>` to `List<CombatLogEntry>`
- Added filter logic to `CombatScreenRenderer.RenderCombatLog()`
- Extended `GameSettings` and `SettingsDto` with four new boolean properties
- Extended `OptionsController` with Combat Log submenu handling
