# v0.4.4: Spells & Chants (The Mystic)

This version implements the "Mystic" archetype's core mechanics: the ability to cast spells using Aetheric energy. It introduces the `cast` command, the `Aether` resource management (Flux), and the concept of multi-turn "Chants" which require concentration and can be interrupted.

**Note on Dependencies:** This plan assumes `v0.4.3 Magic Core` (AetherService, Spell Entity) is either completed or will be implemented as a prerequisite within `v0.4.4a`. Given the current state where `v0.4.3` is pending, `v0.4.4a` includes the necessary infrastructure.

## v0.4.4a: The Catalyst (Magic Core & Casting)

**Goal:** Establish the fundamental data structures for Magic (`Spell`, `School`, `Aether`) and implement the `MagicService` to handle the `cast` command resolution.

### 1. Architecture & Data Flow

*   **Aether (Resource):** A derived stat (like Stamina) but with a "Flux" mechanic.
    *   **Current Aether:** The pool available to spend.
    *   **Flux:** A risk meter that increases with powerful spells (Paradox risk).
*   **Spell Entity:**
    *   `Id`: Unique string (e.g., `spell_wyrdlight`).
    *   `Name`: Display name.
    *   `Cost`: Aether cost.
    *   `CastTime`: `Instant` or `Chant(turns)`.
    *   `School`: `Entropy` (Destruction), `Mending` (Healing), `Vision` (Utility).
    *   `Effect`: The `EffectScript` to run (reusing Combat system).
*   **Command:** `cast [spell_name] [target]`.

**Data Flow: Casting a Spell**
1.  **Input:** Player types `cast light`.
2.  **Lookup:** `SpellRegistry` finds spell matching "light".
3.  **Validation:**
    *   Does player know the spell?
    *   Is there enough Aether?
    *   Is target valid?
4.  **Execution:**
    *   Deduct Cost.
    *   If `Instant`: Resolve Effect immediately.
    *   If `Chant`: Begin `ChantState` (handled in v0.4.4b).

### 2. Logic Decision Trees

**A. Cast Command Input**
1.  **Parse:** `cast <spell> [target]`.
2.  **Identify Spell:** Fuzzy match `<spell>` against known spells.
    *   *Match not found:* "You do not know the Galdr for '<spell>'."
3.  **Check Resources:** `CurrentAether >= Spell.Cost`.
    *   *Fail:* "Your spark is too dim (Need {Cost}, have {Current})."
4.  **Check Target:** If spell requires target, validate presence.
5.  **Result:** Call `MagicService.Cast(player, spell, target)`.

### 3. Code Implementation

**A. Core Models**
`RuneAndRust.Core/Models/Magic/Spell.cs`
`RuneAndRust.Core/Models/Magic/MagicSchool.cs` (Enum)
`RuneAndRust.Core/Models/Magic/AetherPool.cs` (Struct/Class on Character)

**B. Magic Service**
`RuneAndRust.Engine/Services/MagicService.cs`

```csharp
public class MagicService : IMagicService
{
    public CastResult Cast(Character caster, Spell spell, Entity target)
    {
        if (caster.Aether.Current < spell.Cost)
            return CastResult.InsufficientAether;

        caster.Aether.Modify(-spell.Cost);

        if (spell.CastTime == CastTime.Instant)
        {
            _effectService.Apply(spell.Effect, caster, target);
            return CastResult.Success;
        }
        else
        {
            // Start Chant (v0.4.4b)
            return CastResult.StartedChant;
        }
    }
}
```

## v0.4.4b: The Chant (Multi-Turn Actions)

**Goal:** Implement the concept of "Chanting" — actions that span multiple turns. This requires a new state in the Game Loop or Character to track "Concentration".

### 1. Architecture & Data Flow

*   **Concentration State:**
    *   `ActiveChant`: The spell being cast.
    *   `TurnsRemaining`: Integer countdown.
    *   `Target`: The original target.
*   **Interruption:**
    *   Taking damage triggers a **Concentration Check** (Constitution Save).
    *   Movement usually breaks Chanting.
    *   "Wait" command continues the Chant.

**Data Flow: Turn Update (Chanting)**
1.  **Start Turn:** Check `Character.Concentration`.
2.  **If Chanting:**
    *   Decrement `TurnsRemaining`.
    *   Log: "You continue to weave the Galdr..."
    *   **If TurnsRemaining == 0:**
        *   Trigger Spell Effect.
        *   Clear Concentration.
        *   Log: "The Galdr is complete!"
3.  **Input:** If user types `move` or `attack`, warn "This will break your chant."

### 2. Logic Decision Trees

**A. Taking Damage While Chanting**
1.  **Event:** `DamageTakenEvent`.
2.  **Check:** Is victim Chanting?
3.  **Save:** Roll `ConSave` (DC 10 or Damage/2).
4.  **Result:**
    *   *Pass:* "You maintain focus through the pain."
    *   *Fail:* "Your weaving unravels!" -> `CancelChant()`, maybe `AetherBacklash` (dmg).

### 3. Code Implementation

**A. Concentration Model**
`RuneAndRust.Core/Models/Magic/Concentration.cs`

**B. Turn Logic Integration**
Modify `GameLoop` or `TurnManager` to handle `ChantPhase`.

```csharp
// Inside Turn Loop
if (player.Concentration.IsActive)
{
    player.Concentration.Advance();
    if (player.Concentration.IsComplete)
    {
        _magicService.ResolveSpell(player.Concentration.Spell);
    }
    return; // Skip input for this turn (or require 'wait')
}
```
