## Overview

This version introduces two JSON Schemas: Skills Schema (`skills.schema.json`) and Archetypes Schema (`archetypes.schema.json`). These enable IDE autocomplete and validation for skill definitions with attribute links and category classifications, and archetype definitions with playstyle metadata and stat tendencies.

---

## Added

### Configuration Schemas
- **`config/schemas/skills.schema.json`** - JSON Schema Draft-07 for skill definitions
  - Validates `SkillDefinition` with attribute enum (might, fortitude, will, wits, finesse)
  - Validates category enum (Physical, Mental, Social, Knowledge, Specialty)
  - Validates dice pool pattern (`^\\d+d\\d+$`)
  - Supports optional secondaryAttribute and untrainedPenalty

- **`config/schemas/archetypes.schema.json`** - JSON Schema Draft-07 for archetype definitions
  - Validates `ArchetypeDefinition` with statTendency enum (offensive, defensive, balanced)
  - Validates playstyleSummary for character creation UI
  - Enforces kebab-case ID pattern

### Unit Tests
- **`SkillSchemaTests.cs`** - 4 tests
  - `Schema_IsValidJsonSchemaDraft07` - Validates schema structure
  - `ExistingSkillsJson_ValidatesAgainstSchema` - Validates existing configuration
  - `PrimaryAttribute_InvalidEnumValue_FailsValidation` - Tests attribute enum
  - `Category_InvalidEnumValue_FailsValidation` - Tests category enum

- **`ArchetypeSchemaTests.cs`** - 3 tests
  - `Schema_IsValidJsonSchemaDraft07` - Validates schema structure
  - `ExistingArchetypesJson_ValidatesAgainstSchema` - Validates existing configuration
  - `StatTendency_InvalidEnumValue_FailsValidation` - Tests statTendency enum

## Changed

### Configuration Files
