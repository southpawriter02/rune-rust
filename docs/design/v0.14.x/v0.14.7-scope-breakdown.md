## Executive Summary

The Descriptor Configuration Schemas version creates comprehensive JSON schemas for all text descriptor configuration files, enabling full customization of sensory, combat, and environment flavor text without code changes. This supports modding scenarios like custom combat narration, themed atmosphere descriptions, localization variations, or genre-specific text pools.

The work is divided into **three sub-parts**:

| Part | Name | Focus | Est. Tests |
|------|------|-------|------------|
| v0.14.7a | Sensory Descriptor Schema | descriptors.schema.json with pools, weighted selection, context tags, variable substitution | ~6 |
| v0.14.7b | Combat Descriptor Schema | combat-descriptors.schema.json with hit, death, critical, miss descriptions | ~6 |
| v0.14.7c | Environment Descriptor Schema | environment-descriptors.schema.json with room, object, ambient, weather text | ~6 |

- Descriptions on all properties

## v0.14.7a: Sensory Descriptor Schema

[v0.14.7a Design Specification](v0.14.7a-design-specification.md)

### Overview

Create a comprehensive master JSON schema for descriptor definitions that validates pool structure, weighted selection, context tags, themes, and variable substitution placeholders.

### Scope

**In Scope:**
- `descriptors.schema.json` - Master schema for all descriptor files
- Category definition (root-level category identifier)
- Pool definitions (named collections of descriptors)
- Weighted selection (probability distribution via weights)
- Context tags (biome, location, situation filters)
- Theme tags (genre/mood filters)
- Variable substitution patterns (`{placeholder}` syntax)
- Conditional properties (minDamagePercent, maxDamagePercent)

**Out of Scope:**
- Combat-specific schema (v0.14.7b)
- Environment-specific schema (v0.14.7c)
- Descriptor selection logic (runtime concern)

### Key Deliverables

| Type | Count | Details |
|------|-------|---------|
| JSON Schemas | 1 | `descriptors.schema.json` |
| Schema Definitions | 4 | `DescriptorPool`, `Descriptor`, `WeightedText`, `VariablePattern` |
| Configuration Updates | 0 | Base schema referenced by others |
| Unit Tests | ~6 | Schema validation tests |

### Schema Structure

```
descriptors.schema.json
|-- $schema: "http://json-schema.org/draft-07/schema#"
|-- title: "Descriptor Configuration"
|-- description: "Master schema for text descriptor pools"
|-- type: object
|-- properties:
|   |-- $schema: string
|   |-- version: string (optional)
|   |-- category: string (pattern: ^[a-z][a-z0-9-]*$)
|   +-- pools: object (additionalProperties: DescriptorPool)
|-- required: ["category", "pools"]
+-- definitions:
    |-- DescriptorPool
    |   |-- id: string (pattern: ^[a-z][a-z0-9_]*$)
    |   |-- name: string (minLength: 1)
    |   |-- description: string (optional)
    |   +-- descriptors: array of Descriptor (minItems: 1)
    |-- Descriptor
    |   |-- id: string (pattern: ^[a-z][a-z0-9_]*$)
    |   |-- text: string (minLength: 1)
    |   |-- weight: integer (minimum: 1, default: 100)
    |   |-- tags: array of string (optional, context filters)
    |   |-- themes: array of string (optional, genre filters)
    |   |-- minDamagePercent: number (minimum: 0, maximum: 1, optional)
    |   |-- maxDamagePercent: number (minimum: 0, maximum: 1, optional)
    |   |-- minHealthPercent: number (minimum: 0, maximum: 1, optional)
    |   |-- maxHealthPercent: number (minimum: 0, maximum: 1, optional)
    |   |-- variables: array of string (optional, documents {placeholders})
    |   +-- fallbackId: string (optional, use if conditions not met)
    +-- VariablePattern
        |-- pattern: string (regex for {placeholder} syntax)
        +-- description: string
```

## v0.14.7b: Combat Descriptor Schema

[v0.14.7b Design Specification](v0.14.7b-design-specification.md)

### Overview

Create a specialized JSON schema for combat-related descriptors that extends the master schema with combat-specific validations for hit, death, critical, and miss descriptions.

### Scope

**In Scope:**
- `combat-descriptors.schema.json` - Schema for combat text
- Hit descriptions by weapon type (sword, axe, bow, staff, etc.)
- Death descriptions by creature type (humanoid, undead, beast, etc.)
- Critical hit descriptions (enhanced impact text)
- Miss descriptions (failure flavor text)
- Fumble descriptions (critical miss text)
- Damage threshold filters (minDamagePercent, maxDamagePercent)

**Out of Scope:**
- Master descriptor schema (v0.14.7a)
- Environment descriptor schema (v0.14.7c)
- Combat execution logic (runtime concern)

### Key Deliverables

| Type | Count | Details |
|------|-------|---------|
| JSON Schemas | 1 | `combat-descriptors.schema.json` |
| Schema Definitions | 3 | `CombatCategory`, `HitPool`, `DeathPool` |
| Configuration Updates | 2 | Update combat-hits.json, combat-deaths.json |
| Unit Tests | ~6 | Schema validation tests |

### Schema Structure

```
combat-descriptors.schema.json
|-- $schema: "http://json-schema.org/draft-07/schema#"
|-- title: "Combat Descriptor Configuration"
|-- description: "Schema for combat-related text descriptors"
|-- allOf:
|   +-- $ref: "descriptors.schema.json"
|-- properties:
|   +-- category: string (enum: combat-hits, combat-deaths, combat)
+-- definitions:
    |-- CombatDescriptor (extends Descriptor)
    |   |-- (inherits all from Descriptor)
    |   |-- minDamagePercent: number (minimum: 0, maximum: 1)
    |   |-- maxDamagePercent: number (minimum: 0, maximum: 1)
    |   |-- weaponTypes: array of string (optional, filter by weapon)
    |   |-- damageTypes: array of string (optional, filter by damage)
    |   +-- isCritical: boolean (optional, critical-only text)
    |-- HitPoolTypes (enum)
    |   |-- hit_sword, hit_axe, hit_bow, hit_staff
    |   |-- hit_dagger, hit_fist, hit_mace, hit_spear
    |   |-- critical_hit, miss_descriptions, fumble_descriptions
    +-- DeathPoolTypes (enum)
        |-- death_humanoid, death_undead, death_beast
        |-- death_construct, death_elemental, death_demon
        |-- critical_kill, player_death
        |-- near_death_warning, near_death_critical
```

## v0.14.7c: Environment Descriptor Schema

[v0.14.7c Design Specification](v0.14.7c-design-specification.md)

### Overview

Create a specialized JSON schema for environment-related descriptors that extends the master schema with environment-specific validations for room, object, ambient event, and weather descriptions.

### Scope

**In Scope:**
- `environment-descriptors.schema.json` - Schema for environment text
- Room descriptions (entry text pools)
- Object descriptions (examination text)
- Ambient events (random occurrence text)
- Weather descriptions (atmosphere text)
- Location-based filtering (indoor/outdoor, biome tags)
- Sensory categories (lighting, sounds, smells, temperature)

**Out of Scope:**
- Master descriptor schema (v0.14.7a)
- Combat descriptor schema (v0.14.7b)
- Room generation logic (runtime concern)

### Key Deliverables

| Type | Count | Details |
|------|-------|---------|

| Configuration Updates | 3 | Update environmental.json, weather.json, ambient-events.json |
| Unit Tests | ~6 | Schema validation tests |

### Schema Structure

```
environment-descriptors.schema.json
|-- $schema: "http://json-schema.org/draft-07/schema#"
|-- title: "Environment Descriptor Configuration"
|-- description: "Schema for environment-related text descriptors"
|-- allOf:
|   +-- $ref: "descriptors.schema.json"
|-- properties:
|   +-- category: string (enum: environmental, weather, ambient, objects)
+-- definitions:
    |-- EnvironmentDescriptor (extends Descriptor)
    |   |-- (inherits all from Descriptor)
    |   |-- tags: array of LocationTag
    |   |-- themes: array of ThemeTag
    |   |-- timeOfDay: array of string (optional: dawn, day, dusk, night)
    |   +-- season: array of string (optional: spring, summer, autumn, winter)
    |-- LocationTag (enum)
    |   |-- indoor, outdoor, underground, underwater
    |   |-- cave, dungeon, forest, swamp, volcanic, frozen
    |   |-- ruin, safe, danger, atmospheric
    +-- SensoryCategory (enum)
```

## Design Decisions (Confirmed)

### Schema Standards

| Decision | Value | Notes |
|----------|-------|-------|
| **JSON Schema Version** | Draft-07 | Consistent with v0.14.0-v0.14.6 |
| **Category ID Pattern** | `^[a-z][a-z0-9-]*$` | Lowercase kebab-case |
| **Pool ID Pattern** | `^[a-z][a-z0-9_]*$` | Lowercase with underscores |
| **Descriptor ID Pattern** | `^[a-z][a-z0-9_]*$` | Lowercase with underscores |
| **Schema Location** | `config/schemas/` | Centralized schema storage |

### Descriptor Design

| Decision | Value | Notes |
|----------|-------|-------|
| **Weight Default** | 100 | Equal probability baseline |
| **Weight Range** | 1+ | Positive integers only |
| **Tags Structure** | String array | Flexible context filtering |
| **Themes Structure** | String array | Flexible genre filtering |
| **Variable Syntax** | `{placeholder}` | Simple curly brace pattern |

### Pool Organization

| Decision | Value | Notes |
|----------|-------|-------|
| **Category per File** | Single | One category per JSON file |
| **Pool Nesting** | Single level | Pools contain descriptors directly |
| **Descriptor Minimum** | 1 per pool | At least one descriptor required |

### Combat Descriptor Design

| Decision | Value | Notes |
|----------|-------|-------|
| **Weapon Pool Naming** | `hit_<weapon>` | Consistent prefix pattern |
| **Death Pool Naming** | `death_<type>` | Consistent prefix pattern |
| **Damage Thresholds** | 0-1 range | Normalized percentages |

### Environment Descriptor Design

| Decision | Value | Notes |
|----------|-------|-------|
| **Weather Pool Naming** | `weather_<type>_<location>` | Type + indoor/outdoor |
| **Ambient Pool Naming** | `<sense>_<biome>` | Sensory type + optional biome |
| **Location Tags** | Flexible | No fixed enum, string array |
