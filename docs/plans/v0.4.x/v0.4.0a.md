# v0.4.0a: The Legend (Saga Backend)

## Overview

This sub-phase implements the foundational backend for the **Saga System** (Progression). It focuses on tracking **Legend** (Experience Points), managing **Level** thresholds, and awarding **Progression Points** (PP) upon leveling up.

**Parent Plan:** [v0.4.0](./v0.4.0.md)
**Status:** Planned

---

## 1. Core Architecture

The `SagaService` is the authority on character growth. It listens for gameplay events (or is called directly), accumulates Legend, and triggers Level Up events when thresholds are met.

### 1.1 Data Schema Updates

**Entity: `Character`**
| Field | Type | Change | Description |
|-------|------|--------|-------------|
| `Legend` | `int` | **Rename** | Renamed from `ExperiencePoints`. Total accumulated XP. |
| `Level` | `int` | **Modify** | Max level cap handled by service logic (Soft Cap: 10). |
| `ProgressionPoints` | `int` | **New** | Currency for buying Attribute upgrades. |

### 1.2 Service Architecture

- **`ISagaService`**: Core interface for interaction.
- **`SagaService`**: Engine-layer implementation.
- **`MilestoneDefinition`**: Internal struct/record defining requirements per level.
- **`LevelUpEvent`**: Domain event fired when a character levels up.

### 2.2 Saga Service

**File:** `RuneAndRust.Engine/Services/SagaService.cs`

**Milestone Table (Draft):**
| Level | Required Legend (Total) | PP Award |
|-------|-------------------------|----------|
| 1 | 0 | 0 |
| 2 | 1000 | 1 |
| 3 | 2500 | 1 |
| 4 | 4500 | 2 |
| 5 | 7000 | 2 |
| ... | ... | ... |

**Logic:**
1.  **AddLegend(amount):**
    -   `Character.Legend += amount`
    -   Log `LogInformation`.
    -   Call `CheckMilestones(Character)`.
2.  **CheckMilestones:**
    -   Get `NextLevel = CurrentLevel + 1`.
    -   If `Legend >= Milestones[NextLevel].Required`:
        -   Perform Level Up.
        -   Recursively check next level (for massive XP drops).
3.  **PerformLevelUp:**
    -   `Level++`.
    -   `ProgressionPoints += Milestones[Level].Awards`.
    -   `CurrentHP = MaxHP` (Heal on level up?). *Decision: Yes, full heal.*
    -   `CurrentStamina = MaxStamina`.
    -   Publish `LevelUpEvent`.
    -   Log `LogInformation`.
