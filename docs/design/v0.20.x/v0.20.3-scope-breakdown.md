## Executive Summary

v0.20.3 implements the **Jötun-Reader** specialization — the knowledge-focused ability tree for the Adept archetype. The Jötun-Reader deciphers the legacy of the giants who built the World-Machine, combining ancient linguistic mastery with technical expertise to unlock secrets hidden in ruins, machinery, and forgotten texts.

> _"The giants left more than ruins. They left instructions."_

The work is divided into **3 sub-phases**:

| Phase    | Name                         | Focus                                                                  | Est. Tests |
| -------- | ---------------------------- | ---------------------------------------------------------------------- | ---------- |
| v0.20.3a | Framework & Tier 1 Abilities | Lore Insight resource, Deep Scan, Pattern Recognition, Ancient Tongues | ~10        |
| v0.20.3b | Tier 2 Abilities             | Technical Memory, Exploit Weakness, Data Recovery                      | ~8         |
| v0.20.3c | Tier 3 Abilities & Capstone  | Lore Keeper, Ancient Knowledge, Voice of the Giants                    | ~7         |

### Thematic Elements

The Jötun-Reader carries the torch of understanding in a world of forgotten knowledge. Where others see incomprehensible machinery and cryptic inscriptions, the Jötun-Reader sees stories, instructions, and warnings left by the giants who built Aethelgard's World-Machine infrastructure. Their knowledge bridges the gap between the ancient builders and modern survivors, revealing vulnerabilities in enemies and activating technology thought long dormant.

**Design Principles:**

1. **Discovery Rewards**: Lore Insight generates from exploration and examination
2. **Knowledge is Power**: Abilities reveal hidden information about enemies and objects
3. **Technical Mastery**: Excels at interacting with Jötun technology and terminals
4. **Perception Synergy**: Integrates deeply with v0.15.6 Perception system
5. **Coherent Path**: No Corruption risk — working within established academic traditions

### Tier 1 Ability Specifications

#### Deep Scan (Active)

| Property              | Value                                                   |
| --------------------- | ------------------------------------------------------- |
| **ID**                | `deep-scan`                                             |
| **Type**              | Active                                                  |
| **AP Cost**           | 2 AP                                                    |
| **Lore Insight Cost** | 0 (generates insight instead)                           |
| **Effect**            | +2d10 to examine machinery/terminals                    |
| **Restrictions**      | Target must be machinery, terminal, or Jötun technology |

**Mechanics:**

- Add 2d10 to Perception/Investigation check when examining machinery
- Applies to Jötun terminals, machinery, and technological artifacts
- On success, generates 1 Lore Insight
- On critical success (natural 20 or exceed DC by 10+), generates 2 Lore Insight
- Reveals additional hidden information layers
- Can be used once per target per scene

**Integration with Perception System:**

- Adds to Layer checks from v0.15.6
- Bonus applies after base roll
- Stacks with other examination bonuses

#### Pattern Recognition (Passive)

| Property         | Value                                     |
| ---------------- | ----------------------------------------- |
| **ID**           | `pattern-recognition`                     |
| **Type**         | Passive                                   |
| **Effect**       | Auto-succeed Layer 2 for Jötun technology |
| **Restrictions** | None                                      |

**Mechanics:**

- When examining Jötun technology, Layer 2 reveals automatically
- Layer 1 is always revealed (baseline)
- Layer 2 is revealed without requiring a check
- Layer 3+ still requires normal checks
- Generates 1 Lore Insight on first examination of each unique object
- Works passively without requiring ability activation

**Layer System Integration:**
| Layer | Normal Requirement | With Pattern Recognition |
|-------|-------------------|--------------------------|
| Layer 1 | Automatic | Automatic |
| Layer 2 | DC 12 check | Automatic |
| Layer 3 | DC 16 check | DC 16 check |
| Layer 4 | DC 20 check | DC 20 check |

#### Ancient Tongues (Passive)

| Property         | Value                         |
| ---------------- | ----------------------------- |
| **ID**           | `ancient-tongues`             |
| **Type**         | Passive                       |
| **Effect**       | Read Jötun and Dvergr scripts |
| **Restrictions** | None                          |

**Mechanics:**

- Can read and comprehend Jötun script (giant language)
- Can read and comprehend Dvergr script (dwarf language)
- Includes technical notation and runic variations
- Translation is automatic (no check required)
- Generates 1 Lore Insight when reading a new significant text
- Does not grant ability to speak these languages
- Written communication possible in these scripts

**Script Types Unlocked:**
| Script | Origin | Usage |
|--------|--------|-------|
| Jötun Formal | Giants | Official records, laws |
| Jötun Technical | Giants | Machine instructions |
| Dvergr Standard | Dwarves | General writing |
| Dvergr Runic | Dwarves | Inscriptions, wards |

### Tier 2 Ability Specifications

#### Technical Memory (Active)

| Property              | Value                                       |
| --------------------- | ------------------------------------------- |
| **ID**                | `technical-memory`                          |
| **Type**              | Active                                      |
| **AP Cost**           | 1 AP                                        |
| **Lore Insight Cost** | 2                                           |
| **Effect**            | Recall solution to previously solved puzzle |
| **PP Cost**           | 4 PP to unlock                              |
| **Prerequisite**      | 8 PP invested in Jötun-Reader tree          |

**Mechanics:**

- When encountering a puzzle similar to one previously solved, recall the solution
- Requires that the original puzzle was solved (recorded in DiscoveryLog)
- Similar puzzles include same mechanism type or puzzle family
- Provides exact solution or significant hint (reduces DC by 5)
- Can be used once per puzzle
- Does not work on completely unique puzzles

**Puzzle Categories:**
| Category | Examples |
|----------|----------|
| Mechanical | Gear alignments, pressure plates |
| Electrical | Power routing, circuit completion |
| Sequential | Lock combinations, symbol sequences |
| Environmental | Lever puzzles, bridge mechanisms |

#### Exploit Weakness (Active)

| Property              | Value                              |
| --------------------- | ---------------------------------- |
| **ID**                | `exploit-weakness`                 |
| **Type**              | Active                             |
| **AP Cost**           | 2 AP                               |
| **Lore Insight Cost** | 3                                  |
| **Duration**          | Until end of combat                |
| **Effect**            | Identify enemy vulnerabilities     |
| **PP Cost**           | 4 PP to unlock                     |
| **Prerequisite**      | 8 PP invested in Jötun-Reader tree |

**Mechanics:**

- Analyze target enemy to reveal vulnerabilities
- Reveals damage type vulnerabilities (Fire, Cold, etc.)
- Reveals weak points (+2 to hit specific locations)
- Reveals resistances and immunities
- All allies can see the revealed information
- Attacks exploiting weakness deal +1d6 damage
- Bonus damage only applies to attacks targeting revealed weaknesses

**Information Revealed:**
| Category | Details |
|----------|---------|
| Vulnerabilities | Damage types that deal extra damage |
| Resistances | Damage types that deal reduced damage |
| Immunities | Damage types that deal no damage |
| Weak Points | Body locations with +2 to hit |
| Behavioral | Attack patterns, special ability triggers |

#### Data Recovery (Active)

| Property              | Value                                      |
| --------------------- | ------------------------------------------ |
| **ID**                | `data-recovery`                            |
| **Type**              | Active                                     |
| **AP Cost**           | 3 AP                                       |
| **Lore Insight Cost** | 2                                          |
| **Effect**            | Extract information from damaged terminals |
| **PP Cost**           | 4 PP to unlock                             |
| **Prerequisite**      | 8 PP invested in Jötun-Reader tree         |

**Mechanics:**

- Attempt to recover data from damaged, corrupted, or locked terminals
- Bypass security locks with Lore Insight instead of hacking
- Recover partial data from destroyed storage
- Reconstruct fragmented records
- Success provides 1d4 data fragments
- Critical success provides complete recovery + 1 Lore Insight
- Cannot recover data that has been physically destroyed

**Recovery Difficulty:**
| Terminal State | Base DC | Modifier |
|----------------|---------|----------|
| Locked | 12 | +0 |
| Damaged | 14 | +2 |
| Corrupted | 16 | +4 |
| Fragmented | 18 | +6 |
| Nearly Destroyed | 20 | +8 |

### Tier 3 & Capstone Ability Specifications

#### Lore Keeper (Passive)

| Property         | Value                                       |
| ---------------- | ------------------------------------------- |
| **ID**           | `lore-keeper`                               |

| **Effect**       | Auto-succeed Layer 3 for historical objects |
| **PP Cost**      | 5 PP to unlock                              |
| **Prerequisite** | 16 PP invested in Jötun-Reader tree         |

**Mechanics:**

- When examining historical objects, Layer 3 reveals automatically
- Applies to artifacts, relics, and objects of historical significance
- Layer 1 and Layer 2 are always revealed (baseline + Pattern Recognition)
- Layer 3 is revealed without requiring a check
- Layer 4+ still requires normal checks
- Generates 2 Lore Insight on first examination of each unique historical object

#### Ancient Knowledge (Passive)

| Property         | Value                                   |
| ---------------- | --------------------------------------- |
| **ID**           | `ancient-knowledge`                     |
| **Type**         | Passive                                 |
| **Effect**       | Reveal extra lore on expert examination |
| **PP Cost**      | 5 PP to unlock                          |
| **Prerequisite** | 16 PP invested in Jötun-Reader tree     |

**Mechanics:**

- When achieving Expert or Master success on examination, reveal bonus lore
- Bonus lore includes historical context, related locations, and connections
- Provides quest hooks and additional adventure leads
- Generates 1 additional Lore Insight per bonus lore revealed
- Bonus lore is unique content not available to other characters
- Applies to all examination types, not just historical

**Bonus Lore Content:**
| Success Level | Base Layers | Bonus Lore |
|---------------|-------------|------------|
| Partial | 1 | None |
| Success | 1-2 | None |
| Expert | 1-3 | +1 lore entry |
| Master | 1-4 | +2 lore entries |

**Lore Entry Types:**

- Historical context (who made it, why)
- Related locations (where to find more)
- Connected figures (people involved)
- Hidden purposes (secret functions)
- Warning signs (dangers associated)

#### Voice of the Giants (Ultimate/Capstone)

| Property              | Value                               |
| --------------------- | ----------------------------------- |
| **ID**                | `voice-of-the-giants`               |
| **Type**              | Ultimate                            |
| **AP Cost**           | 5 AP                                |
| **Lore Insight Cost** | 8                                   |
| **Range**             | Touch                               |
| **Effect**            | Activate dormant Jötun technology   |
| **Cooldown**          | Once per long rest                  |
| **PP Cost**           | 6 PP to unlock                      |
| **Prerequisite**      | 24 PP invested in Jötun-Reader tree |

**Mechanics:**

- Speak the activation phrases of the giants to awaken dormant technology
- Can activate Jötun machinery that has been offline for centuries
- Activation effect depends on the technology type
- Some technology may be dangerous or have unintended effects
- Activated technology remains active for 1 hour or until deactivated
- Cannot activate destroyed or irreparably damaged technology
- GM determines specific effects based on technology

**Activatable Technology Types:**
| Type | Effect When Activated |
|------|----------------------|
| Power Node | Restores power to connected systems |
| Defense Grid | Activates automated defenses |
| Transport System | Enables fast travel within complex |
| Medical Bay | Provides healing (4d6 to all allies in area) |
| Archive Terminal | Unlocks restricted data access |
| Fabricator | Enables advanced crafting |
| Sensor Array | Reveals all hidden creatures/objects in area |
| Environmental Control | Changes temperature, atmosphere, lighting |

**Activation Risks:**

- Some technology may alert enemies
- Defense grids may target all non-Jötun
- Corrupted systems may malfunction
- Power surges may damage surroundings

## Lore Insight Resource

### Resource Specification

```csharp
/// <summary>
/// Special resource for Jötun-Reader specialization.
/// Lore Insight represents accumulated knowledge and is generated through discovery.
/// </summary>
public sealed record LoreInsightResource
{
    /// <summary>Current Lore Insight points (0-10).</summary>
    public int CurrentInsight { get; private set; }

    /// <summary>Maximum Lore Insight (default 10).</summary>
    public int MaxInsight { get; init; } = 10;

    /// <summary>When insight was last generated.</summary>
    public DateTime? LastGeneratedAt { get; private set; }

    /// <summary>
    /// Spends the specified amount of Lore Insight.
    /// </summary>
    /// <returns>True if insight was available and spent.</returns>
    public bool Spend(int amount)
    {
        if (amount <= 0 || CurrentInsight < amount)
            return false;

        CurrentInsight -= amount;
        return true;
    }

    /// <summary>
    /// Generates the specified amount of Lore Insight.
    /// </summary>
    public void Generate(int amount)
    {
        CurrentInsight = Math.Min(CurrentInsight + amount, MaxInsight);
        LastGeneratedAt = DateTime.UtcNow;
    }

    /// <summary>
    /// Generates Lore Insight from a discovery activity.
    /// Amount varies by discovery type and significance.
    /// </summary>
    public void GenerateFromDiscovery(DiscoveryType type, bool isSignificant = false)
    {
        var baseAmount = type switch
        {
            DiscoveryType.JotunMachinery => 1,
            DiscoveryType.DvergrArtifact => 1,
            DiscoveryType.AncientText => 1,
            DiscoveryType.Terminal => 1,
            DiscoveryType.Ruin => 2,
            DiscoveryType.LoreObject => 2,
            _ => 1
        };

        var amount = isSignificant ? baseAmount + 1 : baseAmount;
        Generate(amount);
    }
}
```

### Lore Insight

| Decision              | Value             | Notes                                                           |

| **Max Insight**       | 10                | Higher than combat resources, reflecting knowledge accumulation |

| Decision                     | Value        | Notes                                      |

| **Exploit Weakness Damage**  | +1d6         | Rewards tactical analysis                  |
