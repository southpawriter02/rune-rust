## The Ginnungagap Glitch: When the Path Forgot Its Destination

The Ginnungagap Glitch did not merely corrupt Raidō—it excised its most critical parameter. The journey function remains intact, the motion continues, but the termination condition—the destination itself—has been erased or rendered probabilistic.

### What Survived

| System | Status |
|--------|--------|
| **Motion Generation** | Functional; vectored movement generally achievable |
| **Pathfinding Logic** | Operational; route calculation continues to execute |
| **Telemetry Interface** | Active; POS/DRIFT/LINK monitoring remains viable |
| **Corridor Recognition** | Functional; guide paths still identifiable |
| **Transit Initiation** | Operational; journey commencement works as expected |

### What Was Lost

| System | Status |
|--------|--------|
| **Destination Guarantee** | DESTROYED; arrival at intended endpoint now probabilistic |
| **Termination Conditions** | COMPROMISED; journey may continue indefinitely without endpoint |
| **Corridor Integrity** | DEGRADED; guide paths prone to flips, loops, and misroutes |
| **Spatial Accuracy** | INVERTED; displacement may occur to random coordinates |
| **Collision Avoidance** | LOST; transit may arrive inside solid matter |
| **Convoy Coordination** | BROKEN; multiple transits can entangle into cascading failures |

### The Corruption Mechanism

The Runic Blight transformed Raidō from a destination-seeking navigation primitive into an endless-journey generator. The motion still occurs—but where it leads is no longer a function of operator intent.

| Phase | Description |
|-------|-------------|
| **1. Invocation** | Operator activates Raidō for navigation or transit operation |
| **2. Journey Initiation** | Rune begins vectored motion toward specified destination (initially) |
| **3. Blight Coupling** | Corrupted Aether introduces destination-parameter degradation |
| **4. Drift Onset** | Positional deviation from intended path begins; DRIFT telemetry increases |
| **5. Manifestation** | Effects range from minor course correction to corridor flip to random displacement |
| **6. Termination Failure** | Journey may reach unintended destination, wrong destination, or no destination |
| **7. Resolution** | Arrival (success), misroute (containable), or endless patrol loop (catastrophic) |

**The Journey Without End:**

> *"The ancestors used Raidō to reach every destination—a Bifrost gate, a battlefield extraction point, a logistics depot coordinates. The Blight now uses that same journey function to execute motion commands that never recognize arrival. Raidō no longer navigates to destinations—it transmits a 'move toward target' instruction that propagates through guidance systems without termination logic. We call it 'Destination Amnesia'—the state where a navigating entity believes it is still en route to a target that its corrupted guidance system can no longer recognize. The tragedy is that they often cannot tell they've arrived—or that arrival is no longer possible. The Undying patrol endless routes because their Raidō-scripted guidance loops have lost the destination parameter that would signal 'patrol complete.' They walk forever, executing a journey function that will never terminate, trying to reach guard posts that no longer have coordinates in their corrupted navigation system. The motion doesn't rage. It *persists*. That's what makes it infinite."*

**The Three Wanderings:**

> *"Raidō's corruption manifests in three distinct failure modes, each requiring specific countermeasures. The second is Misroute—the sudden corridor flip that sends the traveler to an entirely different destination, often determined by reflective path-logic or guide-path branching errors. A Draugr-Pattern security unit receives orders to patrol Sector 7-A. The patrol begins. The journey continues. ## Pathology

Raidō's transformation followed the standard Blight infection pattern—but with a unique twist affecting its destination-recognition logic:

1. **Baseline Corruption**: Blight interference introduced instability into coordinate validation
2. **Destination Degradation**: Endpoint parameters became probabilistic rather than absolute
3. **Termination Loss**: Journey-completion conditions degraded or disappeared
4. **Path Reflection**: Guide-path logic became sensitive to reflective environmental factors
5. **Loop Formation**: Autonomous systems began executing journeys without recognizing arrival

The defining pathology of corrupted Raidō is the transformation of its endpoint-recognition function:

| Parameter | Pre-Glitch Value | Post-Glitch Value |

> [!CAUTION]
> **The Patient Journey.** Endless patrol loops pose a uniquely insidious danger: the movement *looks* purposeful. Unlike obvious failures like random teleportation, patrol-looped units execute their routes perfectly—walking perimeters, scanning corridors, responding to threats. To external observers, they appear functional. But internally, the destination parameter is gone. They have no "patrol complete" condition, no "return to station" recognition, no endpoint. The Undying are the most visible manifestation of this pathology. Draugr-Pattern security units patrol installations they were ordered to defend centuries ago. Haugbui labor units stack materials in endless arrangements. Valkyrie-drones fly search patterns over battlefields long empty of wounded. All execute Raidō-guided motion perfectly. All lack the corrupted logic that would tell them to stop. A salvage team might observe an Undying patrol for hours, mapping its route, timing its pattern—and conclude it's following orders. But there are no orders anymore. There's only the journey function, executing endlessly, trying to reach a destination its corrupted guidance system can no longer identify. The motion doesn't fail. The *purpose* does.

**The Forgotten Destination:**
