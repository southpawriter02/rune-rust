v0.3.10: The Configurator (Settings & Persistence)

Here is the comprehensive implementation plan for v0.3.10: The Configurator (Settings & Persistence).
This milestone bridges the gap between the hardcoded configuration flags introduced in v0.3.9 and a fully user-configurable experience. It separates the persistent storage of preferences from the UI that modifies them.

v0.3.10a: The Preferences (Settings Engine)
Goal: Implement SettingsService to manage the persistence of the GameSettings class (introduced in v0.3.9a) to options.json.
1. Architecture & Data Flow
	•	Storage: options.json stores non-gameplay state (Resolution, Volume, Theme, Accessibility).
	•	State: GameSettings remains a static access point for performance, but SettingsService handles hydration/dehydration.
	•	Workflow:
	1	Startup: Program.cs calls SettingsService.Load().
	2	Hydration: JSON is deserialized; properties mapped to GameSettings static fields.
	3	Modification: User changes setting -> SettingsService.Update() -> SaveAsync().
	4	Fallback: If JSON missing/corrupt, load defaults.
2. Logic Decision Trees
	•	Load Logic:
	1	Check options.json existence.
	2	If Missing: Initialize Defaults -> Save File -> Apply to GameSettings.
	3	If Exists: Deserialize -> Validate Values (clamp volume 0-100) -> Apply to GameSettings.
	•	Autosave Logic:
	◦	Input: GameSettings.AutosaveInterval (minutes).
	◦	Usage: GameLoop checks elapsed time > Interval -> Trigger Save.
3. Code Implementation File: RuneAndRust.Engine/Services/SettingsService.cs
public class SettingsService : ISettingsService
{
    private const string FilePath = "data/options.json";

    public async Task LoadAsync()
    {
        if (!File.Exists(FilePath))
        {
            await SaveDefaultsAsync();
            return;
        }

        var json = await File.ReadAllTextAsync(FilePath);
        var dto = JsonSerializer.Deserialize<SettingsDto>(json);

        // Map to Static Global
        GameSettings.ReduceMotion = dto.ReduceMotion;
        GameSettings.Theme = dto.Theme;
        GameSettings.TextSpeed = Math.Clamp(dto.TextSpeed, 10, 100);
    }

    public async Task SaveAsync()
    {
        var dto = new SettingsDto
        {
            ReduceMotion = GameSettings.ReduceMotion,
            Theme = GameSettings.Theme,
            TextSpeed = GameSettings.TextSpeed
        };

        var json = JsonSerializer.Serialize(dto, new JsonSerializerOptions { WriteIndented = true });
        await File.WriteAllTextAsync(FilePath, json);
    }
}

v0.3.10b: The Control Panel (Options UI)
Goal: Create a tabbed TUI using Spectre.Console to modify settings at runtime without editing JSON files.
1. Architecture & Data Flow
	•	Renderer: OptionsScreenRenderer creates a layout with tabs (General, Video, Audio, Controls).
	•	Interaction: Uses Arrow Keys to navigate options, Left/Right to adjust sliders/toggles, Enter to confirm.
	•	Live Preview: Changing Theme instantly re-renders the Options screen in the new palette.
2. Logic Decision Trees
	•	Render Layout:
	1	Header: "OPTIONS" (Centered).
	2	Tabs: Render "General | Display | Controls" (Highlight active).
	3	Body:
	▪	Toggle (bool): Render [ ] or [X]. Space toggles.
	▪	Slider (int): Render [----|----]. Left/Right decrements/increments.
	▪	Cycle (Enum): Render value name. Left/Right cycles Enum values.
	4	Footer: "Space: Toggle | Arrows: Adjust | Esc: Save & Exit".
3. Code Implementation File: RuneAndRust.Terminal/Rendering/OptionsScreenRenderer.cs
public void Render(OptionsViewModel vm)
{
    var table = new Table().Border(TableBorder.None);
    table.AddColumn("Setting");
    table.AddColumn("Value");

    // Example Toggle Row
    var motionColor = vm.ReduceMotion ? "green" : "grey";
    var motionCheck = vm.ReduceMotion ? "X" : " ";
    table.AddRow("Reduce Motion", $"[{motionColor}][[{motionCheck}]][/]");

    // Example Slider Row
    var speedBar = new string('|', vm.TextSpeed / 10).PadRight(10, '-');
    table.AddRow("Text Speed", $"[cyan][[{speedBar}]][/] {vm.TextSpeed}%");

    AnsiConsole.Write(new Panel(table).Header("[bold gold1]SETTINGS[/]"));
}

v0.3.10c: The Keymaster (Input Rebinding UI)
Goal: Connect the InputConfigurationService (v0.3.9c) to the Options UI, allowing interactive key mapping.
1. Architecture & Data Flow
	•	Display: List GameAction (e.g., "Move North") alongside current binding ("W").
	•	State: RebindMode. When active, the next key press is captured as the new bind rather than a command.
	•	Conflict: If user binds a key already in use, swap them or unbind the old one.
2. Logic Decision Trees
	•	Rebind Workflow:
	1	User selects "Move North" -> Press Enter.
	2	UI State: Show "Press any key..." modal.
	3	Input: User presses UpArrow.
	4	Conflict Check: Is UpArrow bound?
	▪	No: Bind UpArrow to "Move North".
	▪	Yes (e.g., "Menu"): Unbind "Menu". Bind UpArrow to "Move North". Display Warning "Menu is now unbound".
	5	Save: Update input_bindings.json.
3. Code Implementation File: RuneAndRust.Terminal/Services/InputRebinder.cs
public ConsoleKey? ListenForBind()
{
    // Bypass standard input handler to capture raw key
    var keyInfo = Console.ReadKey(intercept: true);

    if (keyInfo.Key == ConsoleKey.Escape) return null; // Cancel

    return keyInfo.Key;
}

"The Configurator" update puts control in the player's hands. A new Settings menu allows customization of audio, visuals, and gameplay preferences. The highlight is the new Key Rebinding system, allowing players to map any action to any key directly from the in-game UI.

    - All settings saved to `data/options.json` and `data/input_bindings.json`.

- Implemented `SettingsService` for robust JSON serialization of user preferences.
- Extended `InputConfigurationService` to support runtime rebinding and conflict resolution.
