# v0.3.22a: The Filter (Log Customization)

**Status:** Planned
**Parent Plan:** [v0.3.22](./v0.3.22.md)
**Dependencies:** None

## Overview
As the combat system grows in complexity (traits, conditions, status effects), the combat log has become a "wall of text." **v0.3.22a** introduces a structured logging system that allows players to filter information based on their preferences (e.g., hiding dice rolls, muting flavor text). This transforms the log from a raw string dump into a queryable data source.

## Goals
1.  **Structured Data:** Replace `List<string>` with `List<CombatLogEntry>` to enable categorization.
2.  **User Control:** Allow players to toggle visibility of Mechanics, Flavor, and System messages via `GameSettings`.
3.  **Visual Clarity:** Use semantic coloring and tagging for easier parsing at a glance.

---

## 1. Architecture & Data Flow

### The Logging Pipeline
1.  **Event Generation:** `CombatService` generates raw events (e.g., Attack, Heal, Effect).
2.  **Formatting:** `CombatLogFormatter` (updated) converts these events into `CombatLogEntry` objects instead of strings.
3.  **Storage:** `CombatViewModel` carries a list of `CombatLogEntry` objects.
4.  **Filtering (Render Time):** `CombatScreenRenderer` checks `GameSettings` to decide which entries to display.

### Data Structures

#### `CombatLogEntry` (New Record)
Located in `RuneAndRust.Core/Models/Combat/`.
```csharp
public record CombatLogEntry(
    string Markup,              // The display text (Spectre.Console markup)
    LogCategory Category,       // Mechanics, Flavor, System, Status
    int TurnNumber,             // Which turn this happened
    bool IsImportant = false    // If true, bypasses filters (e.g., Crits, Deaths)
);
```

#### `LogCategory` (Enum)
```csharp
public enum LogCategory {
    Mechanics,  // Dice rolls, specific damage numbers
    Flavor,     // Barks, atmospheric descriptions
    System,     // Round start, Turn change
    Status,     // Buffs/Debuffs application
    Dialogue    // NPC speech
}
```

#### `GameSettings` (Update)
Extensions to `RuneAndRust.Core/Models/SettingsDto.cs` or `GameSettings.cs`.
- `ShowCombatMechanics` (bool, default true): Show/Hide dice math.
- `ShowFlavorText` (bool, default true): Show/Hide barks/descriptions.
- `ShowSystemMessages` (bool, default true): Show/Hide "Round X begins".

*   **Update `GameSettings`:** Add the boolean flags.

### Step 2: Engine Refactoring
*   **Refactor `CombatService`:**
    -   Change `Queue<string> _combatLog` to `Queue<CombatLogEntry> _combatLog`.
    -   Update `LogCombatEvent` to accept `CombatLogEntry` or overload it to create one.
    -   Update `MaxLogHistory` logic if needed (maybe keep more history since filtering might hide some).
*   **Update `CombatLogFormatter`:**
    -   Ensure it produces `CombatLogEntry` objects.
    -   Tag existing messages (e.g., "Hit for X damage" -> `Mechanics`, "Goblin screeches" -> `Flavor`).

### Step 3: Terminal Rendering
*   **Update `CombatViewModel`:** Change `CombatLog` property type.
*   **Update `CombatScreenRenderer`:**
    -   Inject `GameSettings` (or access via a service).
    -   Implement filtering logic in `RenderCombatLog`.
    -   *Logic:*
        ```csharp
        foreach (var entry in log) {
            if (entry.IsImportant) { Render(entry); continue; }
            if (entry.Category == LogCategory.Mechanics && !Settings.ShowCombatMechanics) continue;
            if (entry.Category == LogCategory.Flavor && !Settings.ShowFlavorText) continue;
            Render(entry);
        }
        ```

### Step 4: UI Controls
*   **Options Menu:** Add a new sub-menu or entries in the "Gameplay" tab to toggle these settings.
