## Overview

v0.4.3 introduces the core systems for **The Weaver**, the magic system of *Rune & Rust*. Unlike traditional mana systems, magic in Domain 4 is governed by **Flux**—a local environmental volatility that increases with every spell cast.

This version focuses on the underlying mechanics of Aether manipulation: tracking local Flux levels, defining the `Spell` entity structure, and implementing the risk/reward mechanics of Corruption.

To manage the complexity of introducing a completely new verb system (`Cast`) and environmental state, this version is split into 3 phases:

| Phase | Codename | Focus |
|-------|----------|-------|
| A | The Aether | Flux state tracking (accumulation/dissipation) and `AetherService`. |
| B | The Weave | `Spell` entity definition, `Cast` verb, and targeting logic. |
| C | The Backlash | Risk mechanics, Corruption checks, and "Aether Sickness". |

## Phase A: The Aether (Flux Mechanics)

**Goal:** Implement the environmental state for Magic (Flux) and the service to manage its volatility.

### 1. Architecture & Data Flow

Flux is not a character resource (like Stamina) but a property of the current location/combat encounter. It represents the thinning of reality.

**Components:**
- **FluxState (Core/Models):** A lightweight model tracking current Flux (0-100) and DissipationRate.
- **AetherService (Engine):** Singleton service responsible for modifying Flux and calculating dissipation over turns.
- **CombatService (Engine):** Must be updated to trigger `AetherService.Dissipate()` at the end of rounds.

**Workflow: Flux Accumulation**
1. **Action:** Player casts a spell.
2. **AetherService:** Adds `Spell.FluxCost` to `FluxState.CurrentValue`.
3. **Clamp:** Value is clamped to 100 (Critical Mass).

**Workflow: Flux Dissipation**
1. **Event:** Round End (Combat) or Tick (Exploration).
2. **AetherService:** Reduces `FluxState.CurrentValue` by `FluxState.DissipationRate` (default 5/turn).

### 2. Logic Decision Trees

#### A. Flux Modification
**Input:** `ModificationAmount` (int)

1. **Calculate New Value:** `Current + Modification`
2. **Check Bounds:**
   - If > 100: Set to 100, Trigger `FluxCriticalEvent` (for UI/VFX).
   - If < 0: Set to 0.
3. **Update State:** Save new value.
4. **Log:** Trace level log of change.

### 3. Code Implementation

#### A. FluxState (Model)
**File:** `RuneAndRust.Core/Models/Magic/FluxState.cs`

```csharp
public class FluxState
{
    public int CurrentValue { get; set; } = 0;
    public int DissipationRate { get; set; } = 5;
    public int CriticalThreshold { get; set; } = 50; // Point where risks begin

    public bool IsCritical => CurrentValue >= CriticalThreshold;
}
```

#### B. AetherService (Engine)
**File:** `RuneAndRust.Engine/Services/AetherService.cs`

```csharp
public interface IAetherService
{
    int GetCurrentFlux();
    void ModifyFlux(int amount, string source);
    void DissipateFlux();
}

public class AetherService : IAetherService
{
    private readonly FluxState _state = new();
    private readonly ILogger<AetherService> _logger;

    public AetherService(ILogger<AetherService> logger)
    {
        _logger = logger;
    }

    public void ModifyFlux(int amount, string source)
    {
        int oldVal = _state.CurrentValue;
        _state.CurrentValue = Math.Clamp(_state.CurrentValue + amount, 0, 100);

        _logger.LogDebug(
            "Flux modified by {Source}: {Old} -> {New} ({Change})",
            source, oldVal, _state.CurrentValue, amount > 0 ? $"+{amount}" : amount.ToString());
    }
    // ...
}
```

## Phase B: The Weave (Spell Engine)

**Goal:** Define the `Spell` entity and the `Cast` action pipeline.

### 1. Architecture & Data Flow

Spells are distinct from Items but share some properties (Description, Name). They are not "equipped" but "Memorized" (future v0.4.4). For this phase, we assume all defined spells are available for testing.

**Components:**
- **Spell (Core/Entities):** Definition of a spell (Cost, Range, Damage, Effects).
- **CastCommand (Engine):** Parsed command for `cast [spell]`.
- **MagicService (Engine):** Handles the execution of the spell, utilizing `AetherService`.

**Workflow: Casting a Spell**
1. **Input:** User types `cast spark`.
2. **Validation:** Check if spell exists, check if player has Action Points.
3. **Execution:**
   - Pay AP cost.
   - Calculate Effect (Damage/Heal).
   - **Flux:** Call `AetherService.ModifyFlux(Spell.FluxCost)`.
4. **Feedback:** Log result to Combat Log.

#### B. MagicService (Engine)
**File:** `RuneAndRust.Engine/Services/MagicService.cs`

```csharp
public class MagicService : IMagicService
{
    private readonly IAetherService _aether;
    private readonly ICombatLogService _log;

    public MagicResult CastSpell(Character caster, Spell spell, Character target)
    {
        // 1. Validate Resources (AP)
        // 2. Calculate Outcome (Hit/Miss/Damage)
        // 3. Apply Flux
        _aether.ModifyFlux(spell.FluxCost, $"Cast {spell.Name}");

        // 4. Return Result
        return new MagicResult { Success = true, ... };
    }
}
```

## Phase C: The Backlash (Risk System)

**Goal:** Implement the consequence of high Flux—Corruption and Backlash.

### 1. Architecture & Data Flow

When Flux is high (`> CriticalThreshold`), every cast carries a risk.

**Logic: The Check**
When `CastSpell` is called:
1. Get `CurrentFlux` (before cast cost is added).
2. If `CurrentFlux > CriticalThreshold`:
   - Calculate `RiskChance = (CurrentFlux - CriticalThreshold)`.
   - Roll d100.
   - If `Roll < RiskChance`: **BACKLASH TRIGGERED**.

**Backlash Effects:**
- **Minor:** Small damage to caster.
- **Major:** "Aether Sickness" status effect (Reduces Max AP or Stats).

### 2. Code Implementation

#### A. Backlash Logic (MagicService extension)

```csharp
private void CheckBacklash(Character caster)
{
    int currentFlux = _aether.GetCurrentFlux();
    int threshold = _aether.GetCriticalThreshold();

    if (currentFlux > threshold)
    {
        int risk = currentFlux - threshold;
        int roll = _dice.Roll(1, 100);

        if (roll <= risk)
        {
            ApplyBacklash(caster);
        }
    }
}
```

#### B. Aether Sickness (Status Effect)
**File:** `RuneAndRust.Core/Data/StatusEffects/AetherSickness.json` (or Code definition)
- **Effect:** -2 Will, -1 Wits.
- **Duration:** Until Rest.
