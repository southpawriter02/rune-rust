## Overview

This version introduces the Background Schema (`backgrounds.schema.json`), defining the JSON Schema validation for character background configurations. The schema enables IDE autocomplete and validation for backgrounds divided into two categories: **Professions** (trained occupations) and **Lineages** (cultural/geographical origins).

---

## Added

### Configuration Schemas
- **`config/schemas/backgrounds.schema.json`** - JSON Schema Draft-07 for background definitions
  - Validates `BackgroundDefinition` objects with all required fields
  - Validates `AttributeBonuses` with range constraint (0-2, bonuses only)
  - Enforces category enum: `Profession`, `Lineage`
  - Enforces kebab-case pattern for `id`
  - Validates starter ability fields (id, name, description)
  - Validates `startingItems` array with unique items

### Unit Tests
- **`BackgroundSchemaTests.cs`** - 5 new unit tests
  - `Schema_IsValidJsonSchemaDraft07` - Validates schema structure
  - `ExistingBackgroundsJson_ValidatesAgainstSchema` - Validates existing configuration
  - `Category_InvalidEnumValue_FailsValidation` - Tests category enum
  - `AttributeBonuses_OutOfRange_FailsValidation` - Tests maximum (> 2)
  - `AttributeBonuses_NegativeValue_FailsValidation` - Tests minimum (< 0)

## Technical Details

### Schema Definitions

| Definition | Purpose |
|------------|---------|
| `BackgroundDefinition` | Validates background with category, ability, and items |
| `AttributeBonuses` | Validates 5 optional attributes (range 0-2) |

### Category Enum

| Category | Pattern | Example |
|----------|---------|---------|
| `Profession` | Single +2 bonus | Soldier: might +2 |
| `Lineage` | Two +1 bonuses | Mountainborn: fortitude +1, might +1 |

### Key Differences from Race Schema

| Aspect | Race Schema | Background Schema |
|--------|-------------|-------------------|
| Attribute Range | -3 to +3 | 0 to +2 |
| Attribute Fields | All required | All optional |
| Categories | None | Profession/Lineage enum |
