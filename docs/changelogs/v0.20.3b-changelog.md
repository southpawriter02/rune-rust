# v0.20.3b Changelog — Jötun-Reader Tier 2 Abilities

## Added

### Domain Layer

- **`DamageType` enum** — 10 damage categories (Slashing, Piercing, Blunt, Fire, Cold, Lightning, Poison, Necrotic, Radiant, Psychic) for vulnerability analysis
- **`VulnerabilityType` enum** — Vulnerable, Resistant, Immune classifications
- **`TechnicalMemoryRecord` value object** — puzzle solution storage with keyword-based pattern matching (50% similarity threshold, DC -5 on similar match)
- **`WeaknessAnalysis` value object** — enemy vulnerability/resistance/immunity/weak-point analysis with formatted combat reports
- **`WeakPoint` value object** — body location with configurable hit bonus (default +2)
- **`RecoveredData` value object** — terminal data fragments with completion estimation and formatted display

### Application Layer

- **`JotunReaderTier2AbilityService`** — implements all Tier 2 ability operations:
    - **Technical Memory** (1 AP, 2 Lore Insight): records puzzle solutions and attempts recall; exact match bypasses DC, similar pattern reduces DC by 5
    - **Exploit Weakness** (2 AP, 3 Lore Insight): analyzes enemy for vulnerabilities, grants +1d6 bonus damage when exploiting weakness, +2 hit bonus on weak points
    - **Data Recovery** (3 AP, 2 Lore Insight): recovers 1d4 data fragments from damaged terminals; critical success (DC+10) grants full recovery + bonus Lore Insight
    - PP prerequisite validation (8 PP threshold, 4 PP per ability)

### Configuration

- **`specializations.json`** — added Tier 2 "Applied Knowledge" ability tier with `technical-memory`, `exploit-weakness`, and `data-recovery` abilities

### Tests

- **`JotunReaderTier2AbilityServiceTests`** — 10 unit tests covering all abilities and prerequisites
