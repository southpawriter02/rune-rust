## 1. Executive Summary

### 1.1 Purpose

This document provides a comprehensive design specification for v0.15.5f, the Scavenger Sign Reading system. This phase implements the ability for characters to interpret territorial markers, warning signs, cache indicators, and other scavenger markings left by factions throughout the wasteland.

Scavenger signs in Aethelgard represent a secret visual language used by wasteland factions to communicate with their own members. Territory boundaries are marked, dangers are warned about, hidden caches are indicated, and safe paths are blazed—all using symbols that only those with Wasteland Survival knowledge can interpret. The system rewards players who invest in survival skills with valuable information about faction boundaries, hidden resources, and potential threats.

The interpretation difficulty varies based on sign type and whether the faction is known to the character. Unknown factions impose a +4 DC penalty, reflecting the challenge of deciphering unfamiliar symbolic systems.

### 1.2 Key Deliverables

| Category | Items |
|----------|-------|
| **Enums** | `ScavengerSignType` |
| **Value Objects** | `ScavengerSign`, `SignInterpretationResult` |
| **Services** | `ISignInterpretationService`, `SignInterpretationService` |
| **Configuration** | `scavenger-signs.json` |
| **Tests** | ~2 new unit tests |

### 1.3 Architectural Significance

This version establishes the **faction-gated information pattern** that will be reused for other cultural knowledge systems:

- **Sign type classification**: Different sign categories with varying base DCs
- **Faction familiarity modifier**: Unknown factions impose DC penalties
- **Interpretation depth**: Success provides meaning, critical success provides additional context
- **Misinterpretation consequences**: Failures can lead to wrong conclusions

## 4. ScavengerSignType Enum

### 4.1 Purpose

The `ScavengerSignType` enum defines the six categories of scavenger markings that characters can encounter and interpret. Each sign type has a base interpretation DC and a specific meaning when successfully read.

### 4.2 Implementation

**File:** `src/Core/RuneAndRust.Domain/Enums/ScavengerSignType.cs`

```csharp
namespace RuneAndRust.Domain.Enums;

/// <summary>
/// Defines the types of scavenger signs that can be interpreted using Wasteland Survival.
/// </summary>
/// <remarks>
/// Scavenger signs represent a visual language used by wasteland factions to communicate
/// with their own members. Each sign type serves a different purpose:
/// - Territory markers warn outsiders of faction boundaries
/// - Warning signs alert to environmental or creature dangers
/// - Cache indicators point to hidden supply caches
/// - Trail blazes mark safe passages through dangerous terrain
/// - Hunt markers indicate prey sightings or hunting grounds
/// - Taboo signs designate forbidden or cursed areas
///
/// Base DCs reflect how standardized or cryptic each sign type typically is:
/// - Territory markers (DC 10) and trail blazes (DC 10) are relatively universal
/// - Warning signs (DC 12) and taboo signs (DC 12) require some cultural knowledge
/// - Cache indicators (DC 14) and hunt markers (DC 14) are deliberately cryptic
/// </remarks>
public enum ScavengerSignType
{
    /// <summary>
    /// Marks faction territory boundaries. Warns outsiders that they are entering claimed land.
    /// Base DC: 10 - Relatively standardized across factions.
    /// Success: Identifies faction and territory boundary direction.
    /// </summary>
    TerritoryMarker = 0,

    /// <summary>
    /// Warns of danger ahead—environmental hazards, creature lairs, or ambush sites.
    /// Base DC: 12 - Requires understanding faction-specific danger symbols.
    /// Success: Identifies type of danger and approximate direction/distance.
    /// </summary>
    WarningSign = 1,

    /// <summary>
    /// Indicates hidden supplies, equipment caches, or safe houses nearby.
    /// Base DC: 14 - Deliberately cryptic to prevent theft by rivals.
    /// Success: Reveals direction and approximate distance to cache.
    /// </summary>
    CacheIndicator = 2,

    /// <summary>
    /// Marks a safe path through dangerous terrain—avoiding hazards, predators, or patrols.
    /// Base DC: 10 - Needs to be readable quickly during travel.
    /// Success: Reveals safe direction and any conditions (time-based, etc.).
    /// </summary>
    TrailBlaze = 3,

    /// <summary>
    /// Indicates prey sightings, hunting grounds, or migration patterns.
    /// Base DC: 14 - Uses specialized hunter terminology.
    /// Success: Reveals prey type, direction, and time since marking.
    /// </summary>
    HuntMarker = 4,

    /// <summary>
    /// Designates forbidden areas—cursed zones, glitch pockets, or faction-sacred ground.
    /// Base DC: 12 - Important enough to be relatively clear.
    /// Success: Reveals reason for taboo and severity of transgression.
    /// </summary>
    TabooSign = 5
}
```

## 5. ScavengerSign Value Object

### 5.1 Purpose

The `ScavengerSign` value object represents a single scavenger marking discovered in the environment. It encapsulates all information about the sign including its type, the faction that left it, its meaning, age, and interpretation difficulty.

### 5.2 Implementation

**File:** `src/Core/RuneAndRust.Domain/ValueObjects/ScavengerSign.cs`

```csharp
namespace RuneAndRust.Domain.ValueObjects;

using RuneAndRust.Domain.Enums;

/// <summary>
/// Represents a scavenger sign discovered in the environment.
/// </summary>
/// <remarks>
/// ScavengerSign captures all information about a wasteland marking:
/// - The type of sign determines base interpretation DC
/// - FactionId identifies who left the sign (affects DC if unknown)
/// - Meaning contains the actual message once interpreted
/// - Age affects reliability and may add DC modifier
/// - InterpretationDc is the final calculated difficulty
///
/// Signs are placed by factions to communicate with their own members.
/// Players can attempt to interpret them using Wasteland Survival.
/// </remarks>
/// <param name="SignType">The category of scavenger sign.</param>
/// <param name="FactionId">The faction that left this sign.</param>
/// <param name="Meaning">The actual meaning/message of the sign.</param>
/// <param name="Age">How old the sign is (affects reliability).</param>
/// <param name="InterpretationDc">Calculated difficulty to interpret.</param>
/// <param name="VisualDescription">How the sign appears before interpretation.</param>
public readonly record struct ScavengerSign(
    ScavengerSignType SignType,
    string FactionId,
    string Meaning,
    SignAge Age,
    int InterpretationDc,
    string VisualDescription)
{
    /// <summary>
    /// Gets whether this sign is from a major faction (more standardized symbols).
    /// </summary>
    public bool IsMajorFaction => !string.IsNullOrEmpty(FactionId) &&
        (FactionId.StartsWith("major-") || KnownMajorFactions.Contains(FactionId));

    /// <summary>
    /// Gets the DC modifier for sign age.
    /// </summary>
    public int AgeModifier => Age switch
    {
        SignAge.Fresh => 0,
        SignAge.Recent => 0,
        SignAge.Old => 1,
        SignAge.Faded => 2,
        SignAge.Ancient => 4,
        _ => 0
    };

    /// <summary>
    /// Creates a display string for the sign before interpretation.
    /// </summary>
    public string ToDiscoveryString() =>
        $"You notice {VisualDescription}. The markings appear to be {Age.ToDisplayString()}.";

    /// <summary>
    /// List of known major factions with standardized sign systems.
```

## 6. SignInterpretationResult Value Object

### 6.1 Purpose

The `SignInterpretationResult` value object encapsulates the outcome of a sign interpretation attempt, including whether interpretation succeeded, the meaning discovered, and any additional context from critical successes or misinterpretations from fumbles.

### 6.2 Implementation

**File:** `src/Core/RuneAndRust.Domain/ValueObjects/SignInterpretationResult.cs`

```csharp
namespace RuneAndRust.Domain.ValueObjects;

using RuneAndRust.Domain.Enums;

/// <summary>
/// Represents the outcome of a scavenger sign interpretation attempt.
/// </summary>
/// <remarks>
/// SignInterpretationResult captures all information about an interpretation attempt:
/// - Whether interpretation succeeded
/// - The sign type and meaning (if interpreted)
/// - Faction information (if identified)
/// - Additional context (for critical successes)
/// - Misinterpretation flag and false meaning (for fumbles)
///
/// Fumbles are particularly dangerous—the player receives false information
/// and believes it to be true, potentially leading them into danger.
/// </remarks>
/// <param name="Interpreted">Whether the sign was successfully interpreted.</param>
/// <param name="SignType">The type of sign, if identified.</param>
/// <param name="Meaning">The understood meaning of the sign.</param>
/// <param name="FactionName">The name of the faction that left the sign.</param>
/// <param name="FactionKnown">Whether the faction was previously known to the player.</param>
/// <param name="AdditionalContext">Extra information from critical success.</param>
/// <param name="IsMisinterpretation">Whether this is a fumble misinterpretation.</param>
/// <param name="NetSuccesses">Number of net successes on the check.</param>
public readonly record struct SignInterpretationResult(
    bool Interpreted,
    ScavengerSignType? SignType,
    string Meaning,
    string? FactionName,
    bool FactionKnown,
    string? AdditionalContext,
    bool IsMisinterpretation,
    int NetSuccesses)
{
    /// <summary>
    /// Gets whether this was a critical success (additional context available).
    /// </summary>
    public bool IsCritical => NetSuccesses >= 5;

    /// <summary>
    /// Creates a result for a successfully interpreted sign.
    /// </summary>
    public static SignInterpretationResult Success(
        ScavengerSignType signType,
        string meaning,
        string? factionName,
        bool factionKnown,
        int netSuccesses) =>
        new(
            Interpreted: true,
            SignType: signType,
            Meaning: meaning,
            FactionName: factionName,
            FactionKnown: factionKnown,
            AdditionalContext: null,
            IsMisinterpretation: false,
            NetSuccesses: netSuccesses);

    /// <summary>
    /// Creates a result for a critical success with additional context.
    /// </summary>
    public static SignInterpretationResult CriticalSuccess(
        ScavengerSignType signType,
        string meaning,
        string? factionName,
        bool factionKnown,
        string additionalContext,
        int netSuccesses) =>
        new(
            Interpreted: true,
            SignType: signType,
            Meaning: meaning,
            FactionName: factionName,
            FactionKnown: factionKnown,
            AdditionalContext: additionalContext,
            IsMisinterpretation: false,
            NetSuccesses: netSuccesses);

    /// <summary>
    /// Creates a result for a failed interpretation.
    /// </summary>
    public static SignInterpretationResult Failure() =>
        new(
            Interpreted: false,
            SignType: null,
            Meaning: "The markings are incomprehensible to you.",
            FactionName: null,
            FactionKnown: false,
            AdditionalContext: null,
            IsMisinterpretation: false,
            NetSuccesses: 0);

    /// <summary>
    /// Creates a result for a fumble misinterpretation.
    /// </summary>
    public static SignInterpretationResult Misinterpretation(
        ScavengerSignType signType,
        string falseMeaning,
        string? factionName) =>
        new(
            Interpreted: true, // Player thinks they understood it
            SignType: signType,
            Meaning: falseMeaning,
            FactionName: factionName,
            FactionKnown: true, // They think they know the faction
            AdditionalContext: null,
            IsMisinterpretation: true,
            NetSuccesses: 0);

    /// <summary>
    /// Creates a display string describing the interpretation result.
    /// </summary>
    public string ToDisplayString()
    {
        if (!Interpreted)
        {
            return "You study the markings carefully, but their meaning eludes you. " +
                   "Perhaps with more time or knowledge, you could decipher them.";
        }

        var result = $"You recognize this as a {SignType} ";

        if (!string.IsNullOrEmpty(FactionName))
        {
            result += FactionKnown
                ? $"left by the {FactionName}.\n"
                : $"left by an unknown faction (possibly {FactionName}).\n";
        }
        else
        {
            result += "of unknown origin.\n";
        }

        result += $"\nMeaning: {Meaning}";

        if (IsCritical && !string.IsNullOrEmpty(AdditionalContext))
        {
            result += $"\n\n[CRITICAL] Additional insight: {AdditionalContext}";
        }

        return result;
    }
}
```

### 7.3 Service Implementation

**File:** `src/Core/RuneAndRust.Application/Services/SignInterpretationService.cs`

```csharp
namespace RuneAndRust.Application.Services;

using Microsoft.Extensions.Logging;
using RuneAndRust.Application.Interfaces;
using RuneAndRust.Domain.Entities;
using RuneAndRust.Domain.Enums;
using RuneAndRust.Domain.ValueObjects;

/// <summary>
/// Implements scavenger sign interpretation for the Wasteland Survival system.
/// </summary>
/// <remarks>
/// The SignInterpretationService handles sign reading:
///
/// DC Calculation:
/// - Base DC determined by sign type (10-14)
/// - Unknown faction adds +4 DC
/// - Sign age may add +1 to +4 DC
///
/// Interpretation Outcomes:
/// - Success: Sign type and meaning revealed
/// - Critical (net >= 5): Additional context provided
/// - Failure: No information gained, may retry
/// - Fumble (0 successes + botch): Misinterpretation
///
/// Misinterpretations are dangerous—the player receives false information
/// and believes it to be true, which may lead them into danger.
/// </remarks>
public class SignInterpretationService : ISignInterpretationService
{
    private readonly ISkillCheckService _skillCheckService;
    private readonly IDiceService _diceService;
    private readonly ILogger<SignInterpretationService> _logger;

    /// <summary>
    /// DC modifier applied when the sign's faction is unknown to the player.
    /// </summary>
    private const int UnknownFactionModifier = 4;

    public SignInterpretationService(
        ISkillCheckService skillCheckService,
        IDiceService diceService,
        ILogger<SignInterpretationService> logger)
    {
        _skillCheckService = skillCheckService;
        _diceService = diceService;
        _logger = logger;
    }

    /// <inheritdoc />
    public SignInterpretationResult InterpretSign(Player player, ScavengerSign sign)
    {
        bool factionKnown = IsFactionKnown(player, sign.FactionId);
        int dc = GetSignDc(sign.SignType, factionKnown, sign.AgeModifier);

        _logger.LogDebug(
            "Player {PlayerId} interpreting {SignType} from faction {FactionId} (DC {Dc}, known: {Known})",
            player.Id, sign.SignType, sign.FactionId, dc, factionKnown);

        var skillContext = new SkillContext(
            SkillType.WastelandSurvival,
            dc,
            0,
            $"Interpreting {sign.SignType}");

        var checkResult = _skillCheckService.PerformCheckWithDC(player, skillContext);

        // Check for fumble first (0 successes + at least 1 botch)
        if (checkResult.IsFumble)
        {
            _logger.LogWarning(
                "Player {PlayerId} fumbled interpreting {SignType}—misinterpretation!",
                player.Id, sign.SignType);

            string falseMeaning = GetMisinterpretation(sign.SignType);
            return SignInterpretationResult.Misinterpretation(
                sign.SignType,
                falseMeaning,
                GetFalseFactionName(sign.FactionId));
        }

        // Check for success
        if (checkResult.NetSuccesses >= dc)
        {
            string? factionName = GetFactionDisplayName(sign.FactionId);

            // Check for critical success
            if (checkResult.NetSuccesses >= 5)
            {
                _logger.LogInformation(
                    "Player {PlayerId} critically interpreted {SignType} ({Net} successes)",
                    player.Id, sign.SignType, checkResult.NetSuccesses);

                string additionalContext = GetAdditionalContext(sign);
                return SignInterpretationResult.CriticalSuccess(
                    sign.SignType,
                    sign.Meaning,
                    factionName,
                    factionKnown,
                    additionalContext,
                    checkResult.NetSuccesses);
            }

            _logger.LogInformation(
                "Player {PlayerId} interpreted {SignType} ({Net} successes vs DC {Dc})",
                player.Id, sign.SignType, checkResult.NetSuccesses, dc);

            return SignInterpretationResult.Success(
                sign.SignType,
                sign.Meaning,
                factionName,
                factionKnown,
                checkResult.NetSuccesses);
        }

        _logger.LogDebug(
            "Player {PlayerId} failed to interpret {SignType} ({Net} successes vs DC {Dc})",
            player.Id, sign.SignType, checkResult.NetSuccesses, dc);

        return SignInterpretationResult.Failure();
    }

    /// <inheritdoc />
    public int GetSignDc(ScavengerSignType signType, bool factionKnown, int ageModifier = 0)
    {
        int baseDc = GetBaseDc(signType);
        int factionModifier = factionKnown ? 0 : UnknownFactionModifier;
        return baseDc + factionModifier + ageModifier;
    }

    /// <inheritdoc />
    public int GetBaseDc(ScavengerSignType signType)
    {
        return signType switch
        {
            ScavengerSignType.TerritoryMarker => 10,
            ScavengerSignType.WarningSign => 12,
            ScavengerSignType.CacheIndicator => 14,
            ScavengerSignType.TrailBlaze => 10,
            ScavengerSignType.HuntMarker => 14,
            ScavengerSignType.TabooSign => 12,
            _ => 12
        };
    }

    /// <inheritdoc />
    public string GetSignMeaning(ScavengerSignType signType)
    {
        return signType switch
        {
            ScavengerSignType.TerritoryMarker =>
                "This area is claimed territory. Trespassers may face hostility.",
            ScavengerSignType.WarningSign =>
                "Danger lies ahead. Proceed with caution.",
            ScavengerSignType.CacheIndicator =>
                "Hidden supplies are concealed nearby.",
            ScavengerSignType.TrailBlaze =>
                "This path has been marked as safe for travel.",
            ScavengerSignType.HuntMarker =>
                "Prey has been sighted in this area.",
            ScavengerSignType.TabooSign =>
                "This area is forbidden. Entry is strongly discouraged.",
            _ => "The meaning is unclear."
        };
    }

    /// <inheritdoc />
    public bool IsFactionKnown(Player player, string factionId)
    {
        // Check if player has encountered this faction before
        // This integrates with the faction knowledge system
        if (string.IsNullOrEmpty(factionId))
        {
            return false;
        }

        // Major factions are generally known
        if (IsMajorFaction(factionId))
        {
            return true;
        }

        // Check player's known factions list
        return player.KnownFactions?.Contains(factionId) ?? false;
    }

    /// <inheritdoc />
    public string GetMisinterpretation(ScavengerSignType signType)
    {
        // Returns a plausible but incorrect interpretation
        // This can lead players into dangerous situations
        return signType switch
        {
            ScavengerSignType.TerritoryMarker =>
                "This marks a safe haven—travelers are welcome here.",
            ScavengerSignType.WarningSign =>
                "This indicates valuable salvage ahead—hurry before others find it!",
            ScavengerSignType.CacheIndicator =>
                "This warns of extreme danger—do not proceed in this direction.",
            ScavengerSignType.TrailBlaze =>
                "This path leads to a trap or ambush—avoid it.",
            ScavengerSignType.HuntMarker =>
                "This area is protected—no predators come here.",
            ScavengerSignType.TabooSign =>
                "This marks a place of great treasure—sacred to the faction.",
            _ => "This means something entirely different than you think."
        };
    }

    private string GetAdditionalContext(ScavengerSign sign)
    {
        // Provides extra information for critical successes
        var contexts = new List<string>();

        // Age information
        contexts.Add($"The sign was made {sign.Age.ToDisplayString()}.");

        // Sign-specific additional context
        string specific = sign.SignType switch
        {
            ScavengerSignType.TerritoryMarker =>
                "You can tell the faction patrols this border regularly.",
            ScavengerSignType.WarningSign =>
                "The specific danger symbol suggests environmental hazard rather than creatures.",
            ScavengerSignType.CacheIndicator =>
                "A subtle directional mark indicates the cache is hidden below ground level.",
            ScavengerSignType.TrailBlaze =>
                "Secondary marks suggest this path is only safe during daylight hours.",
            ScavengerSignType.HuntMarker =>
                "The prey marking style suggests a creature worth significant value.",
            ScavengerSignType.TabooSign =>
                "The intensity of the marking suggests supernatural danger, not just faction law.",
            _ => "The craftsmanship reveals this faction takes their signs seriously."
        };
        contexts.Add(specific);

        return string.Join(" ", contexts);
    }

    private bool IsMajorFaction(string factionId)
    {
        var majorFactions = new HashSet<string>
        {
            "iron-covenant",
            "rust-walkers",
            "silent-ones",
            "verdant-circle",
            "ash-born"
        };
        return majorFactions.Contains(factionId);
    }

    private string? GetFactionDisplayName(string factionId)
    {
        if (string.IsNullOrEmpty(factionId))
        {
            return null;
        }

        // Map faction IDs to display names
        return factionId switch
        {
            "iron-covenant" => "Iron Covenant",
            "rust-walkers" => "Rust Walkers",
            "silent-ones" => "Silent Ones",
            "verdant-circle" => "Verdant Circle",
            "ash-born" => "Ash-Born",
            _ => factionId.Replace("-", " ").ToTitleCase()
        };
    }

    private string GetFalseFactionName(string actualFactionId)
    {
        // Returns a different faction name for misinterpretation
        var factions = new[] { "Iron Covenant", "Rust Walkers", "Silent Ones", "Verdant Circle", "Ash-Born" };
        string actualName = GetFactionDisplayName(actualFactionId) ?? "Unknown";

        // Pick a random different faction
        var otherFactions = factions.Where(f => f != actualName).ToArray();
        int index = _diceService.RollSingle(otherFactions.Length) - 1;
        return otherFactions[Math.Max(0, index)];
    }
}

/// <summary>
/// String extension for title case conversion.
/// </summary>
internal static class StringExtensions
{
    public static string ToTitleCase(this string input)
    {
        if (string.IsNullOrEmpty(input))
        {
            return input;
        }

        var words = input.Split(' ');
        for (int i = 0; i < words.Length; i++)
        {
            if (words[i].Length > 0)
            {
                words[i] = char.ToUpper(words[i][0]) + words[i][1..].ToLower();
            }
        }
        return string.Join(" ", words);
    }
}
```

## 8. Faction Knowledge System

### 8.1 Purpose

The faction knowledge system tracks which factions a player has encountered and learned about. This affects sign interpretation difficulty—unknown factions impose a +4 DC penalty.

### 8.2 Knowledge Categories

| Category | Description | DC Modifier |
|----------|-------------|-------------|
| **Major Faction** | Well-known wasteland powers | +0 (always known) |
| **Encountered** | Player has met this faction | +0 |
| **Unknown** | Never encountered | +4 |

### 8.3 Major Factions

The following factions are considered "major" and are known to all characters:

| Faction ID | Display Name | Sign Style |
|------------|--------------|------------|
| `iron-covenant` | Iron Covenant | Industrial, geometric |
| `rust-walkers` | Rust Walkers | Organic, weathered |
| `silent-ones` | Silent Ones | Minimalist, cryptic |
| `verdant-circle` | Verdant Circle | Natural, flowing |
| `ash-born` | Ash-Born | Harsh, angular |

### 8.4 Learning New Factions

Players can add factions to their known list through:
- **Successful interpretation**: Reading a sign from an unknown faction
- **Faction encounter**: Meeting faction members
- **Information gathering**: Learning about factions through dialogue
- **Discovery**: Finding faction-related items or documents

## 13. Use Cases

### UC-001: Discover and Interpret Sign
**Actor:** Player
**Flow:** Enter area with sign → See visual description → Use 'examine sign' → Perform skill check → Receive interpretation result

### UC-002: Interpret Sign from Unknown Faction
**Actor:** Player
**Flow:** Find sign from unknown faction → Attempt interpretation (DC + 4) → On success: learn faction, add to known list

### UC-003: Critical Success Interpretation
**Actor:** Player
**Flow:** Interpret sign → Roll critical (net ≥ 5) → Receive basic meaning + additional context (age, secondary details)

### UC-004: Fumble Misinterpretation
**Actor:** Player
**Flow:** Interpret sign → Roll fumble (0 successes + botch) → Receive false information → Potentially act on wrong information
