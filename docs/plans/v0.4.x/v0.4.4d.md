## Overview

While v0.4.4a-c implemented the logic for the Mystic's power (Resonance, Galdr, Mastery), **v0.4.4d** focuses on the *presentation*. In a text-based medium, conveying the escalating danger of Aetheric Resonance or the intricate weaving of a Galdr chant requires distinct visual language. This plan covers the TUI components, VFX (screen shake, color pulses), and feedback loops that make magic feel volatile and powerful.

### Narrative alignment
Following Domain 4 constraints, the UI must not simply display numbers. It must represent "The Weave" and "The Resonance" through color, oscillation, and glitched text. A high-resonance state isn't just "Energy: 90/100"; it's a UI that is physically trembling, with text shifting between clear and corrupted. The interface itself becomes a diegetic element—an artifact of the character's perception of reality bending.

---

## Scope

### Included
- **Resonance Gauge**: A dynamic UI element in the sidebar that visualizes personal attunement (0-100) with threshold-based visual states.
- **Galdr Chant Display**: A dedicated panel or overlay during combat that shows the progress of a multi-turn chant.
- **Paradox VFX**: Screen-wide effects (flashing, shaking, color inversion) when Paradox triggers or Resonance overflows.
- **Grimoire UI**: A spellbook interface for managing known spells, viewing mastery, and assigning quick slots.
- **Combat Feedback**: Distinct log messages and grid animations for spell impacts vs physical hits.
- **Glitch Text Engine**: A utility to corrupt text strings deterministically based on Resonance level.

### Excluded
- **New Logic**: No new game mechanics; this is purely presentation layer (Terminal project).
- **Sound**: Audio implementation is reserved for v0.3.19/v1.0.
- **Particle Systems**: Advanced particle physics are out of scope for the TUI; only character-based cellular automata are permitted.

### Event Subscription Model
The UI will listen for `ResonanceChangedEvent`, `GaldrStartedEvent`, `GaldrAdvancedEvent`, and `ParadoxTriggeredEvent` to trigger immediate redraws or VFX sequences, rather than waiting for the next input cycle. The `MagicVfxService` acts as the mediator between Engine events and direct Console manipulation.

## Key Components

### 1. Resonance Widget (`ResonanceDisplayWidget`)
A vertical or horizontal bar that changes character and color based on the Resonance Threshold. It must support both "Compact" (sidebar) and "Detailed" (character sheet) modes.

| Threshold | Color | ASCII Character | Animation Effect | Description |
|-----------|-------|-----------------|------------------|-------------|
| Dim (0-24) | Grey | `.` (Dot) | Static | Inert, barely visible. |
| Steady (25-49) | Cyan | `~` (Wave) | Slow pulse | Smooth, rhythmic flow. |
| Bright (50-74) | Magenta | `≈` (Double Wave) | Fast pulse | Energetic, vibrant. |
| Blazing (75-99) | Orange | `≡` (Triple Bar) | Flicker | Unstable, tearing. |
| Overflow (100) | Red on White | `▓` (Block) | Screen Shake + Glitch text | Critical failure state. |

### 2. Galdr Chant Overlay (`GaldrStatusWidget`)
Displayed during the player's turn if a chant is active.
Format: `[ WEAVING: INFERNO ] [>>>>>-----] (2 Turns Remaining)`
Visuals:
- The progress bar fills with "Runes" (e.g., `ᚠᚢᚦ`) instead of standard blocks.
- If interrupted, the bar "shatters" (turns red and dissolves) before disappearing.
- Positioned: Top-center of the Combat Layout, overlaying the room description if necessary.

### 3. Paradox VFX (`ParadoxEffectRenderer`)
A service that manipulates the `AnsiConsole` directly to create momentary chaos.
- **Minor Paradox**: Text momentarily turns yellow/black.
- **Major Paradox**: The entire screen shifts position (Shake) for 200ms.
- **Catastrophic Paradox**: "Static" overlay (random characters) floods the screen for 0.5s, then clears.
- **Glitch Text**: Randomly replaces characters in a string with 'Zalgo' text or random ASCII symbols based on intensity.

### 4. Grimoire View (`GrimoireRenderer`)
A full-screen TUI (like Inventory) for spell management.
- **Left Panel**: School List (Destruction, Restoration, etc.) with Mastery Level.
- **Center Panel**: Spell List for selected school.
- **Right Panel**: Spell Details (Cost, Effects, Lore) + ASCII Art of the rune.
- **Footer**: Quick Slot assignments (1-5).
- **Navigation**: Arrow keys to navigate, Number keys to assign slots.

## Interaction Flows

### 1. Grimoire Management
1. **Open Grimoire**: Player types `grimoire` or presses `G`.
2. **Browse Schools**: Use Up/Down arrows to select a School (e.g., Destruction).
   - *Visual*: The highlighted school changes color to White. Right panel clears.
3. **Select Spell**: Press Right to enter Spell List, select "Inferno".
   - *Visual*: Center panel gains focus. Spell details appear in Right panel.
4. **View Details**: Press Right to see cost, effects, and lore.
   - *Visual*: Right panel text highlights. Rune ASCII art renders.
5. **Assign Quick Slot**: Press `1` to assign "Inferno" to Slot 1.
   - *Visual*: A small `[1]` badge appears next to "Inferno" in the list. Footer flashes "Assigned to Slot 1".
6. **Exit**: Press `Esc` to return to Exploration/Combat.

### 2. Casting Galdr (The Ritual)
1. **Initiate**: Player types `cast 1` (Inferno).
2. **Visual Feedback**:
   - Log: "You begin weaving the runes of Inferno..."
   - Sidebar: Resonance bar pulses once (Magenta).
   - Overlay: Galdr bar appears `[>----]`.
   - Grid: The caster's character symbol (`@`) starts blinking Cyan.
3. **Next Turn**: Player types `continue` or presses `Wait`.
4. **Visual Feedback**:
   - Log: "The heat intensifies..."
   - Overlay: Galdr bar advances `[>>---]`.
   - VFX: Small spark particles (asterisks `*`) scatter near the player in the grid.
5. **Interruption (Damage)**: Enemy attacks player.
   - Logic: Concentration check fails.
   - Visual Feedback:
     - Overlay: Bar turns Red, shakes, and text changes to "UNRAVELED".
     - Screen: Minor red flash.
     - Log: "The pattern shatters! Energy backlashes!"
6. **Completion (Success)**: Player types `release`.
7. **Visual Feedback**:
   - Screen: Flash Orange (Explosion effect).
   - Log: "INFERNO ERUPTS!"
   - Grid: 3x3 area around target flashes Red.
   - Overlay: Disappears.

### 3. Paradox Trigger (The Glitch)
1. **Trigger**: Player casts spell while Resonance is 90+.
2. **Logic**: Paradox check fails (Major).
3. **Visual Feedback**:
   - **Step 1 (The Warning)**: The TUI borders turn Yellow.
   - **Step 2 (The Shift)**: The entire console viewport shifts right by 2 columns, then left by 2 (Screen Shake).
   - **Step 3 (The Corruption)**: Random characters in the combat log are replaced by `#`, `?`, and `%`.
   - **Step 4 (The Aftermath)**: A lingering "Static" status icon appears.

## Decision Trees

### 1. Render Resonance
```
Get Current Resonance Value
   │
   ├─ < 25 (Dim) ─────> Color: Grey, Char: '.', Effect: None
   ├─ < 50 (Steady) ──> Color: Cyan, Char: '~', Effect: Breathe(2s)
   ├─ < 75 (Bright) ──> Color: Magenta, Char: '≈', Effect: Pulse(1s)
   ├─ < 100 (Blazing) > Color: Orange, Char: '≡', Effect: Flicker(0.2s)
   └─ >= 100 (Overflow) > Color: Red/White, Char: '▓', Effect: Shake + Glitch
```

### 2. Handle Paradox Event
```
Receive ParadoxTriggeredEvent(Severity)
   │
   ├─ Minor ──────────> Log("The air ripples.") + Flash(Yellow)
   │
   ├─ Major ──────────> Log("REALITY BENDS.") + Shake(Intensity: 2) + Flash(Orange)
   │                    └─ Apply Glitch Text to next 3 log lines
   │
   └─ Catastrophic ───> Log("A TEAR IN THE WEAVE!") + Shake(Intensity: 5) + StaticNoise(0.5s)
                        └─ Invert Colors for 1 second
```

### 3. Render Glitch Text
```
Input String: "You cast Fireball"
   │
   ├─ Resonance < 50 ──> Return "You cast Fireball"
   │
   ├─ Resonance < 75 ──> Randomly replace 1 char with similar looking char (e -> ë)
   │                     Result: "You cast Firëball"
   │
   └─ Resonance >= 90 ─> Insert Zalgo chars or garbage
                         Result: "Yøu c@st F#r%b^ll..."
```

## Code Examples

### Resonance Widget Implementation
```csharp
using Spectre.Console;
using Spectre.Console.Rendering;

public class ResonanceDisplayWidget : IRenderable
{
    private readonly int _value;
    private readonly int _max;
    private readonly bool _isVertical;

    public ResonanceDisplayWidget(int value, int max, bool isVertical = false)
    {
        _value = value;
        _max = max;
        _isVertical = isVertical;
    }

    public Measurement Measure(RenderOptions options, int maxWidth)
    {
        return _isVertical ? new Measurement(3, 10) : new Measurement(10, 10);
    }

    public IEnumerable<Segment> Render(RenderOptions options, int maxWidth)
    {
        var percentage = (double)_value / _max;
        var (color, symbol) = GetVisuals(percentage);

        if (_isVertical)
        {
            // Vertical rendering logic for sidebar
            // ...
        }
        else
        {
            var bar = new StringBuilder();
            bar.Append("[");
            for (int i = 0; i < 8; i++)
            {
                if (i < percentage * 8)
                    bar.Append($"[{color}]{symbol}[/]");
                else
                    bar.Append("[grey].[/]");
            }
            bar.Append("]");
            yield return new Segment(bar.ToString());
        }
    }

    private (Color, string) GetVisuals(double pct)
    {
        if (pct >= 1.0) return (Color.Red, "▓");
        if (pct >= 0.75) return (Color.Orange1, "≡");
        if (pct >= 0.50) return (Color.Magenta1, "≈");
        if (pct >= 0.25) return (Color.Cyan1, "~");
        return (Color.Grey, ".");
    }
}
```

## Configuration & Tuning

The visual intensity should be configurable in `options.json` to support user preferences and accessibility needs.

```json
{
  "Visuals": {
    "EnableScreenShake": true,
    "EnableFlashing": true,
    "UseSimpleResonanceBar": false,
    "GlitchTextIntensity": 1.0
  },
  "Colors": {
    "ResonanceLow": "Grey",
    "ResonanceMedium": "Cyan",
    "ResonanceHigh": "Magenta",
    "ResonanceCritical": "Red"
  }
}
```

- **EnableScreenShake**: Defaults to true. If false, replace shakes with text notifications `[SCREEN SHAKES]`.
- **EnableFlashing**: Defaults to true. If false, avoid rapid color inversions.
- **UseSimpleResonanceBar**: Defaults to false. If true, use standard `[###...]` blocks instead of `[≈≈≈...]` to aid screen readers.

## Accessibility & Localization

### Screen Readers
- **Resonance Bar**: Must implement `IHasCulture` or provide an accessible alternative text if inspected.
- **Flashing**: All flash effects must respect the `EnableFlashing` setting to prevent seizures.
- **Glitch Text**: Glitch text is unreadable by TTS. Provide a "Clean Text" mode where glitch effects are disabled or appended as descriptions `(Text Corrupted)`.

### Localization
- All Grimoire text (School names, Spell descriptions) must be pulled from `LocaleService`.
- Paradox messages ("REALITY BENDS") must be localized keys.
- Rune ASCII art is universal and does not need translation, but the "Meaning" tooltip does.

---

## Performance Considerations

- **Redraw Frequency**: The Resonance Widget changes frequently. Ensure it only redraws when the value changes, not every tick.
- **Galdr Overlay**: Should be a lightweight layer on top of the grid. Avoid clearing the entire screen to update just the overlay. Use `AnsiConsole.Cursor.Move` if possible (though Spectre.Console handles this via Live Display).
- **Glitch Text**: Generating Zalgo text can be CPU intensive if done on large blocks. Limit to titles and short log messages.

## Testing Strategy

### Unit Tests
- **ResonanceWidgetTests**:
  - Verify correct color/char output for boundary values (24, 25, 49, 50, etc.).
  - Verify "Simple Mode" rendering.
- **GrimoireRendererTests**:
  - Verify layout generation does not throw on empty spell lists.
  - Verify navigation logic (selection index wrapping).
- **GlitchTextEngineTests**:
  - Verify deterministic output with fixed seed.
  - Verify no corruption when intensity is 0.

### Integration Tests
- **GrimoireIntegrationTests**:
  - Simulate opening Grimoire, selecting spell, assigning slot, and verifying `QuickSlot` state in `Player` object.

### Manual Verification (The "Feel" Test)
1. **The Pulse**: At 75+ Resonance, does the UI feel unstable?
2. **The Impact**: Does a Catastrophic Paradox feel like a punishment?
3. **Readability**: Is the Grimoire legible on a standard 80x24 terminal?
4. **Accessibility**: Turn off flashing/shake in settings. Verify UI is calm and readable.

### Playwright Verification
Since this is TUI, automated UI testing is limited to checking HTML output of a web-based terminal emulator or checking the string buffer. We will rely on **Manual QA** for VFX timing.
