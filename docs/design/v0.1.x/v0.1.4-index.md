## v0.1.4 - World Persistence & Evolution

**Focus:** Living dungeons that remember and change

### Features
- [ ] **Persistent World State**:
  - Cleared rooms stay cleared (with optional respawn timer)
  - Opened chests/solved puzzles stay resolved
  - Destroyed barriers remain destroyed
- [ ] **Monster Repopulation**:
  - Configurable respawn timers per room type
  - Respawn source points (spawner rooms, nests)
  - Clear all spawners = permanent clear
- [ ] **Faction Territory Control**:
  - Monster factions claim dungeon zones
  - Territory expansion based on player actions
  - Cross-faction conflict
- [ ] **Environmental Storytelling** - Procedural evidence:
  - Battle sites (dead monsters, broken equipment)
  - Time-based changes (fires burn out)
- [ ] **Dungeon Evolution Events**:
  - Cave-ins (paths change)
  - Monster migrations
  - Seasonal effects

### Customization Features
- [ ] **Repopulation Rules** - Configurable respawn behavior
- [ ] **Faction Definitions** - Monster group territories and relationships
- [ ] **Evolution Event Definitions** - Triggerable world changes
- [ ] **Persistence Rules** - What persists and what resets

### Data Model Updates
- [ ] Create `RoomState` entity tracking persistent changes
- [ ] Create `FactionDefinition` entity for monster groups
- [ ] Create `TerritoryControl` tracking faction zone ownership
- [ ] Create `WorldEvent` entity for scheduled changes
- [ ] Create `EnvironmentalEvidence` for procedural storytelling
