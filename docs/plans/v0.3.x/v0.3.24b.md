# Implementation Plan - v0.3.24b: The Golden Master (Alpha Packaging)

**Status:** Planned
**Version:** v0.3.24b
**Milestone:** v0.3.24 (The Precursor)

## Goal
Prepare the application for "Release" distribution. This involves setting the version to v0.4.0, configuring production logging levels, and consolidating database migrations into a clean slate.

## 1. Versioning: `BuildInfo`

Establish a single source of truth for version information.

### 1.1 Create `BuildInfo.cs`
- **File:** `RuneAndRust.Core/Constants/BuildInfo.cs`
- **Content:**
  ```csharp
  public static class BuildInfo
  {
      public const string Version = "0.4.0-alpha";
      public const string Codename = "The Saga";
      // This should be updated by the build pipeline in a real scenario, but static is fine for now
      public static readonly DateTime BuildDate = DateTime.UtcNow;

      public static bool IsDebug
      {
          get
          {
  #if DEBUG
              return true;
  #else
              return false;
  #endif
          }
      }
  }
  ```

### 1.2 Update Title Screen (Optional)
- **File:** `RuneAndRust.Terminal/Services/TitleScreenService.cs` (or similar)
- **Change:** Display `BuildInfo.Version` and `BuildInfo.Codename` on the title screen.

## 2. Configuration: Production Logging

Reduce log noise in production builds.

### 2.1 Update `Program.cs`
- **File:** `RuneAndRust.Terminal/Program.cs`
- **Change:**
  - Modify the `Log.Logger` configuration to use conditional logic.
  - **DEBUG:** `MinimumLevel.Debug()`, Write to Console and File.
  - **RELEASE:** `MinimumLevel.Information()`, Write to File. Console logging should be restricted to `Warning` or `Error` to avoid interfering with TUI, or removed entirely if `AnsiConsole` is handling everything.

## 3. Database: Migration Consolidation

Squash all existing migrations into a single initial migration for v0.4.0.

### 3.1 Delete Existing Migrations
- **Directory:** `RuneAndRust.Persistence/Migrations/`
- **Action:** Delete all files except `.gitkeep` (if it exists).

### 3.2 Reset Local Database
- **Action:** Delete the local `RuneAndRust.db` or drop the Postgres database.
- **Command:** `dotnet ef database drop --force` (if EF CLI is available) or manual deletion.

### 3.3 Create New Migration
- **Command:** `dotnet ef migrations add Initial_v0_4_0`
- **Action:** This will generate a snapshot of the current model as the baseline.

### 3.4 Verify Update
- **Command:** `dotnet ef database update`
- **Action:** Ensure the database is created and seeded correctly.

## 4. Verification Steps

1.  **Version Check:** Run the game. Verify the version is displayed as `0.4.0-alpha`.
2.  **Log Check (Release):**
    -   Build in Release.
    -   Run the game.
    -   Check `logs/runeandrust.log`. It should contain `Information` level logs.
    -   Check Console output. It should be clean of debug logs.
3.  **Migration Check:**
    -   Inspect `RuneAndRust.Persistence/Migrations`. Should only contain `Initial_v0_4_0`.
    -   Connect to the database and verify tables exist.

## 5. Post-Verification
- **Commit:** Commit the changes with the message "chore: v0.3.24b Golden Master preparation".
