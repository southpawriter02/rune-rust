# Implementation Plan: v0.4.1d - The Grid (Specialization UI)

**Status:** Ready for Review
**Theme:** "The Grid"
**Scope:** TUI rendering, controller input handling, ViewModel composition, Spectre.Console grid visualization
**Depends On:** v0.4.1a (Data Models), v0.4.1b (Unlock Service), v0.4.1c (Ability Integration)

## Executive Summary

v0.4.1d implements the **Specialization UI** ("The Grid") — a TUI screen for visualizing ability trees and spending Progression Points. Players navigate nodes, view ability details, and purchase unlocks through an interactive terminal interface.

**Key Features:**
- **Tree Visualization:** Tier-grouped table with node states (Locked/Available/Unlocked)
- **Node Selection:** Arrow key navigation with visual selection indicator
- **Purchase Flow:** Enter to buy, with confirmation feedback
- **Detail Panel:** View ability description before purchase
- **Integration:** Accessible from Character Sheet or dedicated menu option

**Layers Touched:** Core (ViewModels, Interfaces), Engine (ViewModel builders), Terminal (Renderer, Controller)
**Patterns Used:** Controller-Renderer separation, ViewModel-driven rendering, State machine navigation
**Target Test Count:** 52 new unit tests

### 3.3 SpecializationGridViewModel

**File:** `RuneAndRust.Core/ViewModels/SpecializationGridViewModel.cs`

```csharp
namespace RuneAndRust.Core.ViewModels;

/// <summary>
/// View model for the full specialization tree grid.
/// </summary>
/// <remarks>See: v0.4.1d (The Grid) for Specialization UI implementation.</remarks>
public class SpecializationGridViewModel
{
    /// <summary>Specialization ID being displayed.</summary>
    public Guid SpecializationId { get; init; }

    /// <summary>Specialization name for header.</summary>
    public string SpecializationName { get; init; } = string.Empty;

    /// <summary>Specialization description/flavor text.</summary>
    public string SpecializationDescription { get; init; } = string.Empty;

    /// <summary>Character's current Progression Points.</summary>
    public int ProgressionPoints { get; init; }

    /// <summary>Character name for header display.</summary>
    public string CharacterName { get; init; } = string.Empty;

    /// <summary>All nodes grouped by tier (1-4).</summary>
    public IReadOnlyDictionary<int, IReadOnlyList<NodeViewModel>> NodesByTier { get; init; }
        = new Dictionary<int, IReadOnlyList<NodeViewModel>>();

    /// <summary>Flat list of all nodes in display order.</summary>
    public IReadOnlyList<NodeViewModel> AllNodes { get; init; } = Array.Empty<NodeViewModel>();

    /// <summary>Currently selected node index (0-based in AllNodes).</summary>
    public int SelectedNodeIndex { get; set; }

    /// <summary>Currently selected node for display.</summary>
    public NodeViewModel? SelectedNode =>
        SelectedNodeIndex >= 0 && SelectedNodeIndex < AllNodes.Count
            ? AllNodes[SelectedNodeIndex]
            : null;

    /// <summary>Total nodes in the tree.</summary>
    public int TotalNodes => AllNodes.Count;

    /// <summary>Count of unlocked nodes.</summary>
    public int UnlockedCount => AllNodes.Count(n => n.Status == NodeStatus.Unlocked);

    /// <summary>Feedback message to display (purchase result, errors).</summary>
    public string? FeedbackMessage { get; set; }

    /// <summary>Whether feedback is a success (green) or error (red).</summary>
    public bool FeedbackIsSuccess { get; set; }

    /// <summary>Index of current specialization (for multi-spec tab switching).</summary>
    public int CurrentSpecIndex { get; set; }

    /// <summary>Total count of unlocked specializations (for tab indicator).</summary>
    public int TotalSpecCount { get; set; }
}
```

## 5. Controller Implementation

### 5.1 SpecializationController

**File:** `RuneAndRust.Terminal/Controllers/SpecializationController.cs`

**Key Design Points:**
- Follows SagaController pattern exactly
- Manages SelectedNodeIndex state
- Calls ISpecializationService for purchases
- Logs at Trace/Debug/Info levels per standard

```csharp
using Microsoft.Extensions.Logging;
using RuneAndRust.Core.Entities;
using RuneAndRust.Core.Enums;
using RuneAndRust.Core.Interfaces;
using RuneAndRust.Core.ViewModels;

namespace RuneAndRust.Terminal.Controllers;

/// <summary>
/// Controller for the Specialization Grid ("The Grid") input handling (v0.4.1d).
/// Manages node selection, tier navigation, and purchase actions.
/// </summary>
/// <remarks>See: v0.4.1d (The Grid) for Specialization UI implementation.</remarks>
public class SpecializationController
{
    private readonly ISpecializationService _specService;
    private readonly ISpecializationGridViewModelBuilder _vmBuilder;
    private readonly ILogger<SpecializationController> _logger;

    private SpecializationGridViewModel? _currentVm;

    /// <summary>
    /// Gets the current view model for rendering.
    /// </summary>
    public SpecializationGridViewModel? CurrentViewModel => _currentVm;

    public SpecializationController(
        ISpecializationService specService,
        ISpecializationGridViewModelBuilder vmBuilder,
        ILogger<SpecializationController> logger)
    {
        _specService = specService;
        _vmBuilder = vmBuilder;
        _logger = logger;
    }

    /// <summary>
    /// Initializes the controller for a character's specialization.
    /// </summary>
    public async Task InitializeAsync(Character character, Guid specializationId)
    {
        _logger.LogDebug("[Spec UI] Initializing for {CharName}, Spec={SpecId}",
            character.Name, specializationId);

        _currentVm = await _vmBuilder.BuildAsync(character, specializationId);

        _logger.LogDebug("[Spec UI] Loaded {NodeCount} nodes, {UnlockedCount} unlocked",
            _currentVm.TotalNodes, _currentVm.UnlockedCount);
    }

    /// <summary>
    /// Processes a keypress and returns the resulting game phase.
    /// </summary>
    public async Task<GamePhase> HandleInputAsync(ConsoleKey key, Character character)
    {
        if (_currentVm == null)
        {
            _logger.LogWarning("[Spec UI] HandleInputAsync called with null VM");
            return GamePhase.Exploration;
        }

        switch (key)
        {
            // Vertical navigation within tier
            case ConsoleKey.UpArrow:
            case ConsoleKey.W:
                NavigateVertical(-1);
                break;

            case ConsoleKey.DownArrow:
            case ConsoleKey.S:
                NavigateVertical(1);
                break;

            // Horizontal navigation between tiers
            case ConsoleKey.LeftArrow:
            case ConsoleKey.A:
                NavigateHorizontal(-1);
                break;

            case ConsoleKey.RightArrow:
            case ConsoleKey.D:
                NavigateHorizontal(1);
                break;

            // Purchase action
            case ConsoleKey.Enter:
            case ConsoleKey.Spacebar:
                await AttemptPurchaseAsync(character);
                break;

            // Tab to switch specializations (multi-spec)
            case ConsoleKey.Tab:
                await SwitchSpecializationAsync(character);
                break;

            // Exit
            case ConsoleKey.Escape:
            case ConsoleKey.Q:
                _logger.LogInformation("[Spec UI] Exiting Grid, returning to Exploration");
                Reset();
                return GamePhase.Exploration;
        }

        return GamePhase.SpecializationMenu;
    }

    private void NavigateVertical(int delta)
    {
        if (_currentVm == null) return;

        var oldIndex = _currentVm.SelectedNodeIndex;
        var newIndex = Math.Clamp(
            oldIndex + delta,
            0,
            _currentVm.AllNodes.Count - 1);

        if (newIndex != oldIndex)
        {
            _currentVm.SelectedNodeIndex = newIndex;
            _logger.LogTrace("[Spec UI] Selection moved {Dir} to index {Index}",
                delta < 0 ? "up" : "down", newIndex);
        }
    }

    private void NavigateHorizontal(int delta)
    {
        if (_currentVm == null || _currentVm.SelectedNode == null) return;

        var currentTier = _currentVm.SelectedNode.Tier;
        var targetTier = Math.Clamp(currentTier + delta, 1, 4);

        if (targetTier == currentTier) return;

        // Find first node in target tier
        if (_currentVm.NodesByTier.TryGetValue(targetTier, out var tierNodes) && tierNodes.Count > 0)
        {
            var targetNode = tierNodes[0];
            var newIndex = _currentVm.AllNodes.ToList().FindIndex(n => n.NodeId == targetNode.NodeId);
            if (newIndex >= 0)
            {
                _currentVm.SelectedNodeIndex = newIndex;
                _logger.LogTrace("[Spec UI] Moved to Tier {Tier}, index {Index}",
                    targetTier, newIndex);
            }
        }
    }

    private async Task AttemptPurchaseAsync(Character character)
    {
        if (_currentVm?.SelectedNode == null) return;

        var node = _currentVm.SelectedNode;

        if (node.Status == NodeStatus.Unlocked)
        {
            _currentVm.FeedbackMessage = "Already inscribed.";
            _currentVm.FeedbackIsSuccess = false;
            _logger.LogDebug("[Spec UI] Purchase rejected: {NodeName} already unlocked", node.Name);
            return;
        }

        if (node.Status == NodeStatus.Locked)
        {
            _currentVm.FeedbackMessage = "Prerequisites not met.";
            _currentVm.FeedbackIsSuccess = false;
            _logger.LogDebug("[Spec UI] Purchase rejected: {NodeName} prerequisites not met", node.Name);
            return;
        }

        var result = await _specService.UnlockNodeAsync(character, node.NodeId);

        if (result.Success)
        {
            _currentVm.FeedbackMessage = $"Inscribed {node.Name}!";
            _currentVm.FeedbackIsSuccess = true;
            _logger.LogInformation("[Spec UI] Successfully inscribed {NodeName} for {Cost} PP",
                node.Name, node.CostPP);

            // Refresh VM to reflect new state
            _currentVm = await _vmBuilder.RefreshAsync(_currentVm, character);
        }
        else
        {
            _currentVm.FeedbackMessage = result.Message;
            _currentVm.FeedbackIsSuccess = false;
            _logger.LogDebug("[Spec UI] Purchase failed: {Reason}", result.Message);
        }
    }

    private async Task SwitchSpecializationAsync(Character character)
    {
        if (_currentVm == null) return;

        if (_currentVm.TotalSpecCount <= 1)
        {
            _logger.LogTrace("[Spec UI] Tab ignored: only one specialization");
            return;
        }

        // Cycle to next spec (implementation depends on character.UnlockedSpecializationIds)
        var nextIndex = (_currentVm.CurrentSpecIndex + 1) % _currentVm.TotalSpecCount;
        // TODO: Get spec ID at nextIndex and reinitialize
        _logger.LogDebug("[Spec UI] Switching to spec index {Index}", nextIndex);
    }

    /// <summary>
    /// Resets the controller state for next entry.
    /// </summary>
    public void Reset()
    {
        if (_currentVm != null)
        {
            _currentVm.SelectedNodeIndex = 0;
            _currentVm.FeedbackMessage = null;
        }
        _currentVm = null;
    }
}
```

## 6. Renderer Implementation

### 6.1 SpecializationGridRenderer

**File:** `RuneAndRust.Terminal/Rendering/SpecializationGridRenderer.cs`

**Key Design Points:**
- Follows SagaScreenRenderer pattern
- Uses Spectre.Console Table for tier-grouped display
- Panel layout with Header, Grid, Detail, Footer sections
- Color-coded node status (consistent with NodeViewModel.StatusMarkup)

```csharp
using Microsoft.Extensions.Logging;
using RuneAndRust.Core.Enums;
using RuneAndRust.Core.Interfaces;
using RuneAndRust.Core.ViewModels;
using Spectre.Console;

namespace RuneAndRust.Terminal.Rendering;

/// <summary>
/// Renders the Specialization Tree Grid using Spectre.Console (v0.4.1d).
/// Displays nodes by tier with visual selection and status indicators.
/// </summary>
/// <remarks>See: v0.4.1d (The Grid) for Specialization UI implementation.</remarks>
public class SpecializationGridRenderer : ISpecializationGridRenderer
{
    private readonly IThemeService _theme;
    private readonly ILogger<SpecializationGridRenderer> _logger;

    public SpecializationGridRenderer(
        IThemeService theme,
        ILogger<SpecializationGridRenderer> logger)
    {
        _theme = theme;
        _logger = logger;
    }

    public void Render(SpecializationGridViewModel vm)
    {
        AnsiConsole.Clear();

        // Header - Spec name and PP balance
        RenderHeader(vm);

        // Main Grid - Nodes by tier
        RenderGrid(vm);

        // Detail Panel - Selected node description
        RenderDetailPanel(vm);

        // Feedback - Purchase result
        RenderFeedback(vm);

        // Footer - Controls
        RenderFooter(vm);

        _logger.LogTrace("[Spec UI] Rendered. Selected: {Index}, Node: {NodeName}",
            vm.SelectedNodeIndex, vm.SelectedNode?.Name ?? "none");
    }

    private static void RenderHeader(SpecializationGridViewModel vm)
    {
        // Spec name with tab indicator for multi-spec
        var tabIndicator = vm.TotalSpecCount > 1
            ? $" [{vm.CurrentSpecIndex + 1}/{vm.TotalSpecCount}]"
            : "";

        AnsiConsole.Write(new Rule($"[bold gold1]✦ {vm.SpecializationName.ToUpperInvariant()}{tabIndicator} ✦[/]")
            .RuleStyle("gold1"));
        AnsiConsole.WriteLine();

        // Character and PP
        AnsiConsole.MarkupLine($"  {vm.CharacterName}    " +
            $"Progression Points: [yellow bold][{vm.ProgressionPoints}][/]    " +
            $"Progress: [cyan]{vm.UnlockedCount}/{vm.TotalNodes}[/]");
        AnsiConsole.WriteLine();
    }

    private void RenderGrid(SpecializationGridViewModel vm)
    {
        var table = new Table()
            .Border(TableBorder.Rounded)
            .BorderColor(Color.Grey)
            .AddColumn(new TableColumn("[grey]TIER 1[/]").Width(18))
            .AddColumn(new TableColumn("[grey]TIER 2[/]").Width(18))
            .AddColumn(new TableColumn("[grey]TIER 3[/]").Width(18))
            .AddColumn(new TableColumn("[grey]CAPSTONE[/]").Width(18));

        // Find max nodes in any tier for row count
        var maxNodesPerTier = vm.NodesByTier.Values.Max(n => n.Count);

        for (int row = 0; row < maxNodesPerTier; row++)
        {
            var cells = new string[4];

            for (int tier = 1; tier <= 4; tier++)
            {
                if (vm.NodesByTier.TryGetValue(tier, out var tierNodes) && row < tierNodes.Count)
                {
                    var node = tierNodes[row];
                    var isSelected = vm.SelectedNode?.NodeId == node.NodeId;
                    cells[tier - 1] = FormatNodeCell(node, isSelected);
                }
                else
                {
                    cells[tier - 1] = "";
                }
            }

            table.AddRow(cells);
        }

        AnsiConsole.Write(table);
        AnsiConsole.WriteLine();
    }

    private static string FormatNodeCell(NodeViewModel node, bool isSelected)
    {
        var indicator = isSelected ? "▶ " : "  ";
        var color = node.Status switch
        {
            NodeStatus.Unlocked => "green",
            NodeStatus.Available => "cyan",
            NodeStatus.Affordable => "yellow",
            NodeStatus.Locked => "red",
            _ => "grey"
        };

        var icon = node.Status switch
        {
            NodeStatus.Unlocked => "◆",
            NodeStatus.Available => "○",
            NodeStatus.Affordable => "○",
            NodeStatus.Locked => "×",
            _ => "?"
        };

        var nameDisplay = node.Name.Length > 12
            ? node.Name[..12] + "…"
            : node.Name;

        return isSelected
            ? $"[bold white]{indicator}{icon} {nameDisplay}[/]"
            : $"[{color}]{indicator}{icon} {nameDisplay}[/]";
    }

    private static void RenderDetailPanel(SpecializationGridViewModel vm)
    {
        if (vm.SelectedNode == null) return;

        var node = vm.SelectedNode;

        var panel = new Panel(
            new Markup($"{Markup.Escape(node.Description)}\n\n" +
                       $"Cost: [yellow]{node.CostDisplay}[/]  |  " +
                       $"Status: {node.StatusMarkup}"))
            .Header($"[bold white]◈ {node.Name} (Tier {node.Tier}){(node.IsCapstone ? " [gold1]★ CAPSTONE[/]" : "")}[/]")
            .Border(BoxBorder.Rounded)
            .BorderColor(Color.Grey)
            .Expand();

        AnsiConsole.Write(panel);
        AnsiConsole.WriteLine();
    }

    private static void RenderFeedback(SpecializationGridViewModel vm)
    {
        if (string.IsNullOrEmpty(vm.FeedbackMessage)) return;

        var color = vm.FeedbackIsSuccess ? "green" : "red";
        AnsiConsole.MarkupLine($"  [{color}]» {Markup.Escape(vm.FeedbackMessage)}[/]");
        AnsiConsole.WriteLine();
    }

    private static void RenderFooter(SpecializationGridViewModel vm)
    {
        var tabHint = vm.TotalSpecCount > 1 ? "Tab: Switch Spec  │  " : "";
        AnsiConsole.MarkupLine($"[dim]↑↓ Navigate  │  ←→ Tier  │  Enter: Inscribe  │  {tabHint}Esc: Return[/]");
    }
}
```

### 11.1 ViewModel Builder Implementation

```csharp
public class SpecializationGridViewModelBuilder : ISpecializationGridViewModelBuilder
{
    private readonly ISpecializationRepository _specRepo;
    private readonly ISpecializationService _specService;
    private readonly ILogger<SpecializationGridViewModelBuilder> _logger;

    public SpecializationGridViewModelBuilder(
        ISpecializationRepository specRepo,
        ISpecializationService specService,
        ILogger<SpecializationGridViewModelBuilder> logger)
    {
        _specRepo = specRepo;
        _specService = specService;
        _logger = logger;
    }

    public async Task<SpecializationGridViewModel> BuildAsync(Character character, Guid specializationId)
    {
        _logger.LogTrace("[SpecVM] BuildAsync for {CharName}, SpecId={SpecId}",
            character.Name, specializationId);

        var spec = await _specRepo.GetByIdAsync(specializationId);
        if (spec == null)
            throw new ArgumentException($"Specialization {specializationId} not found");

        var nodes = await _specRepo.GetNodesForSpecializationAsync(specializationId);

        var nodeVms = new List<NodeViewModel>();
        foreach (var node in nodes.OrderBy(n => n.Tier).ThenBy(n => n.PositionY))
        {
            var status = await ComputeNodeStatusAsync(character, node);
            _logger.LogTrace("[SpecVM] Node {NodeName}: Status={Status}", node.Name, status);

            nodeVms.Add(new NodeViewModel(
                NodeId: node.Id,
                AbilityId: node.AbilityId,
                Name: node.Name,
                Description: node.Description,
                Tier: node.Tier,
                CostPP: node.CostPP,
                Status: status,
                PositionX: node.PositionX,
                PositionY: node.PositionY,
                IsCapstone: node.Tier == 4,
                ParentNodeIds: node.ParentNodeIds));
        }

        var nodesByTier = nodeVms
            .GroupBy(n => n.Tier)
            .ToDictionary(g => g.Key, g => (IReadOnlyList<NodeViewModel>)g.ToList());

        var vm = new SpecializationGridViewModel
        {
            SpecializationId = specializationId,
            SpecializationName = spec.Name,
            SpecializationDescription = spec.Description,
            ProgressionPoints = character.ProgressionPoints,
            CharacterName = character.Name,
            NodesByTier = nodesByTier,
            AllNodes = nodeVms,
            SelectedNodeIndex = 0,
            CurrentSpecIndex = 0,
            TotalSpecCount = character.UnlockedSpecializationIds.Count
        };

        _logger.LogDebug("[SpecVM] Built VM with {NodeCount} nodes", nodeVms.Count);
        return vm;
    }

    private async Task<NodeStatus> ComputeNodeStatusAsync(Character character, SpecializationNode node)
    {
        // Check if already unlocked
        if (character.UnlockedNodeIds.Contains(node.Id))
            return NodeStatus.Unlocked;

        // Check prerequisites
        var (prereqsMet, _) = await _specService.ValidatePrerequisitesAsync(character, node);
        if (!prereqsMet)
            return NodeStatus.Locked;

        // Check PP
        if (character.ProgressionPoints < node.CostPP)
            return NodeStatus.Affordable; // Prereqs met but can't afford

        return NodeStatus.Available;
    }

    public async Task<SpecializationGridViewModel> RefreshAsync(
        SpecializationGridViewModel existing,
        Character character)
    {
        _logger.LogTrace("[SpecVM] RefreshAsync for {SpecName}", existing.SpecializationName);

        var refreshed = await BuildAsync(character, existing.SpecializationId);

        // Preserve selection
        refreshed.SelectedNodeIndex = Math.Min(
            existing.SelectedNodeIndex,
            refreshed.AllNodes.Count - 1);
        refreshed.FeedbackMessage = existing.FeedbackMessage;
        refreshed.FeedbackIsSuccess = existing.FeedbackIsSuccess;

        _logger.LogDebug("[SpecVM] Refreshed, {UnlockedCount} now unlocked",
            refreshed.UnlockedCount);

        return refreshed;
    }
}
```

## 15. Design Decisions

### Why Tier-Grouped Table Instead of Visual Tree?

**Problem:** Rendering a true tree visualization (nodes connected by lines) in TUI is complex and hard to read.

**Decision:** Use a 4-column table where each column represents a tier (1-4).

**Rationale:**
- Spectre.Console Tables are robust and well-tested
- Horizontal layout naturally shows progression (Tier 1 → Capstone)
- Consistent with SagaScreenRenderer's table-based approach
- Easier to navigate with arrow keys
- Defers visual complexity to a future Avalonia UI implementation

### Why Separate Builder from Controller?

**Problem:** Controllers mixing data loading with input handling become complex.

**Decision:** SpecializationGridViewModelBuilder handles data composition; Controller handles input.

**Rationale:**
- Single Responsibility Principle
- Builder is easily testable without Console dependencies
- Controller focuses on state transitions and service calls
- Matches existing SagaController/SagaScreenRenderer split

### Why Four NodeStatus Values Instead of Two?

**Problem:** Binary Locked/Unlocked doesn't convey enough information.

**Decision:** Locked, Available, Affordable, Unlocked.

**Rationale:**
- **Unlocked:** Already purchased (green, diamond icon)
- **Available:** Can purchase now (cyan, circle icon)
- **Affordable:** Prerequisites met but insufficient PP (yellow, indicates save-up)
- **Locked:** Prerequisites not met (red, X icon)
- Provides clear player guidance without tooltip requirements

### Why Store Selection Index in ViewModel?

**Problem:** Controller state becomes complex when managing both selection and VM.

**Decision:** SelectedNodeIndex lives in SpecializationGridViewModel.

**Rationale:**
- Renderer can access selection directly for highlighting
- Refresh can preserve selection across VM rebuilds
- Single source of truth for UI state
- Matches SagaViewModel pattern with SelectedIndex
