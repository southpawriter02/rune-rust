# v0.3.24a: The Broom (Deprecation Cleanup) - Expanded Implementation Plan

## Overview

A focused cleanup pass to remove deprecated code, secure debug commands, and standardize output before the v0.4.0 Alpha release. This ensures the codebase is production-ready and free of "temporary" v0.0.x artifacts.

**Parent Plan:** [v0.3.24](./v0.3.24.md)
**Status:** Planned

---

## 1. Deprecation: Remove `GenerateTestMapAsync`

The `GenerateTestMapAsync` method is a legacy artifact from v0.0.5 used for hardcoded test generation. It must be removed and replaced with the proper `GenerateDungeonAsync` pipeline.

### Files to Modify

| File | Change |
|------|--------|
| `RuneAndRust.Terminal/Program.cs` | Replace call to `GenerateTestMapAsync()` with `GenerateDungeonAsync("the_roots")`. |
| `RuneAndRust.Tests/Engine/DungeonGeneratorTests.cs` | Refactor tests to use `GenerateDungeonAsync` or abstract test map creation. |
| `RuneAndRust.Engine/Services/DungeonGenerator.cs` | Delete the `GenerateTestMapAsync()` method entirely. |

### Code Specification

**Program.cs (Line ~420):**
```csharp
// Before
var startRoomId = dungeonGen.GenerateTestMapAsync().GetAwaiter().GetResult();

// After
var startRoomId = dungeonGen.GenerateDungeonAsync("the_roots").GetAwaiter().GetResult();
```

**DungeonGeneratorTests.cs:**
Tests that rely on the specific 3-room layout of `GenerateTestMapAsync` should be updated to assert against the properties of the generated dungeon or mock the template repo if specific topology is needed.

## 2. Security: Lock Down Debug Commands

Debug commands and "Training Dummies" should not be accessible in Release builds.

### Files to Modify

| File | Change |
|------|--------|
| `RuneAndRust.Engine/Services/CommandParser.cs` | Wrap `debug-combat` case and private fields in `#if DEBUG`. |
| `RuneAndRust.Core/Entities/Enemy.cs` | Change default `Name` from "Training Dummy" to "Unknown Enemy". |

### Code Specification

**CommandParser.cs (Line ~303, ~909):**
```csharp
// Constructor
#if DEBUG
    _debugConsoleRenderer = debugConsoleRenderer;
#endif

// ParseMethod
#if DEBUG
    case "debug-combat":
        // ... existing logic ...
        return ParseResult.None;
#endif
```

**Enemy.cs (Line 22):**
```csharp
public string Name { get; set; } = "Unknown Enemy"; // Was "Training Dummy"
```

### Files to Modify

## 4. Housekeeping: Console.WriteLine Audit

Direct console output bypasses the logger and UI renderer.

### Files to Modify

| File | Change |
|------|--------|
| `RuneAndRust.Engine/Services/EmergencySaveService.cs` | Replace `Console.WriteLine` with `AnsiConsole.MarkupLine` or comment as intentional crash fallback. |

### Code Specification

**EmergencySaveService.cs:**
Since this service runs during a crash, `AnsiConsole` might also be unstable, but `Console.WriteLine` is accepting as a last resort. We will add a comment to clarify intent.

```csharp
// INTENTIONAL: Fallback output during critical system failure
Console.WriteLine($"[System] Emergency save attempted: {result}");
```

## 5. Verification Steps

1.  **Build Release:** Run `dotnet build -c Release`.
2.  **Verify Debug Lock:** Run the game in Release mode.
    - Type `debug-combat`. Expected: "Unknown command".
    - Type `~` (tilde). Expected: "Unknown command" or no reaction.
3.  **Verify Game Start:** Start a new game. Ensure the dungeon generates correctly using "the_roots" template instead of the test map.
4.  **Run Tests:** `dotnet test`. Ensure `DungeonGeneratorTests` still pass without the legacy method.
