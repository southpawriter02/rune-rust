This version introduces the element of risk to the resting mechanic. It implements the Camp Craft skill check to mitigate danger and the procedural generation of Ambush Encounters when safety fails.

v0.3.2b: The Watch (Ambush System)
Goal: Implement the logic for calculating rest safety based on Biome and Skill, determining ambush outcomes, and seamlessly transitioning from a "Rest" attempt into a "Combat" state if triggered.

1. Architecture & Data Flow
The RestService now acts as a gateway. Before executing the recovery logic (v0.3.2a), it must clear a safety check.
Workflow: The Watch Cycle
	1	Input: Player initiates camp (Wilderness Rest).
	2	Risk Calculation:
	◦	Retrieve Room.DangerLevel and Room.BiomeType.
	◦	Calculate Base Ambush Chance (e.g., 10% to 50%).
	3	Mitigation (Camp Craft):
	◦	Roll WITS + Wasteland Survival skill.
	◦	Reduce Base Chance by 5% * NetSuccesses.
	4	Determination:
	◦	Roll d100 against Final Chance.
	5	Branching Outcome:
	◦	Safe: Proceed to PerformRestAsync (v0.3.2a logic).
	◦	Ambush:
	▪	Cancel Rest (Resources consumed, no recovery).
	▪	Call EncounterGenerationService.GenerateAmbush().
	▪	Initialize Combat with Surprise Round (Enemies act first or Player [Stunned] turn 1).
	6	Feedback: Return AmbushResult to UI.

2. Logic Decision Trees
A. Ambush Probability Logic
Input: DangerLevel, WITS, Skill Rank
	1	Determine Base Risk:
	◦	Safe: 0% (Always safe).
	◦	Unstable: 15%.
	◦	Hostile: 30%.
	◦	Lethal: 50%.
	2	Determine Mitigation:
	◦	Pool = WITS + WastelandSurvivalRank.
	◦	Roll = DiceService.Roll(Pool).
	◦	Reduction = Successes * 5.
	3	Calculate Final Chance:
	◦	Chance = Max(5, BaseRisk - Reduction). (Minimum 5% risk in Hostile/Lethal zones, regardless of skill).
	◦	Exception: Safe zones remain 0%.
B. Ambush Generation Logic
Input: Biome, Depth
	1	Budget Calculation:
	◦	Use standard SpawnBudget for the room/depth.
	◦	Modifier: Multiply Budget by 0.8 (Ambushes are usually slightly smaller raiding parties, not entrenched armies).
	2	Composition:
	◦	Prioritize Stealth or Fast enemy types (e.g., Ash-Vargr, Skrap-Hounds).
	3	State Setup:
	◦	Player Status: Apply [Disoriented] (1 turn) to represent waking up groggy.

public async Task<AmbushResult> CheckForAmbushAsync(Character character, Room room)
{
    // 1. Base Risk Calculation
    int baseRisk = room.DangerLevel switch
    {
        DangerLevel.Safe => 0,
        DangerLevel.Unstable => 15,
        DangerLevel.Hostile => 30,
        DangerLevel.Lethal => 50,
        _ => 15
    };

    if (baseRisk == 0) return AmbushResult.Safe(0, 0);

    // 2. Camp Craft Mitigation
    // We assume SkillService or Character has a helper for getting skill rank
    int survivalRank = character.GetSkillRank(SkillType.WastelandSurvival);
    int wits = character.GetAttribute(Attribute.Wits);

    var check = _dice.Roll(wits + survivalRank, "Camp Craft Check");
    int mitigation = check.Successes * 5;

    // 3. Final Calculation (Min 5% risk if not Safe zone)
    int finalRisk = Math.Max(5, baseRisk - mitigation);

    // 4. The Roll
    int roll = _random.Next(1, 101);

    _logger.LogInformation("Ambush Check: Risk {Risk}% (Base {Base} - Mit {Mit}), Roll {Roll}",
        finalRisk, baseRisk, mitigation, roll);

    if (roll <= finalRisk)
    {
        // 5. Generate Encounter
        var encounter = await _encounterGenerator.GenerateEncounterAsync(new EncounterContext
        {
            Biome = room.BiomeType,
            Danger = room.DangerLevel,
            IsAmbush = true // Flag to prioritize fast/stealthy enemies
        });

        return AmbushResult.Triggered(finalRisk, roll, encounter);
    }

    return AmbushResult.Safe(finalRisk, roll);
}

# Changelog: v0.3.2b - The Watch
**Release Date:** 2025-XX-XX

## Summary
Resting in the wilderness is no longer safe. This update introduces the **Ambush System**, requiring players to rely on their Wits and Survival skills to secure a campsite. Failure results in an immediate combat encounter with no recovery benefits.

## Features
- **Camp Craft Check:**
    - Resting in Unstable, Hostile, or Lethal zones triggers a risk calculation.
    - **Wits + Wasteland Survival** determines mitigation. Each success reduces ambush chance by 5%.
    - Minimum risk floor of 5% applies in dangerous zones.
- **Ambush Encounters:**
    - If the check fails, the rest is interrupted.
    - Resources (Ration/Water) are consumed, but NO health/stress is recovered.
    - Enemies spawn immediately, often with initiative advantage.
- **Biome Integration:**
    - Base risk scales with biome danger (Safe: 0% -> Lethal: 50%).

## Technical
- Implemented `AmbushCalculator` logic within `RestService`.
- Linked Resting system to `EncounterGenerationService`.
