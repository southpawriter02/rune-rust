v0.3.9a: The Impact (Visual FX)
Goal: Implement the VisualEffectService to provide visceral feedback for game events (Damage, Crits, Trauma) without breaking the TUI format.
1. Architecture & Data Flow
	•	Trigger: Game events (e.g., AttackResult, TraumaCheck) publish to the EventBus.
	•	Processor: VisualEffectService listens for specific events.
	•	Rendering:
	◦	Screen Shake: Temporarily modifies the root layout padding/margins in a rapid sequence to simulate impact.
	◦	Flash: Changes the terminal background color or panel border colors briefly (e.g., Red flash on damage).
	◦	Text FX: Wraps specific log messages in pulsing or blinking styles.
2. Logic Decision Trees
	•	Effect Trigger Logic:
	1	Event Received: DamageTakenEvent(Amount: 15, IsCrit: True).
	2	Check Settings: Is ReduceMotion enabled?
	▪	Yes: Skip visual FX, play sound cue only (if audio exists).
	3	Determine Intensity:
	▪	Damage > 10% MaxHP -> Heavy Shake (3 frames, high offset).
	▪	Damage < 10% MaxHP -> Light Shake (1 frame).
	4	Execute:
	▪	Update Layout Padding (Offset Left). Render.
	▪	Update Layout Padding (Offset Right). Render.
	▪	Reset.

public async Task TriggerShakeAsync(int intensity)
{
    if (_settings.ReduceMotion) return;

    var originalPadding = _layout.Padding;

    // Simulate shake by adjusting padding rapidly
    for(int i = 0; i < intensity; i++)
    {
        _layout.Padding = new Padding(2, 0, 0, 0); // Shift Right
        _renderer.Refresh();
        await Task.Delay(50);

        _layout.Padding = new Padding(0, 0, 2, 0); // Shift Left
        _renderer.Refresh();
        await Task.Delay(50);
    }

    _layout.Padding = originalPadding; // Reset
    _renderer.Refresh();
}

public Color GetColor(string semanticKey)
{
    var palette = _currentTheme switch
    {
        ThemeType.HighContrast => _highContrastPalette,
        ThemeType.Protanopia => _protanopiaPalette,
        _ => _standardPalette
    };

    return palette.TryGetValue(semanticKey, out var color) ? color : Color.White;
}
