# v0.9.0a Changelog: Core Audio Service

**Version:** 0.9.0a  
**Phase:** Core Audio Service  
**Date:** 2026-01-16  
**Status:** ✅ Complete

---

## Summary

Implements the core audio service with playback capabilities, channel-based volume control (Master, Music, Effects, UI, Voice), and an LRU-evicting audio cache (50 files).

---

## Added

### Application Layer - Enums

| Enum | Description |
|------|-------------|
| `AudioChannel` | 5 channels: Master, Music, Effects, UI, Voice |

### Application Layer - Interfaces

| Interface | Description |
|-----------|-------------|
| `IAudioService` | Core audio playback interface |

**IAudioService Methods:**
- `Play(path, channel, volume, loop) → Guid`
- `Stop/Pause/Resume(Guid)`
- `StopChannel/PauseChannel/ResumeChannel(AudioChannel)`
- `StopAll()`
- `SetChannelVolume/GetChannelVolume`
- `SetChannelMuted/IsChannelMuted`
- `PreloadAsync`, `ClearCache`
- `IsEnabled`, `SetEnabled`

### Infrastructure Layer - Services

| Class | Description |
|-------|-------------|
| `AudioService` | Full implementation of IAudioService |
| `AudioCache` | LRU cache (50 files default) |
| `AudioPlayback` | Playback state tracking |
| `CachedAudio` | Loaded audio data holder |

---

## Default Channel Volumes

| Channel | Default Volume |
|---------|----------------|
| Master | 80% |
| Music | 60% |
| Effects | 80% |
| UI | 70% |
| Voice | 90% |
