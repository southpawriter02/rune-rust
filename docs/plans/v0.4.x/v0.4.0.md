# v0.4.0: The Saga System (Progression & Content)
This milestone shifts the focus from "Survival" (staying alive) to "Growth" (becoming powerful). It implements the long-term progression loops: leveling up via the Saga System, unlocking class-specific Specializations, and navigating the political landscape of Factions.

To manage the complexity of RPG progression logic, this milestone is divided into three major versions, with v0.4.0detailed below.

Milestone 5 Roadmap
	•	v0.4.0: Saga Progression – Core leveling engine, Legend (XP) accumulation, and Attribute/Skill upgrades.
	•	v0.4.1: Specializations – Ability tree unlocking, tier progression, and archetype synergy.
	•	v0.4.2: Factions & Dialogue – Reputation tracking, disposition systems, and skill-gated conversations.

v0.4.0: Saga Progression
Goal: Implement the "Saga System" where players accumulate Legend (XP) to reach Milestones (Levels), earning Progression Points (PP) to permanently upgrade Attributes and Skills.
This version is split into three sub-versions:
	•	v0.4.0a: The Chronicle (Legend Engine) – Backend logic for earning Legend and calculating Milestones.
	•	v0.4.0b: The Choice (Spending Logic) – Engine logic for purchasing upgrades with PP.
	•	v0.4.0c: The Reflection (Saga UI) – The visual interface for leveling up at Sanctuaries.

v0.4.0a: The Chronicle (Legend Engine)
Goal: Implement SagaService to track Legend accumulation, calculate thresholds based on the fibonacci-like progression curve, and trigger Milestone events.
1. Architecture & Data Flow
	•	Currency Split: The system tracks two values:
	◦	Legend: Passive accumulator (XP). Determines Milestone level.
	◦	Progression Points (PP): Active currency. Earned at Milestones, spent on upgrades [Source 247].
	•	Threshold Logic: Milestones require increasing amounts of Legend (500, 600, 700...).
	•	Overflow: Excess Legend must carry over to the next Milestone [Source 248].
Data Flow: The Award Pipeline
	1	Trigger: Event occurs (Enemy Killed, Room Explored, Quest Complete).
	2	Calculation: LegendService calculates amount based on Difficulty Modifiers.
	3	Accumulation: SagaService.AddLegend(amount).
	4	Threshold Check:
	◦	If CurrentLegend >= NextThreshold:
	◦	Increment Milestone.
	◦	Add PP (+1).
	◦	Subtract Threshold from CurrentLegend.
	◦	Recalculate NextThreshold.
	◦	Repeat (in case of massive XP gain).
	5	Event: Publish MilestoneReachedEvent.
2. Logic Decision Trees
A. Legend Award Logic Input: BaseValue, DifficultyLevel
	1	Determine Multiplier:
	◦	Trivial: 0x (No reward for weak enemies) [Source 247].
	◦	Standard: 1.0x.
	◦	Elite/Boss: 1.5x / 2.5x.
	2	Calculate: Amount = BaseValue * Multiplier.
	3	Execute: SagaService.AddLegend(Amount).
B. Milestone Threshold Logic Input: Current Milestone Level (L)
	1	Base Calculation: Threshold = 500 + (L * 100).
	◦	L0 -> L1: 500.
	◦	L1 -> L2: 600.
	◦	L2 -> L3: 700.
	2	Output: Return Threshold Integer.
3. Code Implementation
A. Saga Service File: RuneAndRust.Engine/Services/SagaService.cs
public class SagaService : ISagaService
{
    private readonly GameState _state;
    private readonly IEventBus _eventBus;

    public void AddLegend(int amount)
    {
        var character = _state.CurrentCharacter;
        character.Legend += amount;

        CheckMilestones(character);
    }

    private void CheckMilestones(Character character)
    {
        int threshold = CalculateThreshold(character.Milestone);

        while (character.Legend >= threshold)
        {
            character.Legend -= threshold;
            character.Milestone++;
            character.ProgressionPoints++; // Award PP

            _eventBus.Publish(new MilestoneReachedEvent(character.Milestone));

            // Recalculate for next loop
            threshold = CalculateThreshold(character.Milestone);
        }
    }

    private int CalculateThreshold(int currentMilestone)
    {
        // 500, 600, 700 progression
        return 500 + (currentMilestone * 100);
    }
}

public bool TryUpgradeAttribute(Character character, Attribute attribute)
{
    const int Cost = 3;

    if (character.ProgressionPoints < Cost) return false;

    character.ProgressionPoints -= Cost;

    int currentVal = character.GetAttribute(attribute);
    character.SetAttribute(attribute, currentVal + 1);

    _statService.RecalculateDerivedStats(character); // Update HP/Stamina
    return true;
}

# Changelog: v0.4.0 - Saga Progression

**Release Date:** 2026-03-XX

## Summary
"Saga Progression" introduces the leveling engine of Rune & Rust. Players can now earn Legend (XP) from combat and exploration, reach Milestones, and spend Progression Points (PP) at Runic Anchors to improve their attributes and skills.

## Changes
- **Core:** Added Legend and Milestone tracking to Character entity.
- **Engine:** Implemented `SagaService` with fibonacci-lite progression curve (500, 600, 700...).
- **Combat:** Enemies now award Legend upon defeat based on their Threat Tier.
- **UI:** Added `saga` command (only usable at Sanctuaries) to open the leveling interface.
- **Mechanics:** Attributes can be upgraded for 3 PP; Skills for 2 PP.
