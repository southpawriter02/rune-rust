# v0.19.4a Changelog: Jötunheim & Vanaheim

**Date:** 2026-02-06

## Added

- **Jötunheim biome definition** (Deck 07 — The Industrial Graveyard)
    - Base properties: 10°C, 40% humidity, 0.2 aetheric, 0.5 light, **3.0 scale**, 0.5 corrosion
    - Primary condition: GiantScale (3.0x multiplier for fall damage and physical effects)
    - Vertical range: Z-1 (UpperRoots) to Z+3 (Canopy)
    - 3 zones: The Bone Yards (DC 10), Utgard's Shadow (DC 12), The Grinding Hall (DC 16)
    - Zone progression features increasing corrosion and decreasing temperature/light
    - First realm in the Nine Realms system with IsGiantScale active (scale > 2.0)

- **Vanaheim biome definition** (Deck 03 — The Overgrown Laboratory)
    - Base properties: 28°C, 85% humidity, 0.5 aetheric, 0.6 light, 1.0 scale, 0.4 corrosion
    - Primary condition: MutagenicSpores (cumulative Corruption with Constitution saves)
    - Vertical range: Z-2 (LowerRoots) to Z+3 (Canopy)
    - 3 zones: The Undergrowth (DC 10, 1x corruption), The Canopy Sea (DC 16, 2x corruption), Ymir's Roots (DC 12, 1x corruption)
    - Zone-specific damage dice reflect corruption multiplier: 1d4 (1x), 2d4 (2x), 1d6 (1x)

- **GiantScale environmental condition** already defined in v0.19.3a (type 8); Jötunheim is the first realm to use it as primary condition

- **MutagenicSpores environmental condition** already defined in v0.19.1a (type 4); Vanaheim is the first realm to use it as primary condition

- **10 Jötunheim integration tests** validating base properties, GiantScale condition, vertical range, all 3 zones, faction pools, metadata, and scale factor

- **10 Vanaheim integration tests** validating base properties, MutagenicSpores condition, vertical range, all 3 zones, doubled corruption in Canopy Sea, faction pools, and bio-hazard properties

- **13 Jötunheim domain tests** validating GetEffectiveProperties, GetEffectiveCondition, GetZone lookups, IsGiantScale/IsThermalExtreme derived properties, FullName, HasZones, and VerticalRangeDescription

- **14 Vanaheim domain tests** validating GetEffectiveProperties, GetEffectiveCondition, GetZone lookups, IsGiantScale/IsThermalExtreme/IsAethericallyActive derived properties, FullName, HasZones, and VerticalRangeDescription

## Changed

- Bumped `realm-biomes.json` version from `1.1` to `1.2`
- Total realm biome definitions in config: 5 → 7 (Midgard, Svartalfheim, Helheim, Muspelheim, Niflheim, Jötunheim, Vanaheim)
