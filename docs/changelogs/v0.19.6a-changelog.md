# v0.19.6a Changelog — Transition Zone Generation & Adjacency Verification

**Release Date:** 2026-02-06
**Version:** 0.19.6a
**Focus:** Transition zone generation between realms with property interpolation

---

## Added

### Domain Layer

- **`TransitionZone` record** (`Domain/ValueObjects/TransitionZone.cs`) — Immutable record representing a single transition zone between two realms. Properties include `FromRealm`, `ToRealm`, `SequenceIndex`, `BlendFactor`, `InterpolatedProperties` (blended `RealmBiomeProperties`), and `TransitionTheme`. Includes computed helpers `IsMidpoint`, `DominantRealm`.

### Application Layer

- **`ITransitionZoneGenerator` interface** (`Application/Interfaces/ITransitionZoneGenerator.cs`) — Defines 4 methods for transition zone generation:
    - `GenerateTransition(RealmId, RealmId)` — Single zone at 50% blend
    - `GenerateTransitionSequence(RealmId, RealmId, int)` — Multi-zone sequence (1-3 rooms)
    - `InterpolateProperties(RealmBiomeProperties, RealmBiomeProperties, double)` — Linear property interpolation
    - `CanGenerateTransition(RealmId, RealmId)` — Compatibility check
- **`TransitionZoneGenerator` service** (`Application/Services/TransitionZoneGenerator.cs`) — Full implementation with:
    - Linear interpolation of all 6 `RealmBiomeProperties` fields (temperature, aetheric intensity, humidity, light, scale, corrosion)
    - Compatibility delegation to `IBiomeAdjacencyService`
    - Realm property loading via `IRealmBiomeProvider`
    - Exhaustive structured Serilog logging at DEBUG, INFORMATION, and WARNING levels
    - Value clamping for aetheric intensity, humidity, light level, and corrosion rate

## Technical Notes

- **Interpolation formula:** `result = from + (to - from) × blend` applied to all numeric properties
- **Multi-zone blend distribution:** Evenly spaced: 1 room → 0.50, 2 rooms → 0.33/0.67, 3 rooms → 0.25/0.50/0.75
- **Room count limits:** Enforced 1-3 via `ArgumentOutOfRangeException`
- **Dependency pattern:** Constructor injection with optional `ILogger` (falls back to `NullLogger`)
