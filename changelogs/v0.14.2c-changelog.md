# v0.14.2c Changelog - Item Schema

## Summary

Implemented `items.schema.json` with comprehensive validation for general item definitions, including consumables, materials, quest items, keys, and miscellaneous goods.

## Added

### Schema Files
- **[config/schemas/items.schema.json](file:///Volumes/GitHub/github/southpawriter02/rune-rust/config/schemas/items.schema.json)** - JSON Schema Draft-07 for item configuration
  - `ItemDefinition` - Full item validation with category, rarity, stack/weight
  - `ConsumableEffect` - Effect types (Heal, RestoreMana, Buff, Cure, Damage, Utility)
  - `UseRequirement` - Level, class, and stat prerequisites
  - `VendorPrices` - Buy/sell price validation
  - `StatRequirements` - Minimum stat requirements

### Configuration Files
- **[config/items.json](file:///Volumes/GitHub/github/southpawriter02/rune-rust/config/items.json)** - 17 example items covering all categories:
  - 6 Consumables (potions, elixirs, cures)
  - 4 Materials (ore, leather, dust, dragon scale)
  - 2 QuestItems (ancient tome, royal seal)
  - 2 Keys (dungeon key, master key)
  - 3 Miscellaneous (torch, rope, lockpick)

### Validation Rules
| Property | Constraint | Description |
|----------|------------|-------------|
| `id` | `^[a-z][a-z0-9_]*$` | Lowercase with underscores |
| `category` | Enum | Consumable, Material, QuestItem, Key, Miscellaneous |
| `rarity` | Enum | Common, Uncommon, Rare, Epic, Legendary |
| `stackLimit` | 1-999 | Stack range validation |
| `weight` | >= 0 | Non-negative weight |
| `vendorPrices` | >= 0 | Non-negative buy/sell prices |
| `consumableEffect.type` | Enum | Heal, RestoreMana, RestoreStamina, Buff, Cure, Damage, Utility |
| `cleanseType` | Enum | Specific, AllNegative, Magical, Elemental |
