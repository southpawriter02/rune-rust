Here is the comprehensive implementation plan for v0.3.3b: The Air (Ambient Conditions).
This version implements the "weather" of the dungeon—room-wide, persistent effects that modify statistics and trigger periodic consequences. Unlike Hazards (which are objects), Conditions are properties of the space itself.

v0.3.3b: The Air (Ambient Conditions)
Goal: Implement the ConditionService to manage room-wide buffs/debuffs (Passive) and turn-based environmental ticks (Active), reusing the EffectScript engine from v0.3.3a.

1. Architecture & Data Flow
The Ambient Condition System operates on two distinct triggers: Presence (Stat Modifiers) and Time (Tick Effects).
	•	Data Structure:
	◦	AmbientCondition: A static definition (Name, Description, StatModifiers, TickScript).
	◦	Room: Holds a nullable ConditionId.
	•	Workflow: Room Entry (Passive)
	1	Trigger: Player enters a Room.
	2	Lookup: NavigationService checks Room.ConditionId.
	3	Notification: UI appends the Condition's description to the room text (e.g., "The air hums with a headache-inducing static.").
	4	Stat Integration: StatCalculationService queries ConditionService to apply penalties (e.g., -2 WITS).
	•	Workflow: Turn Start (Active)
	1	Trigger: CombatService.NextTurn starts.
	2	Execution: ConditionService executes the TickScript (e.g., STRESS:2) against the active combatant.
	3	Mitigation: Service checks for specific Traits/Gear that negate the condition.

2. Logic Decision Trees
A. Stat Calculation Logic
Input: Character, Attribute Type
	1	Get Base: Retrieve base Attribute value.
	2	Check Room: Does CurrentRoom have a Condition?
	3	Check Modifiers: Does the Condition modify this Attribute?
	◦	Yes: Apply modifier (e.g., Psychic Resonance -> -1 WILL).
	4	Check Mitigation: Does Character have [Psychic Shielding]?
	◦	Yes: Ignore modifier.
	5	Output: Return effective Attribute.
B. Tick Processing Logic
Input: Combatant, Condition
	1	Validate: Is Condition active?
	2	Check Script: Does Condition have a TickScript?
	◦	No: Return.
	3	Execute: Pass TickScript to EffectScriptExecutor (reused from v0.3.3a).
	◦	Example: Toxic Atmosphere -> DAMAGE:Poison:1d4.
	4	Log: "The toxic air burns [Name] for 3 damage."

B. Engine Layer (Service)
File: RuneAndRust.Engine/Services/ConditionService.cs
public class ConditionService : IConditionService
{
    private readonly EffectScriptExecutor _scriptExecutor;
    private readonly IConditionRegistry _registry;

    public async Task ProcessTurnTickAsync(Room room, Combatant target)
    {
        if (string.IsNullOrEmpty(room.ConditionId)) return;

        var condition = _registry.GetById(room.ConditionId);
        if (string.IsNullOrEmpty(condition.TickScript)) return;

        // Check Mitigation (e.g., Gear or Traits)
        if (IsImmune(target, condition)) return;

        // Execute Script
        await _scriptExecutor.ExecuteAsync(condition.TickScript, target, sourceName: condition.Name);
    }

    public int GetAttributeModifier(Room room, Attribute attribute)
    {
        if (string.IsNullOrEmpty(room.ConditionId)) return 0;
        var condition = _registry.GetById(room.ConditionId);

        return condition.StatModifiers.GetValueOrDefault(attribute, 0);
    }
}

7. Draft Changelog (v0.3.3b)
# Changelog: v0.3.3b - The Haze (Ambient Conditions)
**Release Date:** 2025-XX-XX

## Summary
The environment now pushes back. This update introduces **Ambient Conditions**—room-wide states that apply persistent stat penalties and turn-based damage. From the mind-shredding "Psychic Resonance" of Alfheim to the "Toxic Atmosphere" of Helheim, players must now contend with the air itself.

## Features
- **Ambient Condition System:**
    - **Passive Modifiers:** Conditions can debuff specific attributes (e.g., Deep Cold reduces FINESSE).
    - **Active Ticks:** Conditions execute effects at the start of every combat turn (e.g., accumulating Stress or Poison damage).
- **New Conditions:**
    - **Psychic Resonance:** A low hum that erodes sanity (-1 WILL, +2 Stress/turn).
    - **Toxic Atmosphere:** Corrosive air that damages lungs (1d4 Poison/turn).
    - **Deep Cold:** Numbing frost that slows reactions (-1 FINESSE, 1 Ice damage/turn).
- **UI Integration:** Active conditions are now displayed in the Room Description and HUD.

## Technical
- Implemented `ConditionService` reusing the `EffectScript` engine.
- Integrated environmental penalties into `StatCalculationService`.
