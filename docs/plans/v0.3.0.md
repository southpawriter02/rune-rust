Here is the comprehensive implementation plan for v0.3.0: The Trauma Economy.
This milestone introduces the psychological horror engine of Aethelgard. Unlike standard RPGs where health is the only survival metric, Rune & Rust tracks Psychic Stress (volatile, recoverable) and Runic Blight Corruption(permanent, transformative).

v0.3.0a: The Weight (Psychic Stress)
Goal: Implement the Psychic Stress resource, the Resolve Check mechanic (WILL-based resistance), and the UI feedback for rising stress levels.
1. Architecture & Data Flow
	•	Stress Accumulation: Stress is a negative resource (0-100). It does not deplete like Mana; it builds up.
	•	Resolve System: Before applying Stress, the engine runs a ResolveCheck.
	◦	Input: Incoming Stress Amount, Source Type (Environmental, Horror, Magic).
	◦	Process: Roll WILL attribute dice. Each success reduces incoming stress by 1.
	◦	Output: Net Stress applied to Character.

public void ApplyStress(Character target, int amount, string source)
{
    // 1. Resolve Check (Migration: Use DiceService)
    int will = target.GetAttribute(Attribute.Will);
    var roll = _dice.Roll(will, "Resolve Check");
    int mitigated = roll.Successes;

    // 2. Net Calculation
    int netStress = Math.Max(0, amount - mitigated);

    // 3. Application
    target.PsychicStress = Math.Clamp(target.PsychicStress + netStress, 0, 100);

    _logger.LogInformation("{Name} resisted {Mitigated}/{Amount} stress from {Source}. Current: {Current}",
        target.Name, mitigated, amount, source, target.PsychicStress);

    // 4. Threshold Check (Placeholder for v0.3.0c)
    if (target.PsychicStress >= 100)
        _eventBus.Publish(new BreakingPointEvent(target));
}

public void HandleBreakingPoint(Character character)
{
    var roll = _dice.Roll(character.GetAttribute(Attribute.Will), "Breaking Point");

    if (roll.Successes >= 3) // Standard DC
    {
        character.PsychicStress = 75;
        _statusEffects.Apply(character, StatusEffectType.Disoriented, 2);
        _logger.LogInformation("{Name} STABILIZED at Breaking Point.", character.Name);
    }
    else
    {
        character.PsychicStress = 50;
        var trauma = _traumaRegistry.GetRandomTrauma();
        character.Traumas.Add(trauma);
        _logger.LogWarning("{Name} BROKE! Acquired Trauma: {Trauma}", character.Name, trauma.Name);
    }
}

# Changelog: v0.3.0 - The Trauma Economy
**Release Date:** 2025-XX-XX

## Summary
The "Trauma Economy" is now live. Players must now manage **Psychic Stress** and **Corruption** alongside HP and Stamina. Mental health is no longer flavor text—it is a survival metric.

## Features
- **Psychic Stress (0-100):** Volatile mental damage. Accumulates from horrors and magic usage.
    - **Resolve Checks:** High WILL allows characters to resist incoming stress.
- **The Breaking Point:** Reaching 100 Stress triggers a critical resolve check. Failure results in permanent **Traumas**.
- **Corruption System:** Permanent spiritual decay from heretical actions.
    - **Tiers:** Tainted → Corrupted → Blighted → Fractured → Terminal.
    - **Penalties:** Corruption permanently reduces Max AP and Attributes.
- **Trauma Registry:** Added 10 permanent traumas (e.g., Paranoia, Hypervigilance) that alter gameplay rules.

## Technical
- Implemented `TraumaService` and `CorruptionState`.
- Updated `StatCalculationService` to apply Corruption penalties.
- Integrated `ResolveCheck` into damage pipelines.
