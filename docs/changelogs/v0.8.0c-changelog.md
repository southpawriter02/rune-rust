# v0.8.0c Changelog - Keyboard Shortcuts

## Added

### Enums
- `ShortcutContext` - Context types (Global, MainMenu, Game, Combat, Dialogue, Puzzle)

### Models
- `ShortcutDisplayInfo` - Display record with gesture formatting

### Services
- `IKeyboardShortcutService` - Interface for shortcut management
- `KeyboardShortcutService` - Implementation with context-aware processing

### ViewModels
- `ShortcutOverlayViewModel` - Context-grouped shortcut display

### Views
- `ShortcutOverlayWindow` - F1 help overlay (press any key to close)

### Default Bindings
| Context | Key | Action |
|---------|-----|--------|
| Global | F1 | Help / Shortcuts |
| Global | F5 | Quick Save |
| Global | F9 | Quick Load |
| Global | Escape | Close Window |
| Game | I | Toggle Inventory |
| Game | M | Toggle Map |
| Game | J | Toggle Quest Log |
| Game | C | Toggle Character |
| Combat | A | Attack Mode |
| Combat | M | Move Mode |
| Combat | Space | End Turn |
| Combat | Tab | Next Target |

### Tests
- 9 unit tests for KeyboardShortcutService

## Technical Notes
- Global shortcuts execute regardless of context
- Context shortcuts only execute in matching context
- Key+Modifiers stored as tuples for efficient lookup
- Conflict detection prevents duplicate bindings in same context
