# v0.13.0a Changelog: Status Effect Display

**Release Date:** 2026-01-20

---

## Summary

Implemented Status Effect Display UI components for visualizing buffs and debuffs during combat.

---

## Added

### Presentation Layer

- **StatusEffectDisplayDto** — Data transfer object for effect icon display
  - Effect ID, name, category, type, duration, stacks, damage type, source

- **EffectTooltipDto** — Data transfer object for tooltip content
  - Detailed effect information including stat modifiers and healing/damage per turn

- **StatusEffectDisplayConfig** — Configuration class with nested records
  - `IconConfig` — Effect type icon characters
  - `ColorConfig` — Category and urgency colors
  - `SymbolConfig` — Immunity/stack/duration symbols
  - `DisplayConfig` — Max effects per row, tooltip width

- **StatusEffectRenderer** — Rendering logic for effect icons and tooltips
  - Icon mapping for all `StatusEffectType` values
  - Damage type icons (F=Fire, I=Ice, P=Poison, L=Lightning, etc.)
  - Duration formatting by `DurationType`
  - Stack formatting with max indicator
  - Word-wrapped tooltip box rendering

- **StatusEffectDisplay** — Main UI component
  - Buff and debuff row rendering
  - Keyboard navigation for effect selection
  - Tooltip generation from `ActiveStatusEffect`
  - Immunity indicator display

### Configuration Files

- `config/status-effect-display.json` — Display configuration
- `config/schemas/status-effect-display.schema.json` — JSON Schema validation

### Tests

- `StatusEffectDisplayTests` (4 tests)
  - Buff row rendering with empty/multiple effects
  - Debuff row rendering with icons
  - Tooltip with stacking effects

- `StatusEffectRendererTests` (19 tests)
  - Effect display formatting
  - Icon mapping for all types
  - Duration/stack formatting
  - Damage type icons
