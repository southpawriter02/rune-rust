## Domain 4: Technology Constraints

### Canonical Ground Truth

**Medieval Baseline (POST-GLITCH):**

*   Tech level: ~1200-1400 CE equivalent
*   Lost precision measurement capability
*   Scavenged PRE-GLITCH artifacts (non-replicable)
*   Jötun-Readers lack comprehensive diagnostic tools

**Critical Constraint:**

*   **POST-GLITCH societies CANNOT perform precision measurements**
*   No systematic cartography/surveying capabilities
*   No precision gas analysis, temperature control, frequency monitoring
*   No percentage calculations, decimal precision, exact populations

**PRE-GLITCH Exception:**

*   Technical documentation describing **LOST capabilities** = APPROPRIATE
*   Examples: Ginnungagap Forges specs, L.O.K.I. Protocol diagnostics
*   Key distinction: **LOST capability documentation ≠ current measurement ability**

*   \[ \] No precise travel rates (e.g., "25-35 km/day")?

*   \[ \] PRE-GLITCH technical specs describing LOST capabilities?

*   \[ \] Empirical observations without precision ("sunrise to sunset")?

### Common Violations (Critical Pattern)

**Realm Overview Violations:**

*   Environmental specs: O₂ percentages, temperature ranges, humidity levels
*   Structural integrity: Degradation percentages, load capacity specs
*   Travel protocols: Exact km/day rates, exposure time windows
*   Population: Exact counts ("10,000+ residents")
*   Atmospheric: Gas concentrations (ppm), particle counts
*   Strategic: Percentage-based resource estimates

**Why This Matters:**

> These specifications represent **systematic cartographic/surveying capabilities** that POST-GLITCH medieval societies cannot perform. The measurements imply:

> *   Precision gas analyzers (O₂, H₂S, CO₂ measurement)

> *   Thermometric instruments (±1°C accuracy)

> *   Distance measurement tools (meter precision)

> *   Structural engineering diagnostics (percentage degradation)

> *   Population census systems (exact counts)

> *   Frequency analysis equipment (Hz measurement)

### Pattern Recognition Shortcut

**Ask: "Who could measure this in-universe?"**

*   Pre-Glitch Aesir with ODIN infrastructure → precision OK
*   Current Jötun-Readers without functioning sensors → precision suspicious
*   Medieval baseline populations navigating hazards → systematic quantification contradicts setting

### Remediation Strategies

**Option 1: Qualitative Conversion**

*   Replace percentages with descriptive bands (Low/Moderate/High)
*   Replace exact temperatures with sensation ("freezing", "scorching")
*   Replace distances with travel time ("half-day journey")
*   Replace populations with scale ("crowded settlement", "sparse")

**Option 2: Empirical Approximations**

*   Add uncertainty language ("roughly", "approximately", "estimated")
*   Use observational language ("several hours", "dozens of individuals")
*   Frame as imprecise field estimates with explicit limitations

**Option 3: PRE-GLITCH Attribution**

*   Attribute precise measurements to scavenged instruments
*   Add reliability caveats ("degraded accuracy", "unreliable readings")
*   Frame as fragmented data from defunct systems
