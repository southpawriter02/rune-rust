# (META) Setting Fundamentals: Canonical Ground Rules

---

## Purpose

This resource establishes canonical ground truth across **9 setting domains** and serves as the mandatory validation gate for Wave 2 content migration. It provides governance checklists to identify contradictions before content enters v5.0.

### Strategic Function

| Function | Description |
|----------|-------------|
| **Governance-first approach** | Document canonical ground rules before migrating content |
| **Contradiction detection** | Systematic validation questions per domain |
| **Wave 2 blocker** | No content migrates until validated against these fundamentals |
| **Clean slate guarantee** | Ensures v5.0 begins with accurate, consistent worldbuilding |

---

## How to Use This Resource

### For Content Creators

- **Before drafting:** Review canonical statements for relevant domains
- **During drafting:** Apply validation questions to catch contradictions early
- **Before publishing:** Verify no forbidden contradictions present

### For Wave 2 Migration

- **Pre-migration validation:** Required first step before voice/template checks
- **Domain identification:** Identify which domains apply to content type (see Quick Reference Table in Wave 2 Integration section)
- **Contradiction resolution:** Pause migration if contradictions found; correct in v4.0 first
- **Post-migration spot-check:** Verify no contradictions imported to v5.0

### For Wave 1.5 Remediation

- **Content audit:** Query v4.0 Resources against validation questions
- **Priority ranking:** Use domain criticality to prioritize corrections
- **Batch remediation:** Apply domain-specific correction guidelines
- **Quality assurance:** Validate corrected pages before marking "Ready for Wave 2"
